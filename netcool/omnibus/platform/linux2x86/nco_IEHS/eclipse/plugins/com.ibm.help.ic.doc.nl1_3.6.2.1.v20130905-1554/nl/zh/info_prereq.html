帮助系统的必备要求

安装 IBM® Eclipse Help System 时，您需要确保工作站、操作系统、浏览器和硬件满足该主题中所述的一些要求。

浏览器要求

要使用 IBM Eclipse Help System，需要安装 Microsoft® Internet Explorer V6.0 或 7.0 或者 Firefox 2.0 或 3.0。为了确保可以使用该版本 IBM Eclipse Help System 的所有功能，需要在浏览器中启用 cookie 和 JavaScript™，并且禁用浏览器的阻止弹出窗口功能。

操作系统环境要求

要使用 IBM Eclipse Help System，需要安装 Java™ Runtime Environment（JRE）V6.0。 请参阅下表，了解具体要求。

表 1. 操作系统环境要求
IBM Eclipse Help System 版本 操作系统 操作系统和 JRE 要求
适用于 Microsoft Windows®（32 位）的 IBM Eclipse Help System
  • Windows XP Professional（Intel 32 位）
  • Windows Vista Business（Intel 32 位）
  • 该版本的 IBM Eclipse Help System 包含嵌入式 IBM JRE。如果不想保留嵌入式 JRE，可以将其从软件包中除去。必须在系统上安装 JRE 6.0，才能运行 IBM Eclipse Help System。
适用于 Linux® 的 IBM Eclipse Help System
  • Red Hat Linux Enterprise 5（Intel® 32 位）
  • 该版本的 IBM Eclipse Help System 包含嵌入式 IBM JRE。如果不想保留嵌入式 JRE，可以将其从软件包中除去。必须在系统上安装 JRE 6.0，才能运行 IBM Eclipse Help System。
  • 在这些操作系统上，您可以通过 Konqueror 浏览器在基本模式下打开帮助系统。
  • 要在帮助系统中显示双向字符、双字节字符或多字节字符，必须在系统上安装相应的字体，并将其设置为浏览器的显示字体。
  • 要支持多种语言，可以在启动帮助系统之前，将系统语言环境设置为 UTF-8。
适用于 AIX® 的 IBM Eclipse Help System
  • AIX 5.3（64 位 PPC）
  • 该版本的 IBM Eclipse Help System 包含嵌入式 IBM JRE。如果不想保留嵌入式 JRE，可以将其从软件包中除去。必须在系统上安装 JRE 6.0，才能运行 IBM Eclipse Help System。

硬件要求

z/OS®

要在 z/OS 上运行 IBM Eclipse Help System，必须确保 z/OS 满足对实内存大小和区域大小的要求。

  • 实内存 = <basememory> + <plugsize> +<concurrentuser> * <responseBufferSize>
    • basememory 是启动 IBM Eclipse Help System 软件包必需的内存。在 z/OS 上是 90 MB。
    • plugsize 是文档插件的大小。
    • concurrentuser 是将访问 Help System 的并发用户数。
    • responseBufferSize 可以在 eclipse\plugins\org.eclipse.help.base_3.0.1 目录下的 preferences.ini 文件中找到。
  • 区域大小应设置为与实内存大小相同。

其他操作系统

所需的最低内存为 128 MB RAM。推荐使用 256 MB 或更高。

注： 需要根据文档插件的大小增加内存。 例如，如果文档插件为 100 MB，那么至少需要额外增加 80 MB RAM。

支持定制

虽然推荐保持缺省配置，但您也可以编辑 eclipse/plugins 目录下的 org.eclipse.help.base_<version number>/preferences.ini 文件来配置帮助系统。

请勿修改以下参数，否则帮助系统将无法正常运行：

always_external_browser
对于 Microsoft Windows 操作系统，该参数的缺省配置是 always_external_browser=true。 在单机模式下启动帮助系统时，该设置可以帮助您打开操作系统的缺省浏览器，而不是帮助系统中嵌入的浏览器。对于除 Windows 之外的其他操作系统，您都无需设置 always_external_browser=true。
