拡張検索の技法

引用符、ワイルドカード、ブール演算子を使用した拡張検索を実行すれば、より絞り込んだ検索結果を返すことができます。

  • インフォメーション・センター内で検索する場合は、語または句を二重引用符で囲むことで、その語または句を正確なストリングとして指定できます。

    例えば、「検索」フィールドに "log file" と入力した場合は、log file というストリングのすべての出現例を検索することになります。二重引用符を入力しなければ、log という語と file という語の両方のインスタンスが含まれているトピックが返されます。

  • 検索でアスタリスク (*) や疑問符 (?) などのワイルドカードを使用して、検索の範囲を広げることもできます。
    • 検索照会で複数の不明文字または可変文字に対応させる場合は、アスタリスク (*) を使用します。例えば、par* と入力すると、partly、participate、partial など、par で始まる語が含まれているトピックが返されます。
      注: 検索ストリングには 1 つの制限があります。アスタリスクを組み込んだ検索語を使用する場合は、アスタリスクの前に少なくとも 3 つの文字を入れる必要があります。同様に、ワイルドカードのない検索語は最低 3 文字で構成されていなければなりません。
    • 検索照会で 1 つの文字に対応させる場合は、疑問符 (?) を使用します。 例えば、par? と入力すると、part が含まれているトピックは返されますが、partial または partly が含まれているトピックは返されません。
  • ブール演算子 AND、OR、NOT を使用すれば、より具体的な照会を実行できます。以下は例です。
    database AND "log file"
    database という語と log file という句の両方が含まれているトピックに検索結果を絞り込みます。
    database OR "log file"
    database という語または log file という句のいずれかが含まれているトピックに検索結果を拡張します。
    database NOT "log file"
    database という語が含まれていて log file という句が含まれていないトピックに検索結果を絞り込みます。
    database OR "data base" NOT "log file"
    database という語または data base という句が含まれていて log file という句が含まれていないトピックを検索します。
