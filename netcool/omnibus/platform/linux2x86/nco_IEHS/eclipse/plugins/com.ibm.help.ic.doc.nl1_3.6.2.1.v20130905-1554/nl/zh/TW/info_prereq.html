說明系統的必備需求

設定 IBM® Eclipse Help System 時，請務必確定您的工作站、作業系統、瀏覽器及硬體皆符合本主題中所說明的需求。

瀏覽器需求

如果要使用 IBM Eclipse Help System，您需要安裝 Microsoft® Internet Explorer 6.0 版或 7.0 版，或 Firefox 2.0 或 3.0。為了確保此版本 IBM Eclipse Help System 的所有功能皆能使用，您需要啟用瀏覽器中的 Cookie 及 JavaScript™，並停用瀏覽器封鎖蹦現視窗的功能。

作業系統環境需求

如果要使用 IBM Eclipse Help System，您需要安裝「Java™ 執行時期環境 (JRE) 6.0 版」。 請參閱下表以取得詳細需求。

表 1. 作業系統環境需求
IBM Eclipse Help System 版本 作業系統 作業系統及 JRE 需求
IBM Eclipse Help System（適用於 Microsoft Windows® 32 位元）
  • Windows XP Professional Intel 32 位元
  • Windows Vista Business Intel 32 位元
  • 此版本的 IBM Eclipse Help System 包含內嵌的 IBM JRE。如果您不想要保留內嵌的 JRE，可以將其從套件中移除。您的系統必須安裝 JRE 6.0 才能執行 IBM Eclipse Help System。
IBM Eclipse Help System（適用於 Linux®）
  • Red Hat Linux Enterprise 5 Intel®32 位元
  • 此版本的 IBM Eclipse Help System 包含內嵌的 IBM JRE。如果您不想要保留內嵌的 JRE，可以將其從套件中移除。您的系統上必須有 JRE 6.0 才能執行 IBM Eclipse Help System。
  • 在這些作業系統上，您可以透過 Konqueror 瀏覽器以基本模式開啟說明系統。
  • 如果要在說明系統中顯示雙向字元、雙位元組字元或多位元組字元，請在系統上安裝對應的字型，並將該字型設為瀏覽器的顯示字型。
  • 如果要支援多種語言，請在啟動說明系統之前，將系統語言環境設成 UTF-8。
IBM Eclipse Help System（適用於 AIX®）
  • AIX 5.3 PPC 64 位元
  • 此版本的 IBM Eclipse Help System 包含內嵌的 IBM JRE。如果您不想要保留內嵌的 JRE，可以將其從套件中移除。您的系統必須安裝 JRE 6.0 才能執行 IBM Eclipse Help System。

硬體需求

z/OS®

如果要在 z/OS 上執行 IBM Eclipse Help System，務必確定 z/OS 符合實際記憶體大小及區域大小的需求。

  • 實際記憶體 = <basememory> + <plugsize> +<concurrentuser> * <responseBufferSize>
    • basememory 是啟動 IBM Eclipse Help System 套件所需的記憶體。在 z/OS 上其為 90 MB。
    • plugsize 是文件外掛程式的大小。
    • concurrentuser 是存取「說明系統」的並行使用者數量。
    • responseBufferSize 可在位於 eclipse\plugins\org.eclipse.help.base_3.0.1 目錄中的 preferences.ini 中找到。
  • 區域大小應設成與實際記憶體大小相同。

其他作業系統

需要的記憶體下限為 128 MB RAM。建議使用 256 MB 或以上。

註: 您需要根據文件外掛程式的大小來增加記憶體。例如，如果文件外掛程式的大小是 100 MB，請額外新增至少 80 MB RAM。

支援的自訂作業

建議您使用預設配置，但您可以藉由編輯 eclipse/plugins 目錄下的 org.eclipse.help.base_<版本號碼>/preferences.ini 檔來配置說明系統。

請勿修改下列參數；否則說明系統可能無法正常運作：

always_external_browser
對於 Microsoft Windows 作業系統，此參數的預設配置為 always_external_browser=true。 以獨立模式啟動說明系統時，這個設定會協助您開啟作業系統的預設瀏覽器，而非內嵌在說明系統中的瀏覽器。您不需要為 Windows 以外的作業系統設定 always_external_browser=true。
