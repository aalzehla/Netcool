Configurando as preferências de exportação

Esta seção descreve como configurar as preferências para a exportação do arquivo de regras.

Para configurar as preferências de diretório, clique no Preferências ou a partir do menu principal, clique em Arquivo > Preferênciase em seguida a partir da área da janela esquerda da janela Preferências, clique em Exportar.

  • No campo Incluir caminho, especifique o local das regras que foram instaladas com pacotes de arquivo de regras Netcool Include Library (NCiL) (se estiverem instaladas). A opção somente será necessária se você estiver exportando dados no formato de arquivo do Netcool Include Library e quando o caminho de diretório deve apontar para incluir regras. Como resultado, os novos arquivos de regras não necessitam ser editados manualmente. O valor padrão é NCHOME/omnibus/probes/arch.
  • No campo Formato padrão de arquivo de exportação, especifique um formato Windows/DOS (CRLF) ou um formato Unix (LF) para o arquivo de saída. Essa propriedade permite criar regras em uma máquina Windows e copiá-las para uma máquina UNIX sem executar um utilitário, como o dos2unix.
    Nota: A seleção aplica-se a todos os arquivos de saída e não está restrito aos arquivos de regras. O valor padrão é formato Unix.
  • No campo Tipos de alerta, especifique as opções da lista suspensa exibida quando um campo @Type é editado na visualização Detalhes.

As opções a seguir estarão disponíveis somente se o formato de exportação NCKL foi previamente selecionado:

  • Se a caixa de seleção Incluir descrições varbind for selecionada, as descrições varbind serão incluídas como documentação de comentário. O valor padrão está ativado.
  • Se a caixa de seleção Calcular um valor melhor para @AlertKey for selecionada, o valor @AlertKey será gerado a partir de um objeto-pai varbind. O valor padrão está ativado.
  • Se a caixa de seleção Set @Agent uma vez por empresa for selecionada, o campo @Agent será configurado para cada empresa. O campo @Agent está baseado nos nomes dos módulos MIB em processamento para o arquivo de regras. O valor padrão está ativado.
  • Se a caixa de seleção Incluir modelo @Class for selecionada, o campo @Class será configurado. Se a caixa de seleção Incluir Modelo @Class estiver selecionada, o campo @Class será configurado como 300. O valor padrão está ativado.
