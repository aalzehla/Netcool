Benutzervorgaben für den Export festlegen

In diesem Abschnitt wird beschrieben, wie Sie die Benutzervorgaben für den Export für Regeldateien festlegen.

Klicken Sie zum Festlegen der Benutzervorgaben für Verzeichnisse auf Benutzervorgaben oder klicken Sie im Hauptmenü auf Datei > Benutzervorgaben und dann im linken Bereich des Fensters Benutzervorgaben auf Exportieren.

  • Geben Sie im Feld Include-Pfad die Position der Regeln an, die mit den Paketen für NCiL-Regeldateien (NCiL = Netcool Include Library) installiert wurden (falls zutreffend). Diese Option ist nur dann erforderlich, wenn Sie Daten im NCiL-Dateiformat exportieren und wenn der Verzeichnispfad auf zu berücksichtigende Regeln verweisen muss. Daher müssen die neuen Regeldateien nicht manuell bearbeitet werden. Der Standardwert ist NCHOME/omnibus/probes/arch.
  • Geben Sie im Feld Standardmäßiges Exportdateiformat entweder ein Windows/DOS-Format (CRLF) oder ein Unix-Format (LF) für die Ausgabedatei an. Mit diesem Merkmal können Sie Regeln auf einer Windows-Maschine erstellen und auf eine Unix-Maschine kopieren, ohne ein Dienstprogramm wie dos2unix auszuführen.
    Anmerkung: Diese Auswahl gilt für alle Ausgabedateien und ist nicht auf die Regeldateien beschränkt. Der Standardwert ist das Unix-Format.
  • Geben Sie im Feld Alerttypen die Optionen aus der Dropdown-Liste an, die angezeigt werden, wenn das Feld @Type eines Traps in der Sicht Zusatzinformation bearbeitet wird.

Die folgenden Optionen sind nur dann verfügbar, wenn zuvor das NCKL-Exportformat ausgewählt wurde:

  • Wenn das Kontrollkästchen Beschreibungen von Variablenbindungen berücksichtigen ausgewählt wird, werden die Beschreibungen von Variablenbindungen als Kommentardokumentation hinzugefügt. Standardmäßig ist dieses Kontrollkästchen aktiviert.
  • Wenn das Kontrollkästchen Besseren Wert für @AlertKey berechnen ausgewählt wird, wird der Wert für @AlertKey aus dem übergeordneten Objekt einer Variablenbindung generiert. Standardmäßig ist dieses Kontrollkästchen aktiviert.
  • Wenn das Kontrollkästchen @Agent ein Mal pro Unternehmen festlegen ausgewählt wird, wird das Feld @Agent für jedes Unternehmen festgelegt. Das Feld @Agent basiert auf den Namen der MIB-Module, die für die Regeldatei verarbeitet werden. Standardmäßig ist dieses Kontrollkästchen aktiviert.
  • Wenn das Kontrollkästchen Schablone für @Class hinzufügen ausgewählt wird, wird das Feld @Class festgelegt. Wenn das Kontrollkästchen Schablone für @Class hinzufügen ausgewählt wird, wird das Feld @Class auf den Wert 300 festgelegt. Standardmäßig ist dieses Kontrollkästchen aktiviert.
