使用Netcool MIB Manager

可以使用 MIB 管理器 来解析 MIB 模块并创建 Netcool 规则文件。

MIB 管理器 包含一组基本 MIB 和 RFC MIB，它们位于 NCHOME/omnibus/var/mibmanager/mibs 目录中。 提供这些 MIB 是为了使您能够导入其他 MIB。MIB 管理器 在导入新 MIB 时会自动使用这些绑定的 MIB 来解析依赖关系。首次启动 MIB 管理器 可在“MIB 模块”视图中看见捆绑的 MIB 模块。

MIB 管理器 还包含表示导入捆绑的基本 MIB 的 XML 数据文件。MIB 管理器 使用这些 XML 文件存储它从原始文本 MIB 文件导入的 MIB 数据。

请将 MIB 添加到可从启动 MIB 管理器 的机器访问的目录中。所有基于标准的 MIB 都必须安装在一个单独的目录中，可将该目录放置在搜索路径中以便解析依赖关系。 缺省情况下，基本 SNMP MIB 安装在 NCHOME/omnibus/var/mibmanager/mibs/base 目录中，所有 RFC MIB 都安装在 NCHOME/omnibus/var/mibmanager/mibs/rfc 目录中。

Netcool MIB Manager 窗口

图 1. MIB 管理器 初始窗口
 1  工具栏
提供对最常用的 MIB 管理器 功能的访问权。
 2  透视图
可在两个透视图中的一个（MIB 管理器或 mib2rules）内查看 MIB 数据。MIB 管理器透视图是缺省透视图。mib2rules 透视图专用于对象标识 (OID)，是为熟悉 mib2rules 实用程序的用户提供的。
 3  “设备”视图
使用此视图可创建、更新和删除 MIB 管理器 设备。
 4  “详细信息”视图和“控制台”视图
“详细信息”视图显示了所选 MIB 元素的详细信息。 “控制台”视图显示 MIB 管理器 在当前实例正在运行时生成的所有日志消息。

使用此视图可搜索和删除 MIB 模块，以及搜索特定的 MIB 值。
 6  “OID 树”视图
此视图以树形方式显示所导入的 IBM 模块中的所有 MIB 对象。
