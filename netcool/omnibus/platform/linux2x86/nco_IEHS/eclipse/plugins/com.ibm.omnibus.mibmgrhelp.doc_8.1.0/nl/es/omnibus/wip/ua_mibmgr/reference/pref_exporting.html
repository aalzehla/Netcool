Establecimiento de las preferencias de exportación

En esta sección se describe cómo establecer las preferencias de exportación de archivos de reglas.

Para establecer las preferencias de directorio, pulse Preferencias o, desde el menú principal, pulse Archivo > Preferencias y, a continuación, en el panel de la izquierda de la ventana Preferencias, pulse Exportar.

  • En el campo Incluir vía de acceso, especifique la ubicación de las reglas que se han instalado con los paquetes de archivos de reglas de Netcool Include Library (NCiL) (si están instalados). Esta opción sólo se requiere si se exportan datos en el formato de archivo de Netcool Include Library y cuando la vía de acceso del directorio debe señalar a las reglas incluidas. Como consecuencia, no es necesario editar manualmente los nuevos archivos de reglas. El valor predeterminado es NCHOME/omnibus/probes/arch.
  • En el campo Formato de archivo de exportación predeterminado, especifique el formato Windows/DOS (CRLF) o el formato Unix (LF) para el archivo de salida. Esta propiedad permite crear reglas en una máquina Windows y copiarlas en una máquina UNIX sin ejecutar un programa de utilidad, como dos2unix.
    Nota: Esta selección se aplica a todos los campos de salida y no se limita a los archivos de reglas. El valor predeterminado es el formato Unix.
  • En el campo Tipos de alerta, especifique las opciones de la lista desplegable que se visualizan cuando un campo @Type de interrupción se edita en la vista Detalle.

Las opciones siguientes sólo estarán disponibles si previamente se ha seleccionado el formato de exportación NCKL:

  • Si se selecciona el recuadro de selección Incluir descripciones varbind, las descripciones varbind se añadirán como documentación de comentario. El valor predeterminado es habilitado.
  • Si se selecciona el recuadro de selección Calcular un valor mejor para @AlertKey, el valor @AlertKey se generará a partir de un objeto padre varbind. El valor predeterminado es habilitado.
  • Si se selecciona el recuadro de selección Establecer @Agent una vez por empresa, el campo @Agent se establecerá para cada empresa. El campo @Agent se basa en los nombres de los módulos MIB que se procesan para el archivo de reglas. El valor predeterminado es habilitado.
  • Si se selecciona el recuadro de selección Añadir plantilla @Class, se establecerá el campo @Class. Si se selecciona el recuadro de selección Añadir plantilla @Class, el campo @Class se establecerá en 300. El valor predeterminado es habilitado.
