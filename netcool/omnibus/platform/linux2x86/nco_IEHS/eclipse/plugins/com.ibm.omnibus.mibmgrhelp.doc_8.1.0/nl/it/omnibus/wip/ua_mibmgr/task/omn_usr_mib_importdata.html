Importazione di dati MIB

Come importare dati MIB SNMP in MIB Manager.

Prima di iniziare

Nella finestra Preferenze, specificare le ubicazioni per i MIB regolarmente utilizzati dal fornitore MIB. Verificare che i MIB si trovino in directory specifiche del fornitore e che qualsiasi numero di attrezzatura associato sia situato nelle sottodirectory. Verificare inoltre che queste directory non contengano file di testo non MIB. La maggior parte dei MIB vengono installati nella directory NCHOME/omnibus/var/mibmanager/mibs/base e tutti i MIB RFC vengono installati nella directory NCHOME/omnibus/var/mibmanager/mibs/rfc.

I MIB che si desidera importare potrebbero essere dipendenti dagli altri MIB. Per assicurarsi che MIB Manager possa cercare i MIB dipendenti durante l'esecuzione dell'importazione, impostare il percorso della ricerca sulle ubicazioni dei file MIB dipendenti.

Procedura

Per importare dati MIB:

  1. Fare clic su Importa oppure, dal menu, fare clic su File > Importa.
  2. Nel campo Directory, specificare l'ubicazione dei moduli MIB che si desidera importare.
  3. Per far sì che MIB Manager analizzi tutti i MIB trovati nelle sottodirectory disponibili, selezionare la casella di spunta Attraversa sottodirectory.
  4. Fare clic su Importa.

    La finestra Stato importazione viene visualizzata man mano che vengono analizzati i moduli MIB. Al lato sinistro di questa finestra viene visualizzato il numero di tipi di oggetti MIB diversi, man mano che ciascun oggetto viene rilevato durante l'analisi dei file MIB importati.

    Dopo l'analisi dei MIB, MIB Manager riesamina ogni modulo MIB ed analizza tutti gli oggetti MIB individuati. Dopo l'analisi di tutti gli oggetti, MIB Manager riesamina ogni MIB ed elabora le istruzioni di importazione per individuare eventuali moduli MIB di riferimento. Se non è possibile individuare i moduli MIB di riferimento nei moduli MIB importati in precedenza, MIB Manager ricerca le directory specificate nel percorso di ricerca. Al lato destro della finestra Stato importazione vengono visualizzate icone che riportano lo stato di eventuali istruzioni di importazione trovate nei MIB importati, come riportato nella seguente tabella:
    Icona Descrizione
    Indica che l'istruzione di importazione non è stata risolta.
    Indica che l'istruzione di importazione è stata risolta.
    Indica che non è stato possibile risolvere il modulo MIB a cui si fa riferimento nell'istruzione di importazione.

    È possibile espandere la struttura ad albero delle istruzioni di importazione per visualizzare quali moduli MIB utilizzano le istruzioni di importazione per fare riferimento ad altri moduli MIB.

    Importazione completa viene visualizzato nella parte superiore del frame di stato quando tutti i file MIB importati sono stati elaborati. È possibile visualizzare tutti i messaggi di stato generati durante l'importazione facendo clic su Visualizza cronologia stato e utilizzando l'elenco a discesa Visualizza per selezionare un'opzione di filtro.

  5. Fare clic su Elimina per chiudere la finestra Stato importazione.

Risultati

La vista Moduli MIB viene aggiornata automaticamente per visualizzare i nomi dei moduli MIB importati di recente. La vista Struttura OID si aggiorna automaticamente mostrando tutti gli oggetti MIB appena importati. Per ricreare la struttura ad albero, MIB Manager cerca ripetutamente nei nodi parent della struttura ad albero (iso, ccitt e joint-iso-ccitt) i nodi child fino a quando non trova un nodo child che non abbia elementi child associati. Se non viene rilevato alcun errore, una struttura ad albero MIB completamente compilata viene visualizzata nella vista Struttura ad albero OID, contenente gli oggetti analizzati.

Facendo clic su un oggetto nella vista Struttura ad albero OID, l'oggetto viene visualizzato automaticamente nella vista Moduli MIB. Nella vista Dettagli vengono visualizzate informazioni dettagliate sull'oggetto selezionato.

I moduli MIB importati vengono memorizzati in un file xml che rappresenta la struttura del modulo MIB. I file xml si trovano nella directory NCHOME/omnibus/var/mibmanager/data.

Se vengono rilevate istruzioni di importazione non risolte, contattare il fornitore responsabile di uno o più MIB mancanti, aggiungere il file o i file mancanti al percorso di ricerca e ripetere l'importazione.

Nota:

A volte i fornitori specificano nomi oggetto duplicati oppure specificano un nome oggetto identico ad un nome oggetto specificato in un RFC.

Ad esempio, un oggetto denominato system è definito nel modulo SNMPv2-MI' con OID 1.3.6.1.2.1.1 ed un altro oggetto denominato system è definito nel modulo WINDOWS-NT-PERFORMANCE, con OID 1.3.6.1.4.1.311.1.1.3.1.1.23. Inoltre, un oggetto denominato sysDescr è definito come system.1 nel modulo SNMPv2-MIB ed un altro oggetto denominato sysFileReadOperationsPerSec è definito come system.1 nel modulo WINDOWS-NT-PERFORMANCE. In questo caso, non è chiaro a quale oggetto parent siano associati gli oggetti child.

Quando viene individuato un oggetto con un nome parent duplicato, MIB Manager tenta di trovare il parent che è definito nello stesso modulo MIB. Nell'esempio precedente, si tratta di WINDOWS-NT-PERFORMANCE. Se nessuno dei due parent è definito nello stesso modulo MIB, MIB Manager effettua una ricerca per vedere se il parent è definito in altri moduli MIB dai quali dipende l'oggetto. MIB Manager inoltre identifica gli altri moduli MIB che fanno riferimento al parent e seleziona il nome parent a cui si fa riferimento più comunemente. Ciò potrebbe determinare, tuttavia, una struttura ad albero MIB compilata in modo errato e OID calcolati in modo non corretto. Pertanto, ogni volta che MIB Manager trova un nome oggetto duplicato, inserisce nel log un'avvertenza e registra le azioni intraprese per risolvere la duplicazione in un file di debug.
