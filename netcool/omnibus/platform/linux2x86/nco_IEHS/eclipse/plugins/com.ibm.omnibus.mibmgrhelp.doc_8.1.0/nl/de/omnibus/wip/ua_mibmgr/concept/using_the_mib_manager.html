IBM Tivoli Netcool/OMNIbus Version 8.1

Netcool MIB Manager verwenden

Sie können MIB Manager für das Parsing von MIB-Modulen und die Erstellung von Netcool-Regeldateien verwenden.

Zu MIB Manager gehören eine Reihe von Basis-MIBs und RCF-MIBs, die im Verzeichnis NCHOME/omnibus/var/mibmanager/mibs enthalten sind. Diese MIBs werden zur Verfügung gestellt, um Ihnen den Import zusätzlicher MIBs zu ermöglichen. MIB Manager verwendet die im Paket enthaltenen MIBs beim Import neuer MIBs zum Auflösen von Abhängigkeiten. Die im Paket enthaltenen MIB-Module sind in der Sicht MIB-Module zu sehen, wenn Sie MIB Manager erstmals starten.

In MIB Manager sind außerdem XML-Datendateien enthalten, die einen Import der im Paket enthaltenen Basis-MIBs darstellen. In MIB Manager werden diese XML-Dateien zum Speichern der aus den Rohtext-MIB-Dateien importierten MIB-Daten verwendet.

Speichern Sie Ihre MIBs in einem Verzeichnis, auf das von dem Server aus zugegriffen werden kann, von dem aus der MIB Manager gestartet wird. Alle auf Standards basierenden MIBs müssen in einem separaten Verzeichnis installiert werden, das für die einfache Auflösung von Abhängigkeiten im Suchpfad platziert werden kann. Standardmäßig werden die Basis-SNMP-MIBs im Verzeichnis NCHOME/omnibus/var/mibmanager/mibs/base installiert, alle RFC-MIBs im Verzeichnis NCHOME/omnibus/var/mibmanager/mibs/rfc.

Netcool MIB Manager-Fenster

Beim Start von MIB Manager werden die im Folgenden aufgeführten Elemente im Eingangsfenster von MIB Manager angezeigt. Die nachfolgende Abbildung veranschaulicht diese Elemente.

Abbildung 1. Eingangsfenster von MIB Manager
 1  Symbolleiste
Ermöglicht den Zugriff auf die gebräuchlichsten MIB Manager-Funktionen.
 2  Perspektiven
Bei der Anzeige von MIB-Daten stehen zwei Perspektiven zur Auswahl: MIB Manager und mib2rules. Die Perspektive MIB Manager ist die Standardperspektive. Die Perspektive mib2rules konzentriert sich auf Objekt-IDs (OIDs) und wird für die Benutzer bereitgestellt, die mit dem Dienstprogramm mib2rules vertraut sind.
 3  Sicht Gerät
In dieser Sicht können Sie MIB Manager-Geräte erstellen, aktualisieren und löschen.
 4  Sicht Details und Sicht Konsole
In der Sicht Details werden die Details ausgewählter MIB-Elemente angezeigt. In der Sicht Konsole werden alle Protokollnachrichten angezeigt, die von MIB Manager während der aktuellen Instanzausführung generiert werden.
 5  MIB-Module
Mithilfe dieser Sicht können Sie MIB-Module suchen und löschen sowie bestimmte MIB-Werte suchen.
 6  Sicht OID-Baumstruktur
Diese Sicht bietet eine Darstellung aller MIB-Objekte, die in den importierten MIB-Modulen enthalten sind.
  • Sicht "MIB-Module"
    Die Sicht MIB-Module bietet eine Darstellung aller importierten MIB-Module. In dieser Sicht können Sie MIB-Module suchen und löschen sowie bestimmte MIB-Werte suchen.
  • Sicht "OID-Baumstruktur"
    Die Sicht "OID-Baumstruktur" (OID = Objekt-ID) bietet eine Darstellung aller MIB-Objekte, die in den importierten MIB-Modulen enthalten sind.
  • MIB-Daten importieren
    Vorgehensweise beim Importieren von SNMP-MIB-Daten in MIB Manager.
  • MIB-Daten exportieren
    Vorgehensweise beim Exportieren von SNMP-MIB-Daten aus MIB Manager in verschiedene Formate.
  • SNMP-Traps bearbeiten
    Vorgehensweise beim Bearbeiten von SNMP-Traps.
  • SNMP-Traps generieren
    Vorgehensweise beim Generieren von SNMP-Traps. Diese Funktionalität kann auch zum Testen von Regeldateien verwendet werden, die von MIB Manager generiert wurden.
  • Geräte erstellen und löschen
    Ein MIB Manager-Gerät dient zum Gruppieren zugeordneter MIB-Module und zur Definition aller für den Export erforderlichen Objekte. Sie können Geräte in der MIB Manager-GUI erstellen, aktualisieren und löschen.
