使用 Netcool MIB Manager

您可以使用 MIB Manager 來剖析 MIB 模組及建立 Netcool 規則檔案。

MIB Manager 在 NCHOME/omnibus/var/mibmanager/mibs 目錄中包含一組基本 MIB 和 RFC MIB。 提供這些 MIB 是讓您能夠匯入其他 MIB。MIB Manager 會自動使用這些組合的 MIB，來在匯入新 MIB 期間解析相依關係。當您第一次啟動 MIB Manager 時，可以在「MIB 模組」視圖中看到組合的 MIB 模組。

MIB Manager 還包含 XML 資料檔案，它們代表已匯入組合基本 MIB。MIB Manager 使用這些 XML 檔來儲存它從原始文字 MIB 檔案中匯入的 MIB 資料。

將 MIB 新增至可從啟動 MIB Manager 之伺服器存取的目錄中。所有標準型 MIB 都必須安裝在可放置於搜尋路徑的個別目錄中，以便於解析相依關係。依預設，基本 SNMP MIB 安裝在 NCHOME/omnibus/var/mibmanager/mibs/base 目錄中，而所有 RFC MIB 安裝在 NCHOME/omnibus/var/mibmanager/mibs/rfc 目錄中。

Netcool MIB Manager 視窗

當您啟動 MIB Manager 時，下列元素會顯示在起始 MIB Manager 視窗中：數個元素顯示在起始 MIB Manager 視窗中。下圖顯示這些元素。

圖 1. MIB Manager 起始視窗
 1  工具列
可讓您存取最常用的 MIB Manager 功能。
 2  視景
您可以在下列兩個視景之一中檢視 MIB 資料：MIB Manager 或 mib2rules。MIB Manager 視景是預設視景。mib2rules 視景著重於物件 ID (OID)，且是為熟悉 mib2rules 公用程式的使用者而提供。
 3  裝置視圖
使用此視圖來建立、更新及刪除 MIB Manager 裝置。
 4  詳細資料視圖及主控台視圖
「詳細資料」視圖顯示所選取 MIB 元素的詳細資料。「主控台」視圖顯示 MIB Manager 在現行實例執行時產生的所有日誌訊息。
 5  MIB 模組
使用此視圖來搜尋及刪除 MIB 模組，以及搜尋特定的 MIB 值。
 6  OID 樹狀結構視圖
此視圖呈現所匯入 MIB 模組中的所有 MIB 物件。
  • MIB 模組視圖
    「MIB 模組」視圖可呈現所有匯入的 MIB 模組，並可讓您搜尋 MIB 模組、刪除 MIB 模組以及搜尋特定的 MIB 值。
  • OID 樹狀結構視圖
    「物件 ID (OID) 樹狀結構」視圖會呈現所有位於匯入 MIB 模組中的 MIB 物件。
  • 匯入 MIB 資料
    如何將 SNMP MIB 資料匯入 MIB Manager。
  • 匯出 MIB 資料
    如何將 MIB Manager 中的 SNMP MIB 資料匯出為各種格式。
  • 編輯 SNMP 設陷
    如何編輯 SNMP 設陷。
  • 產生 SNMP 設陷
    如何產生 SNMP 設陷。此功能還可用於測試 MIB Manager 產生的規則檔案。
  • 建立及刪除裝置
    MIB Manager 裝置用來分組關聯的 MIB 模組及定義需要匯出的所有物件。您可以在 MIB Manager GUI 中建立、更新及刪除裝置。
