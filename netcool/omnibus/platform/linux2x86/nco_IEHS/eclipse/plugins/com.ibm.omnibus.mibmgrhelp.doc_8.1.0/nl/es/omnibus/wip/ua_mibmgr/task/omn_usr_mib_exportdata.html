IBM Tivoli Netcool/OMNIbus Versión 8.1

Exportación de datos MIB

Cómo exportar datos MIB de SNMP desde MIB Manager en una variedad de formatos.

Procedimiento

Para exportar datos MIB:

  1. Pulse Exportar o, desde el menú, pulse Archivo > Exportar.
  2. En el campo Directorio, especifique el directorio de destino de los archivos exportados.
  3. En el campo Tipo de archivo, especifique un tipo de archivo para los archivos exportados.

    Debe utilizar la biblioteca Netcool Knowledge Library (NCKL) siempre que sea posible, porque los archivos de reglas NCKL se basan en la funcionalidad del dispositivo. Cuando se utiliza NCKL, se debe especificar el tipo de archivo Netcool Knowledge Library versión 3.x. De este modo se permite añadir los archivos generados a las instalaciones de la biblioteca Netcool Include Library existente.

    Las opciones siguientes están disponibles en la lista desplegable Tipo de archivo:

    Tipo de archivo Descripción
    CSV Seleccione esta opción para especificar que todos los datos del objeto se carguen en archivos CSV, de modo que se pueda visualizar mediante una aplicación de hoja de cálculo.
    Objetos de interrupción CSV Seleccione esta opción para especificar que todos los datos de interrupción y los datos de objeto de enlaces variables asociados se carguen en un sólo archivo CSV, de modo que pueda visualizarse mediante una aplicación de hoja de cálculo.
    HTML con marcos Seleccione esta opción para crear una página web que sea adecuada para la publicación.

    La vista de árbol MIB seleccionada se muestra en la parte izquierda de la página web generada. Puede navegar por la vista de árbol MIB expandiendo y contrayendo los nodos del árbol MIB. Cuando se selecciona un elemento del árbol MIB, su información asociada se visualiza en la parte derecha de la página web.

    A menudo, este formato puede crear un gran número de archivos de exportación, conteniendo cada uno de ellos un archivo HTML para cada objeto de la lista del árbol MIB. El directorio de destino se puede comprimir y se puede ubicar en cualquier servidor web, o las páginas se pueden visualizar directamente desde un disco abriendo el archivo index.html creado en el directorio de exportación.

    Si desea utilizar archivos HTML individuales con un navegador web, exporte en HTML con marcos y, a continuación, utilice los archivos del subdirectorio datos situado debajo del directorio de exportación.
    HTML sin marcos Seleccione esta opción para crear un único archivo denominado oids.html. Este archivo contendrá una lista de todos los objetos seleccionados para la salida, cada uno de ellos separado por reglas horizontales (mediante la etiqueta <HR>). Cada OID se delimita mediante una etiqueta ancla, de modo que cuando se cargue, el navegador se desplazará automáticamente de modo descendente hasta un OID que incluya un símbolo hash (#) en el URL, por ejemplo: file:c:/oids.html#1.3.6.1.4.1.9.

    Este archivo puede resultar muy grande si se exporta un gran número de objetos. Por consiguiente, este formato sólo debe utilizarse para exportar un número pequeño de interrupciones.
    Tabla de búsqueda Netcool Seleccione esta opción para dar salida a todos los valores especificados en una lista tabulada, mostrando los OID en la columna izquierda y los nombres de objeto en la columna derecha. Esta opción es adecuada para la inclusión en cualquier archivo de reglas.
    Netcool Include Library Seleccione esta opción para crear archivos de inclusión por separado para cada empresa, y para generar un único archivo para las interrupciones genéricas (es decir, para las interrupciones que no estén ubicadas bajo el subárbol de la empresa). Se supone que los archivos de inclusión se insertan en archivos snmp.rules existentes: los que ha creado MIB Manager previamente o los que ha creado la biblioteca Netcool Include Library (NCIL).
    Netcool Knowledge Library versión 1.1 Seleccione esta opción para generar archivos desde MIB Manager. Los archivos se incluirán en una implementación existente de Netcool Knowledge Library.
    Netcool Knowledge Library versión 3.x Seleccione esta opción para generar archivos desde MIB Manager. Los archivos se incluirán en una implementación existente de Netcool Knowledge Library.
    Reglas autónomas Seleccione esta opción para crear un único archivo de reglas que contenga todas las interrupciones seleccionadas. Puede especificar este archivo en el archivo de propiedades de cualquier analizador de interrupciones SNMP de Tivoli Netcool/OMNIbus.
    Archivos de texto individuales El formato de archivos de texto individuales se utiliza para el menú del botón derecho del ratón Netcool Event List.

    Cuando se exporta en archivos de texto individuales, se da salida a cada OID en un archivo de texto por separado, por ejemplo 1.3.6.1.4.1.9.1.1.txt. Entonces, se puede añadir una herramienta externa al menú del botón derecho del ratón que utiliza nco_message para mostrar toda la información de objeto para un suceso que tiene el OID especificado en un campo, por ejemplo: cat 1.3.6.1.4.1.9.1.txt |nco_message stdin title "trap info".
    Tivoli Universal Agent El formato de Tivoli Universal Agent exporta archivos .mdl y trapcnfg basándose en el contenido seleccionado.
    Se pueden seleccionar varios subárboles cuando se selecciona este formato para exportar archivos. Si se seleccionan varios subárboles, se crearán archivos .mdl y trapcnfg para todos los objetos ubicados bajo un elemento seleccionado en el árbol MIB.
    Nota: Las tres primeras letras del nombre de aplicación, que siguen a la etiqueta //APPL en el archivo .mdl, deben ser exclusivos dentro de una empresa. MIB Manager intenta cumplir este requisito pero no puede tener en cuenta otros agentes que puedan estar en ejecución. Es posible que tenga que modificar el archivo .mdl para asegurarse de que el nombre de la aplicación sea exclusivo en la empresa.
  4. En el campo Ámbito, utilice la lista desplegable para especificar si MIB Manager da salida sólo a interrupciones, sólo a objetos o a todos los OID.

    Las opciones disponibles dependen del tipo de archivo que se está exportando.

  5. Para limitar el número de objetos incluidos en el archivo generado, seleccione el recuadro de selección Sólo subárbol(es) seleccionado(s).
  6. Pulse Exportar para exportar los archivos al directorio de destino.
