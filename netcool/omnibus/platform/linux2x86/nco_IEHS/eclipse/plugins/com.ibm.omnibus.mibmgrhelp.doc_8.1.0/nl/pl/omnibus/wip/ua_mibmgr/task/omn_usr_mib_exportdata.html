IBM Tivoli Netcool/OMNIbus wersja 8.1

Eksportowanie danych MIB

Informacje na temat sposobu eksportowania danych MIB SNMP z programu MIB Manager do różnych formatów.

Procedura

Aby wyeksportować dane MIB:

  1. Kliknij opcję Eksport lub z menu wybierz opcję Plik > Eksport.
  2. W polu Katalog podaj katalog docelowy dla eksportowanych plików.
  3. W polu Typ pliku podaj typ pliku dla eksportowanych plików.

    Należy użyć biblioteki Netcool Knowledge Library (NCKL) gdzie tylko możliwe, ponieważ pliki reguł NCKL są tworzone na podstawie funkcjonalności urządzenia. Podczas korzystania z NCKL, należy określić typ pliku Netcool Knowledge Library wersja 3.x. Dzięki temu wygenerowane pliki zostają dodane do istniejących instalacji biblioteki Netcool Include Library.

    Dostępne są poniższe opcje z listy rozwijanej Typ pliku:

    Typ pliku Opis
    CSV Wybierz tę opcję, aby określić, że wszystkie dane obiektu są ładowane do plików CSV, dzięki czemu można je wyświetlać za pomocą aplikacji arkusza kalkulacyjnego.
    CSV - pułapka i obiekty Wybierz tę opcję, aby określić, że wszystkie dane pułapki i powiązane dane powiązań zmiennych obiektu są zapisywane w pojedynczym pliku CSV, dzięki czemu można je wyświetlać za pomocą aplikacji arkusza kalkulacyjnego.
    HTML z ramkami Wybierz tę opcję, aby utworzyć stronę WWW, która jest odpowiednia do opublikowania.

    Wybrany widok drzewa MIB jest wyświetlany po lewej stronie wygenerowanej strony WWW. Można poruszać się po widoku drzewa MIB, rozwijając i zwijając węzły w drzewie MIB. Jeśli zaznaczono pozycję na drzewie MIB, powiązane z nią informacje są wyświetlane w prawej części strony WWW.

    Ten format może często powodować powstanie dużej liczby plików do eksportowania, z których każdy zawiera plik HTML dla każdego obiektu w drzewie MIB. Katalog docelowy może być skompresowany i umieszczony na jakimkolwiek serwerze WWW. Strony mogą być także wyświetlane bezpośrednio z dysku. W tym celu należy otworzyć plik index.html, który został utworzony w katalogu eksportowym.

    Jeśli chcesz wyświetlić pojedyncze pliki HTML za pomocą przeglądarki WWW, wyeksportuj pliki do formatu HTML z ramkami, a następnie otwórz pliki w podkatalogu data, który znajduje się w katalogu eksportowym.
    HTML bez ramek Wybierz tę opcję, aby utworzyć pojedynczy plik o nazwie oids.html. Ten plik zawiera listę wszystkich obiektów, z których mają być wygenerowane dane wyjściowe. Każdy obiekt oddzielony jest liniami poziomymi (za pomocą znacznika <HR>). Każdy identyfikator OID jest ujęty w znaczniku odsyłacza, dzięki czemu okno przeglądarki podczas ładowania automatycznie przewija się do identyfikatora OID, który zawiera w swoim adresie URL symbol #, na przykład: plik:c:/oids.html#1.3.6.1.4.1.9.

    Ten plik może być bardzo duży w przypadku eksportowania dużej liczby obiektów. Dlatego ten format powinien być używany tylko w przypadku eksportowania niewielkiej liczby pułapek.
    Tabela wyszukiwania Netcool Wybierz tę opcję, aby wyświetlić wszystkie podane wartości na liście z tabulatorami, gdzie identyfikatory OID są wyświetlane w kolumnie po lewej stronie, a nazwy obiektów w kolumnie po prawej stronie. Ta opcja jest odpowiednia, jeśli celem jest dołączenie informacji do jakiegokolwiek pliku reguł.
    Netcool Include Library Wybierz tę opcję, aby utworzyć osobne pliki dla każdego typu zarządzanego urządzenia, oraz aby wygenerować pojedynczy plik dla pułapek ogólnych (czyli dla jakichkolwiek pułapek, które nie znajdują się poniżej poddrzewa zarządzanego urządzenia). Zakłada się, że pliki są wstawiane w istniejące pliki snmp.rules, czyli takie, które zostały wcześniej utworzone przez program MIB Manager lub Netcool Include Library (NCIL).
    Netcool Knowledge Library wersja 1.1 Wybierz tę opcję, aby wygenerować pliki wyjściowe z programu MIB Manager. Pliki te zostaną uwzględnione w istniejącej implementacji Netcool Knowledge Library.
    Netcool Knowledge Library wersja 3.x Wybierz tę opcję, aby wygenerować pliki wyjściowe z programu MIB Manager. Pliki te zostaną uwzględnione w istniejącej implementacji Netcool Knowledge Library.
    Reguły autonomiczne Wybierz tę opcję, aby utworzyć pojedynczy plik reguł, który zawiera wszystkie wybrane pułapki. Można określić ten plik w pliku właściwości dowolnej sondy pułapki SNMP programu Tivoli Netcool/OMNIbus.
    Pojedyncze pliki tekstowe Format pojedynczych plików tekstowych jest używany dla menu kontekstowego listy zdarzeń Netcool Event List.

    W przypadku eksportowania do indywidualnych plików tekstowych, każdy identyfikator OID jest wyprowadzany do pojedynczego pliku tekstowego, na przykład 1.3.6.1.4.1.9.1.1.txt. Narzędzie zewnętrzne może być następnie dodane do menu kontekstowego, które używa procedury nco_message, aby wyświetlić wszystkie informacje o obiekcie dla zdarzenia, które ma zdefiniowany identyfikator OID w polu, na przykład: cat 1.3.6.1.4.1.9.1.txt |nco_message stdin title "trap info".
    Tivoli Universal Agent Format Tivoli Universal Agent (uniwersalny agent Tivoli) eksportuje pliki .mdl i trapcnfg na podstawie wybranej treści.
    Można wybrać wiele poddrzew, gdy zostanie wybrany ten format eksportowania plików. Jeśli wybrano wiele poddrzew, pliki .mdl i trapcnfg są tworzone dla wszystkich obiektów znajdujących się poniżej zaznaczonego elementu w drzewie MIB.
    Uwaga: Pierwsze trzy litery nazwy aplikacji następujące po znaczniku //APPL w pliku .mdl muszą być unikalne w ramach zarządzanego urządzenia. Program MIB Manager próbuje spełniać to wymaganie, lecz nie może mieć wiedzy dotyczącej innych agentów, które mogą być uruchomione. Może zaistnieć potrzeba modyfikacji pliku .mdl, aby się upewnić, że nazwa aplikacji jest unikalna w zarządzanym urządzeniu.
  4. W polu Zasięg użyj listy rozwijanej, aby określić, czy program MIB Manager wyprowadza tylko pułapki, tylko obiekty, czy wszystkie identyfikatory OID.

    Opcje, które są dostępne, zależą od typu eksportowanego pliku.

  5. Aby ograniczyć liczbę obiektów uwzględnionych w generowanym pliku, zaznacz pole wyboru Tylko wybrane poddrzewa.
  6. Kliknij opcję Eksportuj, aby wyeksportować pliki do katalogu docelowego.
