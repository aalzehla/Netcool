设置导出首选项

本节描述了如何设置规则文件导出首选项。

要设置目录首选项，请单击 首选项，或在主菜单中单击文件 > 首选项，然后在“首选项”窗口的左窗格中单击导出。

  • 在包含路径字段中，指定已随 Netcool 包含库 (NCiL) 规则文件包安装的规则的位置（如果规则文件包已安装）。仅当您要以 Netcool 包含库文件格式导出数据以及目录路径必须指向已包含的规则时，才需要指定此选项。指定此选项后，就不需要手动编辑新的规则文件。缺省值为 NCHOME/omnibus/probes/arch。
  • 在缺省导出文件格式字段中，为输出文件指定 Windows/DOS 格式 (CRLF) 或 Unix 格式 (LF)。此属性使您无需运行实用程序（例如 dos2unix）就能够在 Windows 机器上创建规则并将它们复制到 Unix 机器。
    注： 此选项应用于所有输出文件，并不限于规则文件。缺省值为 Unix 格式。
  • 在警报类型字段中，指定在“详细信息”视图中编辑 @Type 字段时显示的下拉列表选项。

仅当先前选择了 NCKL 导出格式时，以下选项才可用：

  • 如果选中了包含 varbind 描述复选框，那么会将 varbind 描述添加为注释文档。缺省值是已启用。
  • 如果选中了计算更佳的 @AlertKey 值复选框，那么将根据 varbind 父对象生成 @AlertKey 值。缺省值是已启用。
  • 如果选中了为每个企业设置 @Agent 复选框，那么将为每个企业设置 @Agent 字段。@Agent 字段基于为规则文件处理的 MIB 模块的名称。缺省值是已启用。
  • 如果选中了添加 @Class 模板复选框，那么将设置 @Class 字段。如果选中了添加 @Class 模板复选框，那么会将 @Class 字段设置为 300。缺省值是已启用。
