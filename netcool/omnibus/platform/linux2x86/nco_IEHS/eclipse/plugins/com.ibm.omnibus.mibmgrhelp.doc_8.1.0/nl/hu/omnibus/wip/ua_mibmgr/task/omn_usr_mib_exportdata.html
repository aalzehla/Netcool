IBM Tivoli Netcool/OMNIbus 8.1. változat

MIB adatok exportálása

SNMP MIB adatok exportálása a MIB Manager termékből különböző formátumokban.

Eljárás

MIB adatok exportálása:

  1. Kattintson az Exportálás gombra, vagy a menüben kattintson a Fájl > Exportálás menüpontra.
  2. A Könyvtár mezőben adja meg az exportált fájlok célkönyvtárát.
  3. A Fájltípus mezőben adja meg az exportált fájlok fájltípusát.

    Ahol csak lehet, a Netcool Knowledge Library (NCKL) könyvtárat használja, mivel az NCKL szabályfájlok az eszköz működésére épülnek. Az NCKL használatakor meg kell adnia a Netcool Knowledge Library v3.x fájltípust. Ez lehetővé teszi az előállított fájlok hozzáadását a meglévő Netcool Include Library telepítésekhez.

    Az alábbi beállítások állnak rendelkezésre a Fájltípus legördülő listában:

    Fájltípus Leírás
    CSV Ha ezt a beállítást adja meg, akkor minden objektumadat CSV fájlokba kerül betöltésre, így megtekinthető lesz táblázatkezelő alkalmazással.
    CSV Trap objektumok Ha ezt a beállítást adja meg, akkor minden trap adat és az ehhez tartozó különböző kötések objektumadatai egyetlen SCV fájlba kerülnek betöltésre, amely így megtekinthető lesz táblázatkezelő alkalmazással.
    HTML keretekkel Ezzel a beállítással egy közzétételre alkalmas weboldalt hoz létre.

    A kijelölt MIB fanézet az előállított weboldal bal oldalán jelenik meg. Az MIB fa nézeten az MIB fa csomópontjainak kibontásával és összehúzásával haladhat végig. Amikor kiválaszt egy elemet az MIB fából, akkor a társított információk a weboldal jobb oldalán jelennek meg.

    Ez a formátum gyakran hozhat létre nagyszámú exportfájlt, melyek mindegyike egy HTML fájlt tartalmaz, az MIB fa egyes objektumainak megfelelően. A célkönyvtárat be lehet csomagolni zip fájlba, és elhelyezni egy tetszőleges webkiszolgáló alatt, vagy az oldalakat közvetlenül is meg lehet tekinteni egy lemezről, az exportálási könyvtárban létrehozott index.html fájl megnyitásával.

    Ha egyéni HTML fájlokat akar használni webböngészővel, akkor exportáljon a HTML keretekkel elembe, majd használja az exportálási könyvtár alatt található data alkönyvtár fájljait.
    HTML keretek nélkül Ezzel a beállítással hozhat létre egy oids.html nevű egyedülálló fájlt. Ez a fájl fogja tartalmazni a kimenetre kiválasztott objektumok teljes listáját, vízszintes vonalakkal elválasztva (a <HR> címke használatával). Minden egyes OID horgony címkék közé van zárva, így betöltéskor a böngésző automatikusan legörget az URL címében kettőskereszt (#) szimbólumot tartalmazó OID azonosítóhoz, mint a file:c:/oids.html#1.3.6.1.4.1.9.

    Ez a fájl nagyszámú objektum exportálásakor nagyon nagy lehet. Ezért ezt a formátumot csak kisszámú trap exportálására szabad használni.
    Netcool kikeresési táblázat Ezzel a beállítással minden megadott érték egy lapokból álló listába kerül, az OID azonosítók a bal oldali oszlopban az objektumnevek pedig a jobb oldali oszlopban jelennek meg. Ez megfelelő formátum a szabályfájlokba való beillesztéshez.
    Netcool Include Library Ezzel a beállítással különálló include fájlokat hozhat létre az egyes vállalatokhoz, és önálló fájlt hozhat létre az általános trap-ekhez (vagyis azokhoz a trap-ekhez, amelyek nem a vállalati részfa alatt találhatók). Feltételezzük, hogy az include fájlok egy meglévő snmp.rules fájlba kerülnek beillesztésre, amelyeket még korábban a MIB Manager vagy a Netcool Include Library (NCIL) hozott létre.
    Netcool Knowledge Library 1.1 változat Ezzel a beállítással a MIB Manager termékből nyerheti ki a fájlokat. A fájlok egy meglévő Netcool Knowledge Library megvalósítás részei lesznek.
    Netcool Knowledge Library 3.x változat Ezzel a beállítással a MIB Manager termékből nyerheti ki a fájlokat. A fájlok egy meglévő Netcool Knowledge Library megvalósítás részei lesznek.
    Önálló szabályok Ezzel a beállítással egy önálló szabályfájlt hozhat létre, amely az összes kijelölt trap-et tartalmazza. Ezt a fájlt bármelyik Tivoli Netcool/OMNIbus SNMP trap vizsgáló tulajdonságfájljában megadhatja.
    Egyedi szövegfájlok Az egyedi szövegfájlok formátum a Netcool eseménylista menüjénél használatos.

    Amikor egyedi szövegfájlokba exportál, akkor minden OID egy különálló szövegfájlba kerül, például 1.3.6.1.4.1.9.1.1.txt. Ezután felvehető egy külső eszköz a menühöz, amely a nco_message fájlt használja a mezőben megadott OID azonosítójú esemény összes objektuminformációjának megjelenítéséhez, például: cat 1.3.6.1.4.1.9.1.txt |nco_message stdin title "trap info".
    Tivoli Universal Agent A Tivoli Universal Agent formátum .mdl és trapcnfg fájlokat exportál, a kijelölt tartalomtól függően.
    Ha ezt a formátumot választja a fájlok exportálásához, akkor több részfa is kiválasztható. Ha több részfát választ ki, akkor .mdl és trapcnfg fájlok jönnek létre az MIB fában kijelölt elem alatt található összes objektumhoz.
    Megjegyzés: Az alkalmazásnév első három betűjének, melyeket az //APPL címke követ a .mdl fájlban, egyedinek kell lennie a vállalaton belül. A MIB Manager megpróbál megfelelni ennek a követelménynek, de a többi, futó ügynökre nem tud figyelni. Lehet, hogy módosítani kell a .mdl fájlt, így biztosítva, hogy az alkalmazásnév egyedi legyen a vállalaton belül.
  4. A Hatókör mezőben a legördülő lista használatával adhatja meg, hogy a MIB Manager kimenete csak trap-ek, csak objektumok vagy az összes OID legyen.

    Az elérhető lehetőségek az exportált fájltípustól függnek.

  5. Ha korlátozni akarja az objektumok számát az előállított fájlban, akkor jelölje be a Csak a kijelölt részfák jelölőnégyzetet.
  6. A fájlok exportálásához a célkönyvtárba kattintson az Exportálás gombra.
