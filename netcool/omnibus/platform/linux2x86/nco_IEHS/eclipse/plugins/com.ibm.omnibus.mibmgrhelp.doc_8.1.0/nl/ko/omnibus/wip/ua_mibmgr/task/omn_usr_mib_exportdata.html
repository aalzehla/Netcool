IBM Tivoli Netcool/OMNIbus 버전 8.1

MIB 데이터 내보내기

MIB Manager의 SNMP MIB 데이터를 다양한 형식으로 내보내는 방법을 설명합니다.

프로시저

MIB 데이터를 내보내려면 다음을 수행하십시오.

  1. 내보내기를 클릭하거나, 메뉴에서 파일 > 내보내기를 클릭하십시오.
  2. 디렉토리 필드에서는 내보내는 파일의 대상 디렉토리를 지정하십시오.
  3. 파일 유형 필드에는 내보내는 파일의 파일 유형을 지정하십시오.

    NCKL(Netcool Knowledge Library) 규칙 파일은 디바이스의 기능성을 기반으로 하므로 가능할 때면 언제나 NCKL을 사용해야 합니다. NCKL을 사용하는 경우 Netcool Knowledge Library 버전 3.x 파일 유형을 지정해야 합니다. 이를 사용하면 생성된 파일을 기존 Netcool Include Library 설치에 추가할 수 있습니다.

    파일 유형 드롭 다운 목록에서 다음 옵션을 사용할 수 있습니다.

    파일 유형 설명
    CSV 모든 오브젝트 데이터를 CSV 파일로 로드하도록 지정하려면 이 옵션을 선택하십시오. 스프레드시트 애플리케이션을 사용하여 볼 수 있습니다.
    CSV 트랩 오브젝트 모든 트랩 데이터 및 연관된 가변 바인딩 오브젝트 데이터를 단일 CSV 파일로 로드하도록 지정하려면 이 옵션을 선택하십시오. 스프레드시트 애플리케이션을 사용하여 볼 수 있습니다.
    프레임이 있는 HTML 게시하기에 적합한 웹 페이지를 작성하려면 이 옵션을 선택하십시오.

    선택한 MIB 트리 보기가 생성된 웹 페이지의 왼쪽에 표시됩니다. MIB 트리에서 노드를 접었다 펴면서 MIB 트리 보기를 탐색할 수 있습니다. MIB 트리에서 요소를 선택하면, 웹 페이지 오른쪽에 연관된 정보가 표시됩니다.

    이 형식은 종종 많은 수의 내보내기 파일을 작성할 수 있으며, 각각에는 MIB 트리에 나열된 각 오브젝트에 대한 하나의 HTML 파일이 포함되어 있습니다. 대상 디렉토리는 압축되어 웹 서버 아래에 배치되거나, 내보내기 디렉토리에서 작성된 index.html 파일을 열어 페이지를 직접 볼 수 있습니다.

    웹 브라우저와 함께 각 HTML 파일을 사용하려면, 프레임이 있는 HTML로 내보낸 후 내보내기 디렉토리에 있는 data 서브디렉토리에서 파일을 사용하십시오.

    프레임이 없는 HTML oids.html이라는 단일 파일을 작성하려면 이 옵션을 선택하십시오. 이 파일에는 출력용으로 선택된 모든 오브젝트 목록이 포함되며, 각각은 수직 규칙(<HR> 태그 사용)으로 구분됩니다. 각 OID는 앵커 태그로 묶이며, 로드될 때 URL에 해시 기호(#)가 있는 OID(예: file:c:/oids.html#1.3.6.1.4.1.9)로 브라우저가 자동 스크롤됩니다.

    많은 수의 오브젝트를 내보내면 이 파일이 너무 커질 수 있습니다. 그러므로 이 형식은 적은 수의 트랩을 내보내는 경우에만 사용해야 합니다.

    Netcool 검색 테이블 탭 지정된 목록에서 지정된 모든 값을 출력하려면 이 옵션을 선택하십시오. 왼쪽 컬럼에는 OID가 표시되고 오른쪽 컬럼에는 오브젝트 이름이 표시됩니다. 규칙 파일에 포함시킬 때 적합합니다.
    Netcool Include Library 각 엔터프라이즈에 대해 별도의 include 파일을 작성하고, 일반 트랩(즉 엔터프라이즈 서브트리 아래에 없는 트랩의 경우)에 대해 단일 파일을 생성하려면 이 옵션을 선택하십시오. include 파일은 기존 snmp.rules 파일(이전에 MIB Manager에서 작성된 파일 또는 NCIL(Netcool Include Library)에서 작성된 파일)에 삽입된다고 가정합니다.
    Netcool Knowledge Library 버전 1.1 MIB Manager에서 파일을 출력하려면 이 옵션을 선택합니다. 파일은 기존의 Netcool Knowledge Library 구현에 포함됩니다.
    Netcool Knowledge Library 버전 3.x MIB Manager에서 파일을 출력하려면 이 옵션을 선택합니다. 파일은 기존의 Netcool Knowledge Library 구현에 포함됩니다.
    독립형 규칙 선택한 모든 트랩이 포함되는 단일 규칙 파일을 작성하려면 이 옵션을 선택하십시오. Tivoli Netcool/OMNIbus SNMP 트랩 프로브의 특성 파일에서 이 파일을 지정할 수 있습니다.
    Individual Text Files Individual Text Files 형식은 Netcool Event List 마우스 오른쪽 단추 클릭 메뉴에 사용됩니다.

    Individual Text File로 내보내면, 각 OID는 별도의 텍스트 파일(예: 1.3.6.1.4.1.9.1.1.txt)로 출력됩니다. 그런 다음 nco_message를 사용하여 필드에서 지정된 OID가 있는 이벤트에 대한 모든 오브젝트 정보를 표시하는 마우스 오른쪽 단추 클릭 메뉴에 외부 도구를 추가할 수 있습니다(예: cat 1.3.6.1.4.1.9.1.txt |nco_message stdin title "trap info").
    Tivoli Universal Agent Tivoli Universal Agent 형식은 선택한 내용을 기초로 .mdl 및 trapcnfg 파일을 내보냅니다.
    파일을 내보낼 때 이 형식을 선택하면 여러 서브트리를 선택할 수 있습니다. 여러 서브트리가 선택되면, MIB 트리에서 선택한 항목 아래에 있는 모든 오브젝트에 대해 .mdl 및 trapcnfg 파일이 작성됩니다.
    참고: 뒤에 .mdl 파일의 //APPL 태그가 오는 애플리케이션 이름의 처음 세 글자는 엔터프라이즈 내에서 고유해야 합니다. MIB Manager는 이 요구 사항을 준수하지만 실행 중인 다른 에이전트를 인식할 수 없습니다. 애플리케이션 이름이 엔터프라이즈 내에서 고유하도록 .mdl 파일을 수정해야 합니다.
  4. 범위 필드에서 드롭 다운 목록을 사용하여 MIB Manager가 트랩만, 오브젝트만 또는 모든 OID를 출력할지 여부를 지정하십시오.

    사용 가능한 옵션은 내보내는 파일 유형에 따라 달라집니다.

  5. 생성된 파일에 포함시킬 오브젝트 수를 제한하려면 선택된 서브트리만 선택란을 선택하십시오.
  6. 내보내기를 클릭하여 대상 디렉토리로 파일을 내보내십시오.
