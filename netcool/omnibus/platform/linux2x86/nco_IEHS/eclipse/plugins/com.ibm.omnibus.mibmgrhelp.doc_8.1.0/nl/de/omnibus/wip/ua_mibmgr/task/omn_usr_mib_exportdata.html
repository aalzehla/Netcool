IBM Tivoli Netcool/OMNIbus Version 8.1

MIB-Daten exportieren

Vorgehensweise beim Exportieren von SNMP-MIB-Daten aus MIB Manager in verschiedene Formate.

Vorgehensweise

So exportieren Sie MIB-Daten:

  1. Klicken Sie auf Exportieren oder im Menü auf Datei > Exportieren.
  2. Geben Sie im Feld Verzeichnis das Zielverzeichnis für die exportierten Dateien an.
  3. Geben Sie im Feld Dateityp einen Dateityp für die exportierten Dateien an.

    Wenn möglich, müssen Sie Netcool Knowledge Library (NCKL) verwenden, weil die NCKL-Regeldateien auf der Funktionalität des Geräts basieren. Bei Verwendung von NCKL müssen Sie den Dateityp Netcool Knowledge Library Version 3.x angeben. Dadurch können die generierten Dateien den bestehenden Netcool Include Library-Installationen hinzugefügt werden.

    Die folgenden Optionen sind in der Dropdown-Liste Dateityp verfügbar:

    Dateityp Beschreibung
    CSV Wählen Sie diese Option aus, um anzugeben, dass alle Objektdaten in CSV-Dateien geladen werden, sodass sie mit einem Tabellenkalkulationsprogramm angezeigt werden können.
    CSV-Trapobjekte Wählen Sie diese Option aus, um anzugeben, dass alle Trapdaten und zugeordneten Objektdaten für Variablenbindungen in eine einzige CSV-Datei geladen werden, sodass sie mit einem Tabellenkalkulationsprogramm angezeigt werden können.
    HTML mit Frames Wählen Sie diese Option aus, um eine Webseite zu erstellen, die für die Veröffentlichung geeignet ist.

    Die ausgewählte MIB-Baumstruktursicht wird im linken Bereich der generierten Webseite angezeigt. Sie können durch die MIB-Baumstruktursicht navigieren, indem Sie Knoten innerhalb der MIB-Baumstruktur ein- und ausblenden. Wenn ein Element aus der MIB-Baumstruktur ausgewählt wird, werden die zugeordneten Informationen im rechten Bereich der Webseite angezeigt.

    Bei diesem Format werden häufig zahlreiche Exportdateien erstellt, die jeweils eine HTML-Datei für jedes in der MIB-Baumstruktur enthaltene Objekt aufweisen. Das Zielverzeichnis kann komprimiert und für einen beliebigen Web-Server bereitgestellt werden oder die Seiten können direkt von Platte angezeigt werden, indem die Datei index.html geöffnet wird, die im Exportverzeichnis erstellt wird.

    Wenn Sie einzelne HTML-Dateien mit einem Web-Browser verwenden wollen, müssen Sie den Export mit dem Typ HTML mit Frames ausführen und dann die Dateien im Unterverzeichnis data unter dem Exportverzeichnis verwenden.
    HTML ohne Frames Wählen Sie diese Option aus, um eine einzige Datei mit dem Namen oids.html zu erstellen. Diese Datei enthält eine Liste mit allen Objekten, die für die Ausgabe ausgewählt wurden. Diese Objekte sind jeweils durch horizontale Striche (Tag <HR>) voneinander getrennt. Jede OID ist im Tag "anchor" enthalten. Beim Laden wird also automatisch im Browser zu jeder OID geblättert, deren URL das Hashzeichen (#) enthält. Beispiel: file:c:/oids.html#1.3.6.1.4.1.9.

    Diese Datei kann sehr umfangreich werden, wenn Sie eine große Anzahl Objekte exportieren. Daher sollte dieses Format nur beim Export einer kleinen Anzahl Traps verwendet werden.
    Netcool-Referenztabelle Wählen Sie diese Option aus, damit alle angegebenen Werte in einer Liste mit durch Tabulator getrennten Werten ausgegeben werden, bei der die OIDs in der linken und die Objektnamen in der rechten Spalte enthalten sind. Dieses Format ist für das Einfügen in Regeldateien geeignet.
    Netcool Include Library Wählen Sie diese Option aus, um separate Include-Dateien für jedes Unternehmen zu erstellen und eine einzige Datei für die generischen Traps zu erstellen (das sind alle Traps, die nicht in der untergeordneten Baumstruktur des Unternehmens enthalten sind). Es wird angenommen, dass die Include-Dateien in bestehende snmp.rules-Dateien eingefügt werden, die entweder von MIB Manager oder von Netcool Include Library (NCIL) erstellt wurden.
    Netcool Knowledge Library Version 1.1 Wählen Sie diese Option aus, um Dateien in MIB Manager auszugeben. Die Dateien werden in eine bestehende Implementierung von Netcool Knowledge Library aufgenommen.
    Netcool Knowledge Library Version 3.x Wählen Sie diese Option aus, um Dateien in MIB Manager auszugeben. Die Dateien werden in eine bestehende Implementierung von Netcool Knowledge Library aufgenommen.
    Eigenständige Regeln Wählen Sie diese Option aus, um eine einzige Regeldatei zu erstellen, die alle ausgewählten Traps enthält. Sie können diese Datei in der Merkmaldatei eines beliebigen Tests eines Tivoli Netcool/OMNIbus-SNMP-Traps angeben.
    Einzelne Textdateien Das Format "Einzelne Textdateien" wird für das Popup-Menü der Netcool-Ereignisliste verwendet.

    Beim Export in einzelne Textdateien wird jede OID in eine separate Textdatei ausgegeben, z. B. 1.3.6.1.4.1.9.1.1.txt. Danach kann ein externes Tool zum Popup-Menü hinzugefügt werden, bei dem nco_message für die Anzeige aller Objektinformationen für ein Ereignis verwendet wird, dessen OID in einem Feld angegeben wird. Beispiel: cat 1.3.6.1.4.1.9.1.txt |nco_message stdin title "trap info".
    Tivoli Universal Agent Beim Format "Tivoli Universal Agent" werden .mdl- und trapcnfg-Dateien auf der Basis des ausgewählten Inhalts exportiert.
    Mehrere Unterbaumstrukturen können ausgewählt werden, wenn dieses Format für den Export von Dateien verwendet wird. Wenn mehrere Unterbaumstrukturen ausgewählt werden, werden für alle Objekte unterhalb eines ausgewählten Eintrags in der MIB-Baumstruktur .mdl- und trapcnfg-Dateien erstellt.
    Anmerkung: Die ersten drei Buchstaben des Anwendungsnamens nach dem Tag "//APPL" in der .mdl-Datei müssen innerhalb eines Unternehmens eindeutig sein. Für MIB Manager wird versucht, diese Anforderung zu erfüllen, andere ausgeführte Agenten können allerdings nicht überwacht werden. Möglicherweise müssen Sie daher die .mdl-Datei ändern, um sicherzustellen, dass der Anwendungsname im Unternehmen eindeutig ist.
  4. Verwenden Sie im Feld Bereich die Dropdown-Liste, um anzugeben, ob von MIB Manager lediglich Traps, lediglich Objekte oder alle OIDs ausgegeben werden.

    Welche Optionen verfügbar sind, hängt vom zu exportierenden Dateityp ab.

  5. Wählen Sie zum Einschränken der Anzahl der Objekte, die in die generierte Datei eingefügt werden, das Kontrollkästchen Nur ausgewählte untergeordnete Baumstrukturen aus.
  6. Klicken Sie auf Exportieren, um die Dateien in das Zielverzeichnis zu exportieren.
