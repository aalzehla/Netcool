IBM Tivoli Netcool/OMNIbus Versione 8.1

Esportazione di dati MIB

Come esportare i dati MIB SNMP MIB da MIB Manager in una varietà di formati.

Procedura

Per esportare dati MIB:

  1. Fare clic su Esporta oppure, dal menu, fare clic su File > Esporta.
  2. Nel campo Directory, specificare la directory di destinazione per i file esportati.
  3. Nel campo Tipo di file, specificare un tipo di file per i file esportati.

    È necessario utilizzare NCKL (Netcool Knowledge Library) laddove possibile, in quanto i file delle regole NCKL sono basati sulla funzionalità del dispositivo. Quando si utilizza NCKL, è necessario specificare il tipo di file Netcool Knowledge Library versione 3.x. In questo modo i file generati potranno essere aggiunti alle installazioni di Netcool Include Library esistenti.

    Le seguenti opzioni sono disponibili nell'elenco a discesa Tipo di file:

    Tipo di file Descrizione
    CSV Selezionare questa opzione per specificare che tutti i dati oggetto vengono caricati in file CSV, in modo da poter essere visualizzati utilizzando un'applicazione foglio di calcolo.
    Oggetti trap CSV Selezionare questa opzione per specificare che tutti i dati di trap e i dati oggetto binding delle variabili associati vengono caricati in un singolo file CSV, in modo da poter essere visualizzati utilizzando un'applicazione foglio di calcolo.
    HTML con frame Selezionare questa opzione per creare una pagina Web adatta alla pubblicazione.

    La vista Struttura ad albero MIB selezionata viene visualizzata a sinistra della pagina Web generata. È possibile spostarsi nella vista della struttura ad albero MIB espandendo e riducendo i nodi all'interno della struttura ad albero MIB. Quando un elemento viene selezionato nella struttura ad albero MIB, le informazioni associate vengono visualizzate a destra della pagina Web.

    Questo formato può spesso creare un numero elevato di file di esportazione, ciascuno contenente un file HTML per ciascun oggetto elencato nella struttura ad albero MIB. La directory di destinazione può essere compressa e situata al di sotto di un qualsiasi server Web oppure le pagine possono essere visualizzate direttamente da un disco aprendo il file index.html creato nella directory di esportazione.

    Se si desidera utilizzare file HTML singoli con un browser Web, esportare i file in HTML con frame, quindi utilizzare i file nella sottodirectory data situata al di sotto della directory di esportazione.
    HTML senza frame Selezionare questa opzione per creare un singolo file denominato oids.html. Questo file conterrà un elenco di tutti gli oggetti selezionati per l'output, ciascuno separato da righe orizzontali (utilizzando il tag <HR>). Ciascun OID presenta un tag di ancoraggio, in modo che, quando viene caricato, il browser scorre automaticamente verso il basso fino a qualsiasi OID che include un simbolo hash (#) nell'URL, ad esempio: file:c:/oids.html#1.3.6.1.4.1.9.

    Questo file può diventare molto grande se si esporta un numero elevato di oggetti. Questo formato, quindi, deve essere utilizzato solo per l'esportazione di un numero ridotto di trap.
    Tabella di ricerca Netcool Selezionare questa opzione per visualizzare tutti i valori specificati in un elenco a schede, con gli OID visualizzati nella colonna di sinistra e i nomi oggetto visualizzati nella colonna di destra. Questa opzione è adatta per l'inclusione in qualsiasi file delle regole.
    NCIL (Netcool Include Library) Selezionare questa opzione per creare file di inclusione separati per ciascuna azienda e per generare un singolo file per i trap generici, vale a dire, per i trap che non si trovano al di sotto della struttura ad albero secondaria dell'azienda. Si presuppone che i file di inclusione vengano inseriti in file snmp.rules esistenti, quelli creati in precedenza da MIB Manager o quelli creati da NCIL (Netcool Include Library).
    Netcool Knowledge Library versione 1.1 Selezionare questa opzione per emettere file da MIB Manager. I file saranno inclusi in un'implementazione Netcool Knowledge Library esistente.
    Netcool Knowledge Library versione 3.x Selezionare questa opzione per emettere file da MIB Manager. I file saranno inclusi in un'implementazione Netcool Knowledge Library esistente.
    Regole autonome Selezionare questa opzione per creare un singolo file delle regole contenente tutti i trap selezionati. È possibile specificare questo file nel file delle proprietà di qualsiasi probe di trap SNMP di Tivoli Netcool/OMNIbus.
    Singoli file di testo Il formato Singoli file di testo viene utilizzato per il menu Elenco eventi di Netcool visualizzato facendo clic con il pulsante destro del mouse.

    Quando si esporta in singoli file di testo, ciascun OID viene visualizzato in un file di testo separato, ad esempio 1.3.6.1.4.1.9.1.1.txt. Al menu di scelta rapida può essere quindi aggiunto uno strumento esterno che utilizza nco_message per visualizzare tutte le informazioni sull'oggetto relative ad un evento che presenta l'OID specificato in un campo, ad esempio : cat 1.3.6.1.4.1.9.1.txt |nco_message stdin title "trap info".
    Tivoli Universal Agent Il formato Tivoli Universal Agent esporta i file .mdl e trapcnfg in base al contenuto selezionato.
    Più strutture ad albero secondarie possono essere selezionate quando questo formato viene scelto per l'esportazione di file. Se vengono selezionate più strutture ad albero secondarie, i file .mdl e trapcnfg vengono creati per tutti gli oggetti situati al di sotto di un elemento selezionato nella struttura ad albero MIB.
    Nota: le prime tre lettere del nome applicazione, seguite dal tag //APPL nel file .mdl, devono essere univoche all'interno di un'azienda. MIB Manager tenta di uniformarsi a questo requisito ma non può essere al corrente di eventuali altri agent in esecuzione. Potrebbe essere necessario modificare il file .mdl per verificare che il nome applicazione sia univoco nell'azienda.
  4. Nel campo Ambito, utilizzare l'elenco a discesa per specificare se MIB Manager esegue l'output solo dei trap, solo degli oggetti i di tutti gli OID.

    Le opzioni disponibili dipendono dal tipo di file che si desidera esportare.

  5. Per limitare il numero di oggetti inclusi nel file generato, selezionare la casella di spunta Solo strutture ad albero secondarie selezionate.
  6. Fare clic su Esporta per esportare i file nella directory di destinazione.
