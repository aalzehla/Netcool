IBM Tivoli Netcool/OMNIbus Version 8.1

Sicht "MIB-Module"

Die Sicht MIB-Module bietet eine Darstellung aller importierten MIB-Module. In dieser Sicht können Sie MIB-Module suchen und löschen sowie bestimmte MIB-Werte suchen.

Sie können durch die Sicht MIB-Baumstruktur navigieren, indem Sie Knoten innerhalb der MIB-Baumstruktur ein- und ausblenden. Die Verzweigungen der höchsten Stufe in der MIB-Baumstruktur enthalten die Namen der MIB-Module. In den einzelnen Verzweigungen für MIB-Module sind die anderen Elemente angeordnet, aus denen sich die jeweilige MIB zusammensetzt.

Sie können die Daten in der Sicht MIB-Module filtern, indem Sie aus der Dropdown-Liste Sicht eine der folgenden Optionen auswählen:
  • Alles
  • Module
  • Aufzählungen
  • Textkonventionen
  • Syntaxen
  • Importe
  • Exporte
  • Spalten
  • Beschreibungen

Um in der Sicht MIB-Module einen bestimmten Wert zu suchen, müssen Sie den Text in das Feld Suchen eingeben und entweder die Eingabetaste drücken oder auf ("Suchen") klicken.

MIB Manager beginnt beim ausgewählten Element oder, falls momentan kein Element ausgewählt ist, beim ersten Element in der Liste mit der Suche. Klicken Sie zum Suchen des nächsten übereinstimmenden Elements in der Liste nochmals auf ("Suchen").

Wenn ein Element gefunden wird, wird es automatisch hervorgehoben und in der Sicht MIB-Module angezeigt.

Geben Sie im Feld Suchen einen gültigen regulären Ausdruck an oder geben Sie einfachen Text ein, der einen vollständigen Objektnamen oder einen Teil davon darstellt.

Um ein oder mehrere MIB-Module zu löschen, klicken Sie mit der rechten Maustaste auf die ausgewählten MIB-Module und wählen Sie danach im Popup-Menü den Eintrag Löschen aus. Die ausgewählten MIB-Module werden aus MIB Manager gelöscht. Die ursprünglichen MIB-Quellendateien, die für den MIB-Import bereitgestellt wurden, werden allerdings nicht gelöscht.

Anmerkung: Sie dürfen ein MIB-Modul nur dann löschen, wenn es Fehler enthält und eine neue Ersatzversion davon vorhanden ist. Das Löschen von MIB-Modulen über diese Methode kann zu fehlenden MIB-Abhängigkeiten führen. Außerdem können während des Exportprozesses unvollständige Regeldateien generiert und Warnungen oder Fehlernachrichten in den Protokolldateien gespeichert werden.
