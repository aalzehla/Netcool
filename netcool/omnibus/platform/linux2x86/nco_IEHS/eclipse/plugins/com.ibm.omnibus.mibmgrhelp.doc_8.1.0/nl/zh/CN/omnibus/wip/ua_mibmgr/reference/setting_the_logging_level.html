设置日志记录级别

您可以更改日志记录级别以提供更详细的日志或跟踪文件，从而帮助对 MIB 管理器 进行调试。提高日志记录级别所带来的额外信息在您对解析 MIB 是发生的特定文件进行调试时很有用。

要更改日志记录级别，请在主窗口菜单栏中单击编辑 > 调试。日志文件有六种消息级别可供您选择以更改所捕获信息的详细程度：无、错误、警告、参考、调试和详细。

下表简要描述了每种日志记录级别：

表 1. 日志记录级别及其关联描述的列表
日志级别 描述
无

指定不输出任何调试消息的日志级别。

应用程序的性能将提高，但是解析期间不会显示任何警告或错误消息。
error

指定仅对影响结果的错误进行记录的日志级别。

这包括 MIB 管理器 确定的任何会导致主窗口中所显示的数据不正确的错误。
warn

指定对警告和错误消息进行记录的日志级别。

警告表示可能导致数据错误而必须检查的状况。但是，该状况不会导致数据损坏。
info

指定对参考、警告和临界消息进行记录的日志级别。

参考消息包含与总体任务进度相关的消息。例如，在解析对象时，它们将提供对象的完整文本。
调试
指定对所有消息进行记录的日志级别。
注： 请仅在您尝试解决错误状况时才以调试方式运行 MIB 管理器。仅解析中等数量的 MIB 所产生的数据量就会迅速填满文件系统。
详细

指定对所有参考消息进行记录的日志级别。

详细消息包括变量、数组和散列更改值和状态时的状态。
