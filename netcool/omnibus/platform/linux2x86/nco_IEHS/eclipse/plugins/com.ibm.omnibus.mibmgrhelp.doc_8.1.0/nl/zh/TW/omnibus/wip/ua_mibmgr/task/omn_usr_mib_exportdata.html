IBM Tivoli Netcool/OMNIbus 8.1 版

匯出 MIB 資料

如何將 MIB Manager 中的 SNMP MIB 資料匯出為各種格式。

程序

若要匯出 MIB 資料，請執行下列動作：

  1. 按一下 匯出，或從功能表中按一下檔案 > 匯出。
  2. 在目錄欄位中，指定匯出檔的目的地目錄。
  3. 在檔案類型欄位中，指定匯出檔的檔案類型。

    無論如何，您都必須使用 Netcool Knowledge Library (NCKL)，因為 NCKL 規則檔案是基於裝置的功能。使用 NCKL 時，必須指定 Netcool Knowledge Library 3.x 版檔案類型。如此即可讓產生的檔案新增至現有的 Netcool Include Library 安裝中。

    檔案類型下拉清單中提供了下列選項：

    檔案類型 說明
    CSV 選取此選項來指定所有物件資料都載入 CSV 檔，以便可以使用試算表應用程式來檢視。
    CSV 設陷物件 選取此選項來指定所有設陷資料及關聯的變數連結物件資料都載入單一 CSV 檔案，以便可以使用試算表應用程式來檢視。
    具有訊框的 HTML 選取此選項以建立適合發佈的網頁。

    選取的 MIB 樹狀結構視圖會顯示在產生的網頁左側。您可以透過展開及收合 MIB 樹狀結構中的節點，來導覽「MIB 樹狀結構」視圖。從 MIB 樹狀結構選取元素時，其關聯資訊會顯示在網頁的右側。

    此格式通常可建立大量匯出檔，針對 MIB 樹狀結構中列出的每個物件，每個匯出檔案都包含 HTML 檔案。目標目錄可進行壓縮並放置在任何 Web 伺服器下，或者可以開啟匯出目錄中建立的 index.html 檔來直接從磁碟檢視頁面。

    若要利用 Web 瀏覽器來使用個別的 HTML 檔案，請匯出至具有訊框的 HTML，然後使用位於匯出目錄下 data 子目錄中的檔案。
    沒有訊框的 HTML 選取此選項來建立單一檔案 oids.html。 此檔案將包含針對輸出選取的所有物件清單，每個物件以水平尺規（使用 <HR> 標籤）來區隔。每個 OID 由錨點標籤括住，以便在載入 OID 時，瀏覽器會自動向下捲動到在 URL 中包括雜湊符號 (#) 的任何 OID，例如：file:c:/oids.html#1.3.6.1.4.1.9。

    如果匯出大量物件，則這個檔案會變得非常大。因此，這個格式只應用於匯出少量設陷。
    Netcool Lookup Table 選取此選項來輸出所有指定的值（以加上標籤的清單形式），其中 OID 顯示在左欄，物件名稱顯示在右欄。這適合併入任何規則檔。
    Netcool Include Library 選取此選項以為每個企業建立個別併入檔，以及為一般設陷（亦即，不是位於企業子樹狀結構之下的任何設陷）產生單一檔案。 假設併入檔已插入現有的 snmp.rules 檔案（之前由 MIB Manager 建立的檔案或由 Netcool Include Library (NCIL) 建立的檔案）中。
    Netcool Knowledge Library 1.1 版 選取此選項以從 MIB Manager 輸出檔案。這些檔案將併入現有的 Netcool Knowledge Library 實作中。
    Netcool Knowledge Library 3.x 版 選取此選項以從 MIB Manager 輸出檔案。這些檔案將併入現有的 Netcool Knowledge Library 實作中。
    獨立式規則 選取此選項以建立一個包含所有所選設陷的單一規則檔。您可以在任何 Tivoli Netcool/OMNIbus SNMP 設陷探測器的內容檔中指定這個檔案。
    個別文字檔 「個別文字檔」格式用於 Netcool Event List 的右鍵功能表。

    匯出至個別文字檔時，每個 OID 會匯出到個別文字檔，例如 1.3.6.1.4.1.9.1.1.txt。 外部工具隨後可新增至右鍵功能表，該右鍵功能表使用 nco_message 來顯示在欄位中指定了 OID 之事件的所有物件資訊；例如 cat 1.3.6.1.4.1.9.1.txt |nco_message stdin title "trap info"。
    Tivoli Universal Agent Tivoli Universal Agent 格式根據選取的內容匯出 .mdl 和 trapcnfg 檔。
    選取此格式來匯出檔案時，可以選取多個子樹狀結構。如果選取多個子樹狀結構，則會針對位於 MIB 樹狀結構中所選取項目下的所有物件，建立 .mdl 和 trapcnfg 檔。
    註： 在 .mdl 檔中，應用程式名稱的前三個字母，後跟 //APPL 標籤，在企業中必須唯一。MIB Manager 嘗試遵循此需求，但無法知道可能在執行中的任何其他代理程式。您可能需要修改 .mdl 檔以確保應用程式名稱在企業中是唯一的。
  4. 在範圍欄位中，使用下拉清單來指定 MIB Manager 是只輸出設陷、只輸出物件還是輸出所有 OID。

    選項是否可用視正在匯出的檔案類型而定。

  5. 若要限制產生的檔案中併入的物件數，請選取僅限選取的子樹狀結構勾選框。
  6. 按一下匯出以將檔案匯出至目的地目錄。
