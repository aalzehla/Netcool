IBM Tivoli Netcool/OMNIbus, Версия 8.1

Экспорт данных MIB

Как экспортировать данные SNMP MIB из MIB Manager в различных форматах.

Процедура

Чтобы экспортировать данные MIB:

  1. Щелкните по значку Экспорт или в меню выберите пункт Файл > Экспорт.
  2. В поле Каталог задайте каталог назначения для экспортированных файлов.
  3. В поле Тип файла задайте тип файла для экспортированных файлов.

    Везде, где возможно, необходимо использовать Netcool Knowledge Library (NCKL), так как файлы правил NCKL основаны на функциональных возможностях устройства. При использовании NCKL необходимо задать тип файла Netcool Knowledge Library версии 3.x. Это позволит добавить сгенерированные файлы к существующим установкам Netcool Include Library.

    В выпадающем списке Тип файла доступны следующие опции:

    Тип файла Описание
    CSV Выберите эту опцию, чтобы указать, что все данные объектов загружаются в файлы CSV; их можно будет увидеть в прикладной программе электронной таблицы.
    Объекты прерывания CSV Выберите эту опцию для указания, чтобы все данные прерываний и данные объектов связывания связанных переменных загружались в один файл CSV и их можно было увидеть в прикладной программе электронной таблицы.
    HTML с фреймами Выберите эту опцию для создания подходящей для публикации Web-страницы.

    Представление выбранного дерева MIB будет показано в левой части сгенерированной Web-страницы. Вы можете перемещаться по представлению дерева MIB, раскрывая и сворачивая узлы на этом дереве. Когда в дереве MIB выбран некоторый элемент, связанная с ним информация будет показана в правой части Web-страницы.

    Использование этого формата часто приводит к созданию большого количества файлов экспорта, содержащих файлы HTML для каждого перечисленного в дереве MIB объекта. Каталог назначения можно сжать с помощью архиватора zip и разместить на любом Web-сервере или просматривать страницы непосредственно с диска, открыв файл index.html, созданный в каталоге экспорта.

    Если необходимо использовать индивидуальные файлы HTML в Web-браузере, проведите экспорт в HTML с фреймами, а затем используйте файлы в подкаталоге data, расположенном в каталоге экспорта.
    HTML без фреймов Выберите эту опцию для создания одного файла с именем oids.html. Этот файл будет содержать список всех объектов, выбранных для вывода и разделенных горизонтальными линиями (с использованием тега <HR>). Каждый OID заключен в тег привязки, так что при его загрузке браузер автоматически прокручивает страницу до любого OID, включающего хеш-символ (#) в URL, например: file:c:/oids.html#1.3.6.1.4.1.9.

    Этот файл может сильно вырасти, если вы экспортируете много объектов. Поэтому такой формат следует использовать только при экспорте небольшого количества прерываний.
    Таблица поиска Netcool Выберите эту опцию, чтобы выводить все заданные значения в виде табличного списка, в котором OID перечисляются в левом столбце, а имена объектов - в правом. Такая опция подходит для использования в любом файле правил.
    Netcool Include Library Выберите эту опцию для создания отдельных файлов включения для каждого предприятия и для генерирования одного файла для общих прерываний (то есть таких, которые расположены вне поддерева предприятия). Предполагается, что файлы включения вставлены в существующие файлы snmp.rules, созданные ранее с помощью MIB Manager или Netcool Include Library (NCIL).
    Netcool Knowledge Library версии 1.1 Выберите эту опцию, чтобы выводить файлы из MIB Manager. Эти файлы буду включены в существующую реализацию Netcool Knowledge Library.
    Netcool Knowledge Library версии 3.x Выберите эту опцию, чтобы выводить файлы из MIB Manager. Эти файлы буду включены в существующую реализацию Netcool Knowledge Library.
    Автономные правила Выберите эту опцию для создания одного файла правил, содержащего все выбранные прерывания. Этот файл можно задать в файле свойств любого зонда прерываний SNMP Tivoli Netcool/OMNIbus.
    Индивидуальные текстовые файлы Формат индивидуальных текстовых файлов используется для меню списка событий Netcool, появляющегося по щелчку правой кнопки мыши.

    При экспорте в индивидуальные текстовые файлы каждый OID - это вывод в отдельный текстовый файл, например, 1.3.6.1.4.1.9.1.1.txt. К меню правой кнопки мыши можно добавить внешний инструмент, использующий nco_message для вывода всей информации об объекте для события с заданным в поле OID, например, cat 1.3.6.1.4.1.9.1.txt |nco_message stdin title "trap info".
    Универсальный агент Tivoli Универсальный агент Tivoli экспортирует файлы .mdl и trapcnfg на основе выбранного контента.
    Когда этот формат выбирается для экспорта файлов, можно выбрать несколько поддеревьев. Если несколько таких поддеревьев выбраны, файлы .mdl и trapcnfg создаются для всех объектов, расположенных ниже выбранного в дереве MIB элемента.
    Прим.: Первые три буквы в имени прикладной программы, следующие за тегом //APPL в файле .mdl, должны быть уникальными для предприятия. MIB Manager пытается выполнить это требование, но ему могут не следовать другие запущенные агенты. Вам может потребоваться изменить файл .mdl, чтобы обеспечить уникальность имени прикладной программы для предприятия.
  4. В поле Область используйте выпадающий список для указания содержимого вывода MIB Manager - только прерывания, только объекты или все OID.

    Доступные опции зависят от типа экспортируемых файлов.

  5. Для ограничения количества объектов, входящих в генерируемые файлы, включите переключатель Только выбранные поддеревья.
  6. Нажмите кнопку Экспорт, чтобы экспортировать файлы в каталог назначения.
