IBM Tivoli Netcool/OMNIbus 버전 8.1

Netcool MIB Manager 사용

MIB Manager를 사용하여 MIB 모듈을 구문 분석하고 Netcool 규칙 파일을 작성할 수 있습니다.

MIB Manager에는 기본 MIB와 RFC MIB 세트가 포함되며 이는 NCHOME/omnibus/var/mibmanager/mibs 디렉토리에 있습니다. 이러한 MIB는 추가 MIB를 가져오기 위해 제공됩니다. MIB Manager는 이 번들 MIB를 자동으로 사용하여 새 MIB를 가져오는 동안 종속성을 분석합니다. 번들 MIB 모듈은 MIB Manager를 처음 시작할 때 MIB 모듈 보기에 표시됩니다.

또한 MIB Manager에는 번들 기본 MIB 가져오기를 나타내는 XML 데이터 파일이 포함됩니다. MIB Manager는 이러한 XML 파일을 사용하여 원시 텍스트 MIB 파일에서 가져오는 MIB 데이터를 저장합니다.

MIB Manager가 시작되는 서버에서 액세스할 수 있는 디렉토리에 MIB를 추가하십시오. 모든 표준 기반 MIB는 쉽게 종속성을 분석할 수 있도록 검색 경로에 배치할 수 있는 별도의 디렉토리에 설치되어야 합니다. 기본적으로 기본 SNMP MIB는 NCHOME/omnibus/var/mibmanager/mibs/base 디렉토리에 설치되고 모든 RFC MIB는 NCHOME/omnibus/var/mibmanager/mibs/rfc 디렉토리에 설치됩니다.

Netcool MIB Manager 창

MIB Manager를 시작하면 초기 MIB Manager 창에 다음 요소가 표시됩니다. 초기 MIB Manager 창에 여러 요소가 표시됩니다. 다음 그림에 해당 요소가 표시되어 있습니다.

그림 1. MIB Manager 초기 창
 1  도구 모음
가장 자주 사용하는 MIB Manager 기능에 액세스합니다.
 2  퍼스펙티브
MIB Manager 또는 mib2rules 퍼스펙티브 중 하나에서 MIB 데이터를 볼 수 있습니다. MIB Manager 퍼스펙티브는 기본 퍼스펙티브입니다. mib2rules 퍼스펙티브는 OID(Object Identifier)에 중점을 두며 mib2rules 유틸리티에 친숙한 사용자에게 제공됩니다.
 3  디바이스 보기
이 보기를 사용하여 MIB Manager 디바이스를 작성하고 업데이트하며 삭제할 수 있습니다.
 4  세부사항 보기와 콘솔 보기
세부사항 보기에는 선택한 MIB 요소의 세부사항이 표시됩니다. 콘솔 보기에는 현재 인스턴스가 실행되는 동안 MIB Manager에서 생성되는 모든 로그 메시지가 표시됩니다.
 5  MIB 모듈
MIB 모듈을 검색하고 삭제하며 특정 MIB 값을 검색하려면 이 보기를 사용합니다.
 6  OID 트리 보기
이 보기는 가져온 MIB 모듈에 있는 모든 MIB 오브젝트를 표시합니다.
  • MIB 모듈 보기
    MIB 모듈 보기에서는 가져온 모든 MIB 모듈의 표시를 제공하므로, MIB 모듈을 검색하고, MIB 모듈을 삭제하고, 특정 MIB 값을 검색할 수 있습니다.
  • OID 트리 보기
    OID(Object Identifier) 트리 보기는 가져온 MIB 모듈에 있는 모든 MIB 오브젝트의 표시를 제공합니다.
  • MIB 데이터 가져오기
    SNMP MIB 데이터를 MIB Manager로 가져오는 방법입니다.
  • MIB 데이터 내보내기
    MIB Manager의 SNMP MIB 데이터를 다양한 형식으로 내보내는 방법을 설명합니다.
  • SNMP 트랩 편집
    SNMP 트랩을 편집하는 방법입니다.
  • SNMP 트랩 생성
    SNMP 트랩을 생성하는 방법입니다. MIB Manager에서 생성된 규칙 파일을 테스트하는 데에도 이 기능을 사용할 수 있습니다.
  • 디바이스 작성과 삭제
    MIB Manager 디바이스는 연관된 MIB 모듈을 그룹화하고 내보내기에 필요한 모든 오브젝트를 정의하는 데 사용됩니다. MIB Manager GUI에서 디바이스를 작성하고 업데이트하며 삭제할 수 있습니다.
