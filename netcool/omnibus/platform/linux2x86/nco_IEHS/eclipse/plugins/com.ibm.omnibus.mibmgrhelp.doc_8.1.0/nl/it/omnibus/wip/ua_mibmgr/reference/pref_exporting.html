Impostazione di preferenze di esportazione

Questa sezione descrive come impostare le preferenze di esportazione dei file delle regole.

Per impostare le preferenze di directory, fare clic su Preferenze oppure, dal menu, fare clic su File > Preferenze, quindi dal pannello di sinistra della finestra Preferenze, fare clic su Esporta.

  • Nel campo Percorso di inclusione, specificare l'ubicazione delle regole installate con i pacchetti dei file delle regole Netcool Include Library (NCiL) (se installati). Questa opzione è richiesta solo se si desidera esportare i dati nel formato file di Netcool Include Library e quando il percorso di directory deve puntare a regole incluse. Di conseguenza, non è necessario che i nuovi file delle regole vengano modificati manualmente. Il valore predefinito è NCHOME/omnibus/probes/arch.
  • Nel campo Formato file di esportazione predefinito, specificare un formato Windows/DOS (CRLF) o un formato Unix (LF) per il file di output. Questa proprietà consente di creare regole su una macchina Windows e di copiarle su una macchina UNIX senza eseguire un programma di utilità come dos2unix.
    Nota: questa selezione è valida per tutti i file di output e non è limitata ai file delle regole. Il valore predefinito è in formato Unix.
  • Nel campo Tipi di avviso, specificare le opzioni dell'elenco a discesa visualizzato quando il campo @Type di un trap viene modificato nella vista Dettagli.

Le seguenti opzioni sono disponibili solo se è stato precedentemente selezionato il formato di esportazione NCKL:

  • Se è selezionata la casella di spunta Includi descrizioni varbind, le descrizioni varbind vengono aggiunte come una documentazione di commento. Il valore predefinito è abilitato.
  • Se è selezionata la casella di spunta Calcola un valore ottimale per @AlertKey, il valore @AlertKey viene generato dall'oggetto parent varbind. Il valore predefinito è abilitato.
  • Se è selezionata la casella di spunta Imposta @Agent una volta per ciascuna azienda, il campo @Agent viene impostato per ciascuna azienda. Il campo @Agent si basa sui nomi dei moduli MIB da elaborare per i file delle regole. Il valore predefinito è abilitato.
  • Se è selezionata la casella di spunta Aggiungi modello @Class, viene impostato il campo @Class. Se è selezionata la casella di spunta Aggiungi modello @Class, il campo @Class è impostato su 300. Il valore predefinito è abilitato.
