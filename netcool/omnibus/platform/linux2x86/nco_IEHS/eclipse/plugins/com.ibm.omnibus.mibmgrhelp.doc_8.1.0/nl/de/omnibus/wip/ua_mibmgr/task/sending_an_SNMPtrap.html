IBM Tivoli Netcool/OMNIbus Version 8.1

SNMP-Traps generieren

Vorgehensweise beim Generieren von SNMP-Traps. Diese Funktionalität kann auch zum Testen von Regeldateien verwendet werden, die von MIB Manager generiert wurden.

Vorgehensweise

So generieren Sie einen SNMP-Trap:

  1. Klicken Sie auf Trap.
  2. Geben Sie im Feld Zieladresse den Hostnamen oder die IP-Adresse eines Computers mit einem SNMP-Trapd-Service an.
  3. Geben Sie im Feld Zielport den UDP-Port an, an dem der SNMP-Trapd-Service empfangsbereit ist.
  4. Geben Sie im Feld Community eine gültige Zeichenfolge für den Trapd-Manager an.
  5. Je nach dem zu generierenden Traptyp wird im Fenster SNMP-Trap generieren unter Umständen der Abschnitt Variablenbindungen angezeigt. Die verfügbaren Felder im Abschnitt Variablenbindungen hängen ebenfalls vom generierten Traptyp ab.

    Bewegen Sie die Maus über Feldnamen im Abschnitt Variablenbindungen, um Details zur Feldwertsyntax anzuzeigen. Mehrere Werte können durch Kommas voneinander getrennt angegeben werden. Diese Werte werden wahlfrei für die Erstellung des Traps ausgewählt.

  6. Geben Sie im Feld Wiederholungsmesswert die Anzahl Traps (1 bis 1000) an, die generiert werden können.

    Wenn Sie eine große Anzahl angeben, werden Traps in rascher Folge generiert, was im Trapd-Manager zu einem Alarmsturm führen kann. Bei Werten größer als eins (> 1) und Werten, die in Form von durch Semikolons unterteilten Auflistungen angegeben werden, wird von MIB Manager wahlfrei eine Auswahl aus den angegebenen Auflistungen getroffen und Traps werden mit den beliebigen Werten als Variablenbindungen generiert.

  7. Klicken Sie auf Ausführen, um den Trap zu generieren.

Ergebnisse

Das Feld Ergebnis im Fenster SNMP-Trap generieren gibt an, ob der Trap erfolgreich gesendet wurde.
