IBM Tivoli Netcool/OMNIbus Version 8.1

Protokollierungsstufe festlegen

Sie können die Protokollierungsstufe ändern, um ein detaillierteres Protokoll oder Tracedateien als Unterstützung beim Debugging von MIB Manager bereitzustellen. Die zusätzlichen Informationen, die durch Erhöhen der Protokollierungsstufen bereitgestellt werden, können das Debugging bestimmter Probleme während des Parsing von MIBs erleichtern.

Klicken Sie zum Ändern der Protokollierungsstufe in der Menüleiste des Hauptmenüs auf Bearbeiten > Debug. Protokolldateien weisen sechs Nachrichtenstufen auf, die Sie durchlaufen können, um den erfassten Detaillierungsgrad zu erhöhen: "Keine", "Fehler", "Warnung", "Info", "Debug" und "Ausführlich".

Die folgende Tabelle enthält Kurzbeschreibungen der einzelnen Protokollierungsstufen:

Tabelle 1. Liste der Protokollierungsstufen mit zugehörigen Beschreibungen
Protokollierungsstufe Beschreibung
Keine

Gibt eine Protokollierungsstufe an, bei der keine Debugnachrichten ausgegeben werden.

Die Leistung der Anwendung wird verbessert, beim Parsing werden aber keine Warnungen oder Fehlernachrichten angezeigt.
Fehler

Gibt eine Protokollierungsstufe an, bei der nur Fehler ausgegeben werden, die sich auf die Ergebnisse auswirken.

Dazu gehören alle Fehler, die von MIB Manager festgestellt werden und die zur Anzeige falscher Daten im Hauptfenster führen.

Warnung

Gibt eine Protokollierungsstufe an, bei der Warnungen und Fehlernachrichten protokolliert werden.

Eine Warnung gibt eine Bedingung an, die zu fehlerhaften Daten führen kann und daher geprüft werden muss. Die Bedingung führt aber nicht zur Beschädigung von Daten.
Info

Gibt eine Protokollierungsstufe an, bei der Informationsnachrichten, Warnungen und kritische Nachrichten protokolliert werden.

Informationsnachrichten enthalten Details bezüglich des Gesamtfortschritts einer Task. Sie bieten beispielsweise den vollständigen Text für Objekte während des Parsing.
Debug
Gibt eine Protokollierungsstufe an, bei der alle Nachrichten protokolliert werden.
Anmerkung: Führen Sie MIB Manager nur dann im Debugmodus aus, wenn Sie eine Fehlerbedingung beheben wollen. Auch wenn das Parsing nur für wenige MIBs ausgeführt wird, wird durch das produzierte Datenvolumen rasch Ihr Dateisystem mit Daten gefüllt.
Ausführlich

Gibt eine Protokollierungsstufe an, bei der alle Informationsnachrichten protokolliert werden.

Eine Nachricht mit ausführlichen Informationen enthält den Status von Variablen, Feldgruppen und Hashes bei Änderungen an deren Werten und Statuszuständen.
