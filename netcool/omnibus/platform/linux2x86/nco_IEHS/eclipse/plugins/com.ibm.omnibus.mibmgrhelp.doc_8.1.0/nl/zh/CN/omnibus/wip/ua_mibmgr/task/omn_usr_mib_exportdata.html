IBM Tivoli Netcool/OMNIbus V8.1

导出 MIB 数据

如何将 SNMP MIB 数据从 MIB 管理器 导出为各种格式。

过程

要导出 MIB 数据：

  1. 单击 导出，或在菜单中单击文件 > 导出。
  2. 在目录字段中，指定导出文件的目标目录。
  3. 在文件类型字段中，指定导出文件的文件类型。

    您必须尽可能地使用 Netcool Knowledge Library (NCKL)，因为 NCKL 规则文件基于设备的功能。使用 NCKL 时，必须指定 Netcool Knowledge Library V3.x 文件类型。这将使生成的文件能够添加到已安装的 Netcool Include Library 中。

    文件列表下拉列表中提供了以下选项：

    文件类型 Description
    CSV 选择此选项可指定将所有对象数据装入多个 CSV 文件中，以便使用电子表格应用程序进行查看。
    CSV 陷阱对象 选择此选项可指定将所有陷阱数据以及相关联的变量绑定对象装入单个 CSV 文件中，以便使用电子表格应用程序进行查看。
    具有框架的 HTML 选择此选项可创建适合于发布的 Web 页面。

    选定的 MIB 树形视图将显示在生成的 Web 页面的左侧。您可以通过展开和折叠 MIB 树内的节点来浏览 MIB 树视图。从 MIB 树中选择元素后，与该元素关联的信息将显示在 Web 页面的右侧。

    此格式通常会创建大量导出文件，每个文件中都包含一个 HTML 文件，该 HTML 对应于 MIB 树中的每个对象。可以对目标目录进行压缩并放置在任意 Web 服务器下，也可以通过打开导出目录中创建的 index.html 文件直接从磁盘查看页面。

    如果您要将个别 HTML 文件用于 Web 浏览器，请导出为具有框架的 HTML，然后使用位于导出目录下的 data 子目录中的文件。
    没有框架的 HTML 选择此选项可创建名为 oids.html 的单个文件。此文件将包含选择输出的所有对象的列表，各个对象间以水平线（使用 <HR> 标记）分隔。每个 OID 都以锚点标记括起，这样，在装入该 OID 时，浏览器就会自动下滚至 URL 中包含井字符号 (#) 的任何 OID，例如：file:c:/oids.html#1.3.6.1.4.1.9。

    如果您导出大量对象，那么此文件可能非常大。因此，此格式应该用于导出少量陷阱。
    Netcool 查找表 选择此选项可在表式列表中输出所有指定值，其中，左侧列中显示 OID，右侧列中显示对象名称。这适用于包含任何规则文件。
    Netcool 包含库 选择此选项可为每个企业创建单独的包含文件，并为一般陷阱（即不在企业子树下的所有陷阱）生成单个文件。 假设包含文件将插入现有的 snmp.rules 文件中，无论是 MIB 管理器 先前创建的文件还是 Netcool 包含库 (NCIL) 创建的文件。
    Netcool Knowledge Library V1.1 选择此选项以从 MIB 管理器 输出文件。这些文件将包含在已实施的 Netcool Knowledge Library 中。
    Netcool Knowledge Library V3.x 选择此选项以从 MIB 管理器 输出文件。这些文件将包含在已实施的 Netcool Knowledge Library 中。
    独立规则 选择此选项可创建单个规则文件，其中包含选定的所有陷阱。可以在任何 Tivoli Netcool/OMNIbus SNMP 陷阱探测器的属性文件中指定此文件。
    个别文本文件 “个别文本文件”格式用于 Netcool 事件列表右键单击菜单。

    导出为个别文本文件时，将把每个 OID 输出到一个单独的文本文件，例如 1.3.6.1.4.1.9.1.1.txt。然后可以将外部工具添加到右键单击菜单中，该菜单使用 nco_message 在字段中显示已指定了 OID 的事件的所有对象信息，例如 cat 1.3.6.1.4.1.9.1.txt |nco_message stdin title "trap info"。
    Tivoli Universal Agent Tivoli Universal Agent 格式根据所选的内容导出 .mdl 和 trapcnfg 文件。
    选择此格式用于导出文件时，可以选择多个子树。如果选择了多个子树，那么将为位于 MIB 树中选定项下的所有对象创建 .mdl 和 trapcnfg 文件。
    注： 应用程序程序的前三个字母（跟在 .mdl 文件中的 //APPL 标记之后）在企业内必须唯一。MIB 管理器 会尝试遵守此要求，但它无法了解可能正在运行的其他任何代理程序。您可能需要修改 .mdl 文件以确保应用程序名称在企业内唯一。
  4. 在范围字段中，使用下拉列表指定 MIB 管理器 是仅输出陷阱、仅输出对象，还是输出所有 OID。

    可用的选项取决于您所导出的文件类型。

  5. 要限制生成的文件中包含的对象数，请选中仅选定的子树复选框。
  6. 单击导出将文件导出至目标目录。
