Опции командной строки MIB Manager

В MIB Manager есть утилита командной строки, при помощи которой можно импортировать данные MIB и файлы правил экспорта. Вы можете вводить команды вручную или можете вызывать их программным образом.

Чтобы запустить утилиту командной строки, введите следующую команду:
  • $NCHOME/omnibus/bin/nco_mibmanager_batch [опции]
  • %NCHOME%\omnibus\bin\nco_mibmanager_batch [опции]

    В операционных системах Windows выходная информация из MIB Manager появляется на новой консоли.

В следующей таблице описаны все доступные опции командной строки. Опции -exportmibs, -exportobjects и -exportoids позволяют анализировать регулярные выражения, соответствующие механизму шаблонов Java.

Указывая каталоги, используйте полные пути каталогов. Например: nco_mibmanager_batch -importdir "C:\anydir\mibs"

Табл. 1. Опции командной строки MIB Manager
Опция командной строки Описание
-clean Используйте эту опцию при первом запуске утилиты командной строки MIB Manager, если вы ранее использовали графическую утилиту MIB Manager. Если вы не укажете эту опцию, появится ошибка.
-exportdir строка Используйте эту опцию, чтобы задать каталог, в который экспортируются сгенерированные файлы правил. Заключите путь каталога в двойные кавычки (").

Примеры:

nco_mibmanager_batch -exportdir "/home/user/export_cmd" -filetype csv
-exportmibs строка Используйте эту опцию, чтобы задать имя экспортируемого объекта MIB. Можно использовать регулярные выражения, чтобы задать несколько MIB.

Например, показанная ниже команда экспортирует объекты из MIB "RMON-MIB":

nco_mibmanager_batch -exportmibs RMON-MIB -filetype csv

Показанная ниже команда экспортирует объекты из всех MIB, имена которых содержат строку "MON":

nco_mibmanager_batch -exportmibs .*MON.* -filetype csv
-exportnewdirectory Используйте эту опцию, чтобы указать, что экспортированные файлы записываются в новый каталог в существующем каталоге экспорта. Новому подкаталогу автоматически присваивается имя с временной меткой. Используйте эту опцию, чтобы не допустить перезаписи существующих файлов правил.

Например:

nco_mibmanager_batch -exportdir "/home/user/export_cmd" -exportnewdirectory -filetype csv
-exportobjects строка Используйте эту опцию, чтобы задать имя объекта MIB, который нужно экспортировать. Можно использовать регулярные выражения, чтобы задать несколько объектов.

Например, показанная ниже команда экспортирует все объекты с именами "coldStart":

nco_mibmanager_batch -exportobjects coldStart -filetype csv

Показанная ниже команда экспортирует все объекты, имена которых начинаются с "cold":

nco_mibmanager_batch -exportobjects cold.* -filetype csv
-exportoids строка Используйте эту опцию, чтобы задать OID, который нужно экспортировать. Можно использовать регулярные выражения, чтобы задать несколько OID.

Например, показанная ниже команда экспортирует все объекты с OID "1.3.6.1.6.3.1.1.5.1":

nco_mibmanager_batch -exportoids 1.3.6.1.6.3.1.1.5.1 -filetype csv

Следующая команда экспортирует все объекты, у которых OID начинается с "1.3.6.1.6.3.1.1.5":

nco_mibmanager_batch -exportoids 1.3.6.1.6.3.1.1.5.* -filetype csv

Следующая команда экспортирует все объекты с OID "1.3.6.1.6.3.1.1.5.1" или "1.3.6.1.6.3.1.1.5.2":

nco_mibmanager_batch -exportoids 1.3.6.1.6.3.1.1.5.1|1.3.6.1.6.3.1.1.5.2 -filetype csv
-exportscope

ALL|TRAPS|OBJECTS

 Используйте эту опцию, чтобы указать, к какому типу должны относиться экспортируемые объекты MIB. Эта опция принимает следующие значения:
  • ALL: Экспортируются все объекты MIB.
  • TRAPS: Экспортируются только прерывания.
  • OBJECTS: Экспортируются только объекты.

Примеры:

nco_mibmanager_batch -exportdir "/home/user/export_cmd" -exportnewdirectory -filetype csv -exportscope ALL
-filetype

csv|
csv_trap_objects|
html_with_frames|
html_without_frames|
lookup|ncil|nckl_1_1|
nckl_3_0|standalone|
text|
tivoli_universal_agent

 Используйте эту опцию, чтобы задать формат файлов, в которые будут экспортироваться сгенерированные файлы. Эта опция необходима для операций экспорта.
Доступные параметры:
  • csv: Все данные объектов записываются в CSV-файлы.
  • csv_trap_objects: Все данные прерываний и связанные данные объектов привязок переменных записываются в один файл CSV.
  • html_with_frames: Создается веб-страницы, подходящая для публикации.
  • html_without_frames: Создается один файл oids.html, содержащий список всех объектов, выбранных для вывода.
  • lookup: Выводит все указанные значения в виде списка, разделенного табуляторами и подходящего для включения в любой файл правил.
  • ncil: Создается отдельный файл включения для каждого предприятия, и генерируется один файл для общих прерываний.
  • nckl_1_1: Создаются файлы, подходящие для использования в сочетании с Netcool Knowledge Library V1.1.
  • nckl_3_0: Создаются файлы, подходящие для использования в сочетании с Netcool Knowledge Library V3.
  • standalone: Создается один файл правил, содержащий все выбранные прерывания.
  • text: Создается отдельный текстовый файл для каждого OID.
  • tivoli_universal_agent: Создаются файлы .mdl и trapcnfg.
-help Используйте эту опцию, чтобы увидеть справочную информацию об опциях командной строки.
-importdir строка Используйте эту опцию, чтобы задать каталог, из которого импортируются файлы MIB. Заключите путь каталога в двойные кавычки (").

Примеры:

nco_mibmanager_batch -importdir "/home/user/mibs"
-importdirtraverse Используйте эту опцию, чтобы указать, что MIB Manager производит просмотр в подкаталогах, когда ищет файлы MIB для импорта.

Примеры:

nco_mibmanager_batch -importdir "/home/user/mibs" -importdirtraverse
-messagelevel

ERROR|WARN|
INFO|DEBUG|VERBOSE|NONE

 Используйте эту опцию, чтобы задать уровень записи сообщений в журнал. Эта опция принимает следующие значения:
  • ERROR: В журнал записываются только сообщения об ошибках.
  • WARN: В журнал записываются сообщения с предупреждениями и сообщения об ошибках.
  • INFO: В журнал записываются информационные, предупреждающие и критические сообщения.
  • DEBUG: В журнал записываются все сообщения.
  • VERBOSE: Подробное сообщение включает в себя состояние переменных, массивов и хешей при всяком изменении значения или состояния.
  • NONE: В журнал не записывается никаких сообщений.
-messagelog строка Используйте эту опцию, чтобы задать каталог, в который записывается файл журнала сообщений, mibmanager.log. Заключите путь каталога в двойные кавычки (").

Значение по умолчанию - ДОМАШНИЙ_КАТАЛОГ_NC/omnibus/log.
-searchpath строка Используйте эту опцию, чтобы задать разделенный точками с запятой список каталогов, в которых MIB Manager может производить поиск для разрешения зависимостей MIB. Заключите путь каталога в двойные кавычки (").

Примеры:

nco_mibmanager_batch -importdir "/home/user/mibs" -searchpath "/home/user/mib_dependencies"

Приведенная ниже команды производит поиск зависимостей в трех указанных каталогах:

nco_mibmanager_batch -importdir "/home/user/mibs" -searchpath "/home/user/mib_dependencies;/home/user/mib_other;/home/user/mib_test"
