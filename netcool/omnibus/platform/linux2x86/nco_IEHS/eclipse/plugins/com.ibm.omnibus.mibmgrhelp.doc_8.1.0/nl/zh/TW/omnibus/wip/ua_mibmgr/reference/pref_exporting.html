設定匯出喜好設定

本節說明如何設定規則檔匯出喜好設定。

若要設定目錄喜好設定，請按一下 喜好設定，或從主功能表中按一下檔案 > 喜好設定，然後從「喜好設定」視窗的左窗格中，按一下匯出。

  • 在併入路徑欄位中，指定隨 Netcool Include Library (NCiL) 規則檔套件（如果已安裝的話）一起安裝的規則位置。 只有在您匯出 Netcool Include Library 檔案格式的資料且目錄路徑必須指向併入規則時，才需要這個選項。因此，不需要手動編輯新規則檔案。預設值為 NCHOME/omnibus/probes/arch。
  • 在預設匯出檔格式欄位中，為輸出檔指定 Windows/DOS 格式 (CRLF) 或 Unix 格式 (LF)。這個內容可讓您在 Windows 機器上建立規則，然後將規則複製到 UNIX 機器而不執行公用程式（如 dos2unix）。
    註： 這個選項適用於所有輸出檔，並不受限於規則檔案。預設值為 Unix 格式。
  • 在警示類型欄位中，當設陷的 @Type 欄位在「詳細資料」視圖中編輯時，指定顯示的下拉清單選項。

如果之前已選取 NCKL 匯出格式，則只能使用下列選項：

  • 如果選取併入 varbind 說明勾選框，則 varbind 說明會新增為註解說明文件。 預設值是 enabled。
  • 如果選取計算 @AlertKey 的較佳值勾選框，則會從 varbind 母項物件產生 @AlertKey 值。預設值是 enabled。
  • 如果選取每個企業設定一次 @Agent 勾選框，則會為每個企業設定 @Agent 欄位。@Agent 欄位基於針對規則檔處理的 MIB 模組名稱。 預設值是 enabled。
  • 如果選取 Add @Class template 勾選框，則會設定 @Class 欄位。如果選取新增 @Class 範本勾選框，則 @Class 欄位會設為 300。預設值是 enabled。
