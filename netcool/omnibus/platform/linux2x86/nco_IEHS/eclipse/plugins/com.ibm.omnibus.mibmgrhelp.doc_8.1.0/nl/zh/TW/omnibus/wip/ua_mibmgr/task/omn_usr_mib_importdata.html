IBM Tivoli Netcool/OMNIbus 8.1 版

匯入 MIB 資料

如何將 SNMP MIB 資料匯入 MIB Manager。

開始之前

在「喜好設定」視窗中，指定一般由供應商 MIB 使用之 MIB 的位置。請確保您的 MIB 位於供應商特定的目錄中，且任何關聯的設備號碼位於子目錄中。另請確保這些目錄不包含任何非 MIB 文字檔。大部分 MIB 都安裝在 NCHOME/omnibus/var/mibmanager/mibs/base 目錄中，而所有 RFC MIB 都安裝在 NCHOME/omnibus/var/mibmanager/mibs/rfc 目錄中。

您正在匯入的 MIB 可能仰賴於其他 MIB。若要確保 MIB Manager 可以在執行匯入時搜尋相依的 MIB，請設定相依 MIB 檔案位置的搜尋路徑。

程序

若要匯入 MIB 資料，請執行下列動作：

  1. 按一下 匯入，或從功能表中按一下檔案 > 匯入。
  2. 在目錄欄位中，指定要匯入的 MIB 模組位置。
  3. 若要讓 MIB Manager 剖析在可用子目錄中找到的所有 MIB，請選取遍訪子目錄勾選框。
  4. 按一下匯入。

    剖析 MIB 模組時，會顯示「匯入狀態」視窗。 在剖析匯入的 MIB 檔案期間探索到每個 MIB 物件類型時，此視窗的左側會顯示不同的 MIB 物件類型號碼。

    剖析 MIB 之後，MIB Manager 會檢查每個 MIB 模組並剖析找到的所有 MIB 物件。 剖析所有物件之後，MIB Manager 會檢查每個 MIB 並處理 import 陳述式以尋找所有參照的 MIB 模組。 如果在之前匯入的 MIB 模組中找不到任何參照的 MIB 模組，則 MIB Manager 會搜尋在搜尋路徑中指定的目錄。「匯入狀態」視窗右側顯示的圖示，用來顯示所匯入 MIB 中找到的任何 import 陳述式狀態，如下表所示：
    圖示 說明
    指出 import 陳述式尚未解析。
    指出 import 陳述式已解析。
    指出無法解析 import 陳述式中參照的 MIB 模組。

    您可以展開 import 陳述式樹狀結構來顯示哪些 MIB 模組使用 import 陳述式來參照其他 MIB 模組。

    匯入完成會在已處理完所有匯入的 MIB 檔案時，顯示在狀態訊框的頂部。您可以按一下檢視狀態歷程並使用檢視下拉清單選取過濾選項，來檢視匯入期間產生的所有狀態訊息。

  5. 按一下跳出以關閉「匯入狀態」視窗。

結果

「MIB 模組」視圖會自動重新整理，以顯示最近匯入的 MIB 模組名稱。「OID 樹狀結構」視圖會自動重新整理，以顯示所有最近匯入的 MIB 物件。為了重建樹狀結構，MIB Manager 會遞迴地搜尋樹狀結構的母節點（iso、ccitt 及 joint-iso-ccitt）來尋找子節點，直到找到沒有相關聯子項的子節點為止。如果未偵測到任何錯誤，則完全移入的 MIB 樹狀結構會顯示在「OID 樹狀結構」視圖中，其中包含剖析的物件。

按一下「OID 樹狀結構」視圖中的物件，即會自動顯示「MIB 模組」視圖中的物件。「詳細資料」視圖會顯示所選取物件的相關詳細資訊。

匯入的 MIB 模組儲存在 XML 檔案中，該檔案代表該 MIB 模組的結構。XML 檔案位於 NCHOME/omnibus/var/mibmanager/data 目錄中。

如果偵測到無法分辨的 import 陳述式，請聯絡負責遺漏的一個以上 MIB 的供應商，將遺漏的一個以上檔案新增至搜尋路徑並重複匯入作業。

註：

供應商有時會指定重複的物件名稱，或指定的物件名稱與 RFC 中指定的物件名稱相同。

例如，物件 system 定義於模組 SNMPv2-MIB（OID 為 1.3.6.1.2.1.1）中，而另一個物件 system 定義於模組 WINDOWS-NT-PERFORMANCE（OID 為 1.3.6.1.4.1.311.1.1.3.1.1.23）中。 此外，物件 sysDescr 在模組 SNMPv2-MIB 中定義為 system.1，而另一個物件 sysFileReadOperationsPerSec 在模組 WINDOWS-NT-PERFORMANCE 中定義為 system.1。 在此情況下，不清楚與子物件相關聯的母項物件。

找到具有重複母項名稱的物件時，MIB Manager 會嘗試尋找定義於相同 MIB 模組中的母項。在前一個範例中，則為 WINDOWS-NT-PERFORMANCE。如果沒有任何母項定義於相同的 MIB 模組中，則 MIB Manager 會繼續搜尋以查看母項是否定義於該物件相依的其他 MIB 模組中。 MIB Manager 還會識別參照該母項的其他 MIB 模組，並選取最常參照的母項名稱。但是，這樣做會導致錯誤地移入 MIB 樹狀結構以及錯誤地計算 OID 數目。因此，MIB Manager 每次找到重複的物件名稱時，都會在除錯檔案中記載警告並記錄解決重複問題所採取的動作。
