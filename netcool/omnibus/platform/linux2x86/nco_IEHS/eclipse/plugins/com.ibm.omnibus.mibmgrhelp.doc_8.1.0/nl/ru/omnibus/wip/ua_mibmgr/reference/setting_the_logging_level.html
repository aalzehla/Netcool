Задание уровня записи в журнал

Уровень записи в журнал можно изменить для создания более подробных файлов журналов или трассировок, что помогает при отладке MIB Manager. Дополнительная информация, предоставляемая при увеличении уровня записи в журнал, может помочь в отладке конкретных ошибок при синтаксическом анализе MIB.

Для изменения уровня записи в журнал выберите на панели меню главного окна пункт Изменить > Отладка. Для файлов журналов есть шесть уровней сообщений, которые можно использовать для последовательного повышения уровня захватываемых подробностей: none (нет), error (ошибка), warn (предупреждение), info (информация), debug (отладка) и verbose (подробный).

В следующей таблице предоставлено краткое описание каждого уровня записи в журнал:

Табл. 1. Список уровней записи в журнал со связанными описаниями.
Уровень журнала Описание
none

Задает уровень записи в журнал, при котором никакие отладочные сообщения не выводятся.

Производительность прикладной программы повышается, но при синтаксическом анализе никакие сообщения с предупреждениями или ошибками не выводятся.
error

Задает уровень записи в журнал, при котором в журнал заносятся только записи об ошибках, влияющих на результаты.

Сюда относятся все ошибки, обнаруженные MIB Manager, которые могут привести к выводу в главном окне неправильных данных.
warn

Задает уровень записи в журнал, при котором записываются сообщения об ошибках и предупреждения.

Предупреждение обозначает возникновение условий, которые могут привести к ошибочным данным и поэтому должны быть проверены. Однако эти условия не приводят к повреждению данных.
info

Задает уровень записи в журнал, при котором в журнал записываются информационные, предупреждающие и критические сообщения.

Информационные сообщения содержат подробности об общем ходе выполнения задачи. Например, в этих сообщениях приводится полный текст для объектов, создающийся при их синтаксическом анализе.
debug
Задает уровень записи в журнал, при котором записываются все сообщения.
Прим.: Запускайте MIB Manager в режиме debug только при необходимости разобраться с условиями возникновения ошибок. Объем данных, производимых при синтаксическом анализе даже умеренного числа MIB быстро заполнит всю вашу файловую систему.
verbose

Задает уровень записи в журнал, при котором записываются все информационные сообщения.

Сообщение verbose включает в себя состояние переменных, массивов и хешей при всяком изменении значения или состояния.
