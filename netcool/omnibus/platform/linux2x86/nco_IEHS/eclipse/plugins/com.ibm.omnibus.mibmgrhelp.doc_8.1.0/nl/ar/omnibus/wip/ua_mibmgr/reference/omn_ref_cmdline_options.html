اختيارات سطر أمر MIB Manager

MIB Managerبه وظيفة سطر الأمر التى يمكنك استخدامها لاستقبال بيانات MIB وارسال ملفات القواعد. يمكنك اصدار الأوامر يدويا أو استدعائها برمجيا.

لبدء وظيفة سطر الأمر، قم باستخدام الأمر التالي:
  • $NCHOME/omnibus/bin/nco_mibmanager_batch [الاختيارات]
  • %NCHOME%\omnibus\bin\nco_mibmanager_batch [الاختيارات]

    على أنظمة تشغيل Windows، يتم عرض المخرجات منMIB Manager فى شاشة تحكم رئيسية جديدة.

الجدول التالي يصف اختيارات سطر الأمر المتاحة. تستطيع الاختيارات -exportmibs و -exportobjects و -exportoids التحليل اللغوى للتعبيرات المعتادة التى تتماشى مع آلية نموذج Java.

قم باستخدام مسارات الدليل الكاملة عندما تقوم بتحديد الدلائل. على سبيل المثال: nco_mibmanager_batch -importdir "C:\anydir\mibs"

جدول 1. اختيارات سطر أمر MIB Manager
اختيار سطر الأمر الوصف
-clean قم باستخدام هذا الاختيار عندما تقوم بتشغيل وظيفة سطر أمر MIB Manager لأول مرة، اذا قمت فى السابق باستخدام وظيفة MIB Manager البيانية. ان لم تستخدم هذا الاختيار، يتم عرض خطأ.
-exportdir string قم باستخدام هذا الاختيار لتحديد الدليل الذى يتم ارسال ملفات القواعد التى يتم تكوينها اليه. قم بتضمين مسار الدليل بين علامات تنصيص مزدوجة (").

على سبيل المثال:

nco_mibmanager_batch -exportdir "/home/user/export_cmd" -filetype csv
-exportmibs string قم باستخدام هذا الاختيار لتحديد اسم MIB المقرر ارساله. يمكنك استخدام التعبيرات المعتادة لتحديد العديد من MIBs.

على سبيل المثال، الأمر التالي يحقق ارسال العناصر من MIB يسمى "RMON-MIB":

nco_mibmanager_batch -exportmibs RMON-MIB -filetype csv

الأمر التالي يحقق ارسال العناصر من كل MIBs التى تحتوى أسماؤها على "MON":

nco_mibmanager_batch -exportmibs .*MON.* -filetype csv
-exportnewdirectory قم باستخدام هذا الاختيار لتحديد أن الملفات التى يتم ارسالها تكتب الى دليل جديد، داخل دليل الارسال القائم. يتم تسمية الدليل الفرعى الجديد آليا بخاتم الوقت. قم باستخدام هذا الاختيار لضمان ألا يتم احلال ملفات القواعد القائمة.

على سبيل المثال:

nco_mibmanager_batch -exportdir "/home/user/export_cmd" -exportnewdirectory -filetype csv
-exportobjects string قم باستخدام هذا الاختيار لتحديد اسم عنصر MIB المقرر ارساله. يمكنك استخدام التعبيرات المعتادة لتحديد العديد من العناصر.

على سبيل المثال، الأمر التالي يحقق ارسال كل العناصر بالاسم "coldStart":

nco_mibmanager_batch -exportobjects coldStart -filetype csv

الأمر التالي يحقق ارسال كل العناصر التى تبدأ اسمائها بواسطة "cold":

nco_mibmanager_batch -exportobjects cold.* -filetype csv
-exportoids string قم باستخدام هذا الاختيار لتحديد OID ليتم ارساله. يمكنك استخدام التعبيرات المعتادة لتحديد OID متعددة.

على سبيل المثال، الأمر التالي يحقق ارسال كل العناصر ذات OID "1.3.6.1.6.3.1.1.5.1":

nco_mibmanager_batch -exportoids 1.3.6.1.6.3.1.1.5.1 -filetype csv

الأمر التالي يحقق ارسال كل العناصر التى لها OID يبدأ بواسطة "1.3.6.1.6.3.1.1.5":

nco_mibmanager_batch -exportoids 1.3.6.1.6.3.1.1.5.* -filetype csv

الأمر التالي يحقق ارسال كل العناصر ذات OIDs "1.3.6.1.6.3.1.1.5.1" أو "1.3.6.1.6.3.1.1.5.2":

nco_mibmanager_batch -exportoids 1.3.6.1.6.3.1.1.5.1|1.3.6.1.6.3.1.1.5.2 -filetype csv
-exportscope

ALL|TRAPS|OBJECTS

 قم باستخدام هذا الاختيار لتحديد أى نوع من عناصر MIB يتم ارساله. هذا الاختيار يأخذ القيم التالية:
  • ALL: يتم ارسال كل عناصر MIB.
  • TRAPS: يتم ارسال الصفحات الشرطية فقط.
  • OBJECTS: يتم ارسال العناصر فقط.

على سبيل المثال:

nco_mibmanager_batch -exportdir "/home/user/export_cmd" -exportnewdirectory -filetype csv -exportscope ALL
-filetype

csv|
csv_trap_objects|
html_with_frames|
html_without_frames|
lookup|ncil|nckl_1_1|
nckl_3_0|standalone|
text|
tivoli_universal_agent

 قم باستخدام هذا الاختيار لتحديد نسق الملف الذى يتم به ارسال الملفات التى يتم تكوينها. يعد هذا الاختيار ضروريا لعمليات الارسال.
المعاملات المتاحة تكون كما يلى:
  • csv: يتم كتابة كل بيانات العناصر الى ملفات CSV.
  • csv_trap_objects: يتم كتابة كل بيانات الصفحة الشرطية وبيانات عنصر وحدات ربط المتغير الى ملف CSV منفرد.
  • html_with_frames: يتم تكوين صفحة انترنت تناسب عملية النشر.
  • html_without_frames: يقوم بتكوين ملف منفرد يسمى oids.html يحتوى على كشف بكل العناصر التى تم تحديدها للمخرجات.
  • lookup: يتم اخراج كل القيم المحددة فى كشف مبوب مناسب لكى يتم تضمينه فى أى ملف للقواعد.
  • ncil: يتم تكوين ملفات تضمين منفصلة لكل مشروع ويتم تكوين ملف منفرد للصفحات الشرطية العامة.
  • nckl_1_1: يتم تكوين الملفات المناسبة للاستخدام مع Netcool Knowledge Library V1.1.
  • nckl_3_0: يتم تكوين الملفات المناسبة للاستخدام مع Netcool Knowledge Library V3.x.
  • standalone: يتم تكوين ملف قواعد منفرد يحتوى على كل الصفحات الشرطية المحددة.
  • text: يتم تكوين ملف نصى منفصل لكل OID.
  • tivoli_universal_agent: يتم تكوين ملفات .mdl و trapcnfg.
-help قم باستخدام هذا الاختيار لعرض معلومات المساعدة حول اختيارات سطر الأمر.
-importdir string قم باستخدام هذا الاختيار لتحديد الدليل الذى يتم استقبال ملفات MIB منه. قم بتضمين مسار الدليل بين علامات تنصيص مزدوجة (").

على سبيل المثال:

nco_mibmanager_batch -importdir "/home/user/mibs"
-importdirtraverse قم باستخدام هذا الاختيار لتحديد أن MIB Manager يتجاوز الدلائل الفرعية عندما يقوم بالبحث عن ملفات MIB ليتم استقبالها.

على سبيل المثال:

nco_mibmanager_batch -importdir "/home/user/mibs" -importdirtraverse
-messagelevel

ERROR|WARN|
INFO|DEBUG|VERBOSE|NONE

 قم باستخدام هذا الاختيار لتحديد مستوى تسجيل الرسالة. هذا الاختيار يأخذ القيم التالية:
  • ERROR: يتم تسجيل رسائل الخطأ فقط.
  • WARN: يتم تسجيل رسائل الخطأ والتحذير.
  • INFO: يتم تسجيل الرسائل الاعلامية ورسائل التحذير والرسائل الهامة.
  • DEBUG: يتم تسجيل كل الرسائل.
  • VERBOSE: تتضمن الرسائل الطويلة حالة المتغيرات والمصفوفات وعلامات # بينما تتغير الحالة والقيمة.
  • NONE: لا يتم تسجيل رسائل.
-messagelog string قم باستخدام هذا الاختيار لتحديد الدليل الذى يتم كتابة ملف سجل الرسالة، mibmanager.log، اليه. قم بتضمين مسار الدليل بين علامات تنصيص مزدوجة (").

المفترض هو NCHOME/omnibus/log.
-searchpath string قم باستخدام هذا الاختيار لتحديد كشف محدد بالفاصلة المنقوطة يحتوى على الدلائل يستطيع MIB Manager الحث فيه لحل ارتباطات MIB. قم بتضمين مسار الدليل بين علامات تنصيص مزدوجة (").

على سبيل المثال:

nco_mibmanager_batch -importdir "/home/user/mibs" -searchpath "/home/user/mib_dependencies"

الأمر التالي يبحث فى 3 دلائل خاصة عن الارتباطات:

nco_mibmanager_batch -importdir "/home/user/mibs" -searchpath "/home/user/mib_dependencies;/home/user/mib_other;/home/user/mib_test"
