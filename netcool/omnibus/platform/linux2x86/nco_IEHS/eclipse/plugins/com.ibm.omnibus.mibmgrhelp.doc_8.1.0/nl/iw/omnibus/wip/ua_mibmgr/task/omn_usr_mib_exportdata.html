IBM Tivoli Netcool/OMNIbus, מהדורה 7.4

יצוא נתונים של MIB

כיצד לייצא נתונים של SNMP MIB data מ-MIB Manager למגוון תבניות.

כדי לייצא נתוני MIB:

  1. לחצו על יצוא ולחילופין, מהתפריט, לחצו על קובץ > יצוא.
  2. בשדה מדריך, ציינו את מדריך היעד עבור הקבצים שיוצאו.
  3. בשדה סוג קובץ, ציינו את סוג הקובץ של הקבצים שיוצאו.

    יש להשתמש בסוג Netcool Knowledge Library‏ (NCKL) מתי שניתן, מאחר שכללי ה-NCKL מבוססים על התיפקודיות של ההתקן. בעת השימוש ב-NCKL, עליכם לציין את סוג הקובץ Netcool Knowledge Library מהדורה 3.x. פעולה זו תאפשר להוסיף את הקבצים המופקים להתקנות קיימות של Netcool Include Library.

    האפשרות שלהלן זמינות מהרשימה הנפתחת סוג קובץ:

    סוג קובץ תיאור
    CSV בחרו באפשרות זו כדי לציין שכל נתוני האובייקטים נטענים לקובצי CSV, כך שניתן יהיה להציג אותם באמצעות יישום של גיליון אלקטרוני.
    אובייקטי לכידה של CSV בחרו באפשרות זו כדי לציין שכל נתוני הלכידות ונתוני האובייקטים של איגודי המשתנים המשויכים נטענים לקובץ CSV יחיד, כך שיהיה ניתן להציג אותם באמצעות יישום של גיליון אלקטרוני.
    HTML עם מסגרות בחרו באפשרות זו כדי ליצור דף אינטרנט המתאים לפרסום.

    התצוגה של עץ ה-MIB שנבחר מוצגת בצדו השמאלי של דף האינטרנט המופק. ניתן לנווט בתצוגה של עץ ה-MIB על-ידי הרחבה וצמצום של הצמתים בעץ ה-MIB. בעת בחירת מרכיב מעץ ה-MIB, המידע המשויך אליו מוצג בצדו הימני של דף האינטרנט.

    מבנה זה יכול ליצור לעיתים קרובות מספר גדול של קובצי יצוא, שכל אחד מהם מכיל קובץ HTML עבור כל אובייקט בעץ ה-MIB. ניתן לדחוס את מדריך היעד ולמקם אותו תחת כל שרת אינטרנט, או שניתן להציג את הדפים ישירות מהדיסק על-ידי פתיחת הקובץ index.html שנוצר במדריך הייצוא.

    אם ברצונכם להשתמש בקובצי HTML נפרדים עם דפדפן אינטרנט, ייצאו אל HTML עם מסגרות ולאחר מכן השתמשו בקבצים שבתת-מדריך data הממוקם תחת מדריך הייצוא.
    HTML ללא מסגרות בחרו באפשרות זו כדי ליצור קובץ יחיד בשם oids.html. קובץ זה יכיל רשימה של כל האובייקטים שנבחרו להפקת פלט, כאשר כל אחד מהם מופרד על-ידי כללים אופקיים (באמצעות התווית <HR>). כל OID תחום בתווית עוגן כך שבעת טעינתו הדפדפן יגלול כלפי מפה באופן אוטומטי לכל OID שכולל סמל של סולמית (#) ב-URL, לדוגמה: file:c:/oids.html#1.3.6.1.4.1.9.

    הנפח של קובץ זה עשוי לגדול מאוד אם אתם מייצאים מספר רב של אובייקטים. לכן, יש להשתמש בתבנית זו כדי לייצוא מספר קטן של לכידות בלבד.
    טבלת בדיקת מידע של Netcool בחרו באפשרות זו כדי להפיק פלט של כל הערכים שצוינו ברשימה עם לשוניות, כאשר רכיבי ה-OIDs מוצגים בעמודה השמאלית ושמות האובייקטים מוצגים בעמודה הימנית. ניתן לכלול פריט זה בכל קובץ של כללים.
    Netcool Include Library בחרו באפשרות זו כדי ליצור קובצי הכללה נפרדים לכל ארגון, וכדי להפיק קובץ יחיד עבור הלכידות הגנריות (כלומר, לכל הלכידות שאינן ממוקמות מתחת לתת-עץ של הארגון). ההנחה היא שקובצי ההכללה מתווספים לקובצי snmp.rules קיימים אשר נוצרו קודם לכן על-ידי MIB Manager או שנוצרו על-ידי Netcool Include Library‏ (NCIL).
    Netcool Knowledge Library מהדורה 1.1 בחרו באפשרות זו כדי להפיק פלט של קבצים מ-MIB Manager. הקבצים ייכללו ביישום קיים של Netcool Knowledge Library.
    Netcool Knowledge Library מהדורה 3.x בחרו באפשרות זו כדי להפיק פלט של קבצים מ-MIB Manager. הקבצים ייכללו ביישום קיים של Netcool Knowledge Library.
    כללים עצמאיים בחרו באפשרות זו כדי ליצור קובץ כללים יחיד המכיל את כל הלכידות שנבחרו. ניתן לציין קובץ זה בקובץ Properties של כל בדיקת לכידה של IBM Tivoli Netcool/OMNIbus SNMP trap probe.
    קובצי תמליל נפרדים התבנית 'קובצי תמליל נפרדים' משמשת עבור תפריט הלחיצה הימנית של רשימת האירועים של Netcool.

    כאשר אתם מייצאים לקובצי תמליל נפרדים, הפלט של כל OID נוצר כקובץ תמליל נפרד, לדוגמה 1.3.6.1.4.1.9.1.1.txt. לאחר מכן, ניתן להוסיף כלי חיצוני לתפריט הלחיצה הימנית המשתמש ב-nco_message להצגת כל מידע האובייקטים עבור האירוע שצוין עבורו OID בשדה, לדוגמה: cat 1.3.6.1.4.1.9.1.txt |nco_message stdin title "trap info".
    Tivoli Universal Agent התבנית של Tivoli Universal Agent מייצאת קבצים מסוג .mdl ו-trapcnfg בהתאם לתוכן שנבחר.
    ניתן לבחור בתת-עצים מרובים בעת בחירה בתבנית זו לייצוא קבצים. אם בוחרים בתת-עצים מרובים, קבצים מסוג .mdl ו-trapcnfg נוצרים עבור כל האובייקטים הממוקמים מתחת לפריט שנבחר בעץ ה-MIB.
    הערה: שלוש האותיות הראשונות בשם היישום, המופיעות אחרי התווית ‎//APPL בקובץ .mdl חייבות להיות ייחודיות בתוך הארגון. MIB Manager מנסה לעמוד בדרישה, זו אך הוא אינו יכול לדעת אודות סוכנים אחרים פעילים אחרים. ייתכן שיהיו עליכם לשנות את קובץ ה-.mdl כדי להבטיח ששם היישום ייחודי בארגון.
  4. בשדה היקף, השתמשו ברשימה הנפתחת כדי לציין אם על MIB Manager להפיק פלט עבור לכידות בלבד, אובייקטים בלבד, או עבור כל רכיבי ה-OID.

    האפשרויות הזמינות תלויות בסוג הקובץ שאותו אתם מייצאים.

  5. כדי להגביל את מספר האובייקטים הכלולים בקובץ המופק, בחרו בתיבת הסימון תת-עצים נבחרים בלבד.
  6. לחצו על יצוא כדי לייצא את הקבצים למדריך היעד.
