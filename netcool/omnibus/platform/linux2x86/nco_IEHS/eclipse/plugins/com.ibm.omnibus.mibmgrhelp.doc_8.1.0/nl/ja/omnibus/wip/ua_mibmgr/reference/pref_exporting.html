エクスポート設定のセットアップ

このセクションでは、ルール・ファイルのエクスポート設定を セットアップする方法について説明します。

ディレクトリー設定をセットアップするには、 「設定」をクリックするか、メインメニューから、 「ファイル」 > 「設定」をクリックして、「設定」ウィンドウの左側のペインから 「エクスポート」をクリックします。

  • 「インクルード・パス」フィールドに、 Netcool Include Library (NCiL) ルール・ファイル・パッケージ (インストールされている場合) を 使用してインストールされたルールの場所を指定します。 このオプションが必要なのは、データを Netcool Include Library ファイル・フォーマットで エクスポートし、組み込まれたルールを ディレクトリー・パスが指す必要がある場合のみです。結果として、新規ルール・ファイルを手動で編集する 必要はありません。デフォルト値は NCHOME/omnibus/probes/arch です。
  • 「デフォルトのエクスポート・ファイル・フォーマット」フィールドで、 出力ファイル用に Windows/DOS フォーマット (CRLF) または Unix フォーマット (LF) を 指定します。このプロパティーを使用することで、Windows マシンでルールを作成して、ユーティリティー (dos2unix など) を実行することなく、 作成したルールを UNIX マシンにコピーできます。
    注: この選択肢は、ルール・ファイルだけでなく、 すべての出力ファイルに適用されます。デフォルト値は Unix フォーマットです。
  • 「アラート・タイプ」フィールドで、 「詳細」ビューの、トラップの「@Type」フィールド編集時に表示される ドロップダウン・リスト・オプションを指定します。

以下のオプションは、NCKL エクスポート・フォーマットが 事前に選択されている場合のみ使用できます。

  • 「varbind の説明を含める」チェック・ボックスが選択されている場合、 varbind の説明はコメント資料として追加されます。 デフォルト値は「使用可能」です。
  • 「@AlertKey のより適切な値を計算」チェック・ボックスが選択されている場合、 「@AlertKey」の値は、 varbind 親オブジェクトから生成されます。デフォルト値は「使用可能」です。
  • 「エンタープライズごとに @Agent を 1 回設定」チェック・ボックスが選択されている場合、 「@Agent」フィールドは、エンタープライズごとに 設定されます。「@Agent」フィールドは、ルール・ファイル用 に処理される MIB モジュールの名前に基づいています。 デフォルト値は「使用可能」です。
  • 「@Class テンプレートを追加」チェック・ボックスが選択されている場合、 「@Class」フィールドが設定されます。「@Class テンプレートを追加」チェック・ボックスが選択されている場合、「@Class」フィールドが 300 に設定されます。デフォルト値は「使用可能」です。
