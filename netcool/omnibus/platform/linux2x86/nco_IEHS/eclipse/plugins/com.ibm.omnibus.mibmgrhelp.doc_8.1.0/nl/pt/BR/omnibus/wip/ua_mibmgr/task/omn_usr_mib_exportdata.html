IBM Tivoli Netcool/OMNIbus Versão 8.1

Exportando dados MIB

Como exportar dados MIB de SNMP do MIB Manager para uma variedade de formatos.

Procedimento

Para exportar dados MIB:

  1. Clique no Exportar ou a partir do menu, clique em Arquivo > Exportar.
  2. No campo Diretório, especifique o diretório de destino para os arquivos de exportação.
  3. No campo Tipo de Arquivo, especifique o tipo de arquivo para os arquivos de exportação.

    É necessário usar o Netcool Knowledge Library (NCKL) sempre que possível, porque os arquivos de regras NCKL baseiam-se na funcionalidade do dispositivo. Ao usar o NCKL, é necessário especificar o tipo de arquivo do Netcool Knowledge Library versão 3.x. O que irá ativar os arquivos gerados a serem adicionados às instalações existentes do Netcool Include Library.

    As opções a seguir estão disponíveis a partir da lista suspensa Tipo de Arquivo:

    Tipo de arquivo Descrição
    CSV Selecione esta opção para especificar que todos os dados do objeto sejam carregados nos arquivos CSV, de modo que ele possa ser visualizado usando um aplicativo de planilha.
    Objetos Trap CSV Selecione esta opção para especificar que todos os dados trap e dados do objeto de ligações de variáveis associadas sejam carregados em um único arquivo CSV, de modo que ele possa ser visualizado usando um aplicativo de planilha.
    HTML com quadros Selecione esta opção para criar uma página da web que seja adequada para publicação.

    A visualização da árvore MIB selecionada será exibida no lado esquerdo da página da web gerada. É possível navegar pela visualização da árvore MIB expandindo e reduzindo nós na árvore MIB. Quando um elemento é selecionado da árvore MIB, as informações associadas são exibidas no lado direito da página da web.

    Este formato pode frequentemente criar um grande número de arquivos de exportação, cada um contendo um arquivo HTML para cada objeto listado na árvore MIB. O diretório de destino pode estar compactado e localizado em qualquer servidor ou as páginas podem ser visualizadas diretamente a partir de um disco abrindo-se o arquivo index.html criado no diretório de exportação.

    Se você deseja usar arquivos HTML individuais com um navegador da web, faça a exportação para HTML com quadros e em seguida use os arquivos no subdiretório dados localizado no diretório de exportação.
    HTML sem quadros Selecione esta opção para criar um arquivo único chamado oids.html. Este arquivo irá conter uma lista de todos os objetos selecionados para a saída, cada qual separado por réguas horizontais (usando a tag <HR>). Cada OID encontra-se delimitada por uma tag âncora de modo que quando estiver carregada, o navegador irá rolar automaticamente até encontrar algum OID que inclua um símbolo hash (#) na URL, por exemplo: file:c:/oids.html#1.3.6.1.4.1.9.

    Este arquivo poderá se tornar muito grande se você estiver exportando um grande número de objetos. Por esta razão, este formato deverá somente ser usado para a exportação de um pequeno número de traps.
    Netcool Lookup Table Selecione esta opção para realizar a saída de todos os valores especificados na lista tabulada, com os OIDs exibidos na coluna a esquerda e os nomes dos objetos exibidos na coluna a direita. O que é adequado para a inclusão no arquivo de regras.
    Netcool Include Library Selecione esta opção para criar arquivos de inclusão separados para cada empresa e para gerar um arquivo único para os traps genéricos (ou seja, traps que não estejam localizados abaixo da subárvore da empresa). Presume-se que os arquivos include estejam inseridos nos arquivos snmp.rules existentes, naqueles previamente criados pelo MIB Manager ou naqueles criados pelo Netcool Include Library (NCIL).
    Netcool Knowledge Library versão 1.1 Selecione esta opção para gerar saída de arquivos do MIB Manager. Os arquivos serão incluídos em uma implementação existente do Netcool Knowledge Library.
    Netcool Knowledge Library versão 3.x Selecione esta opção para gerar saída de arquivos do MIB Manager. Os arquivos serão incluídos em uma implementação existente do Netcool Knowledge Library.
    Regras Independentes Selecione esta opção para criar um arquivo de regras único que contenha todos os traps selecionados. É possível especificar esse arquivo no arquivo de propriedades de qualquer análise de trap SNMP do Tivoli Netcool/OMNIbus.
    Individual Text Files O formato Individual Text Files é usado para o menu ativado pelo botão direito do Netcool Event List.

    Ao efetuar a exportação para os arquivos de texto individuais, cada OID efetua a saída para um arquivo de texto separado, por exemplo 1.3.6.1.4.1.9.1.1.txt. Uma ferramenta externa poderá então ser incluída ao menu ativado pelo botão direito que usa nco_message para exibir todas as informações para um evento que tenha o OID especificado em um campo, por exemplo: cat 1.3.6.1.4.1.9.1.txt |nco_message stdin título "trap info".
    Tivoli Universal Agent O formato Tivoli Universal Agent exporta arquivos .mdl e trapcnfg baseado no conteúdo selecionado.
    Diversas subárvores podem ser selecionadas quando este formato é selecionado para a exportação de arquivos. Se diversas subárvores forem selecionadas, os arquivos .mdl and trapcnfg serão criados para todos os objetos localizados abaixo de um item selecionado na árvore MIB.
    Nota: As primeiras três letras do nome do aplicativo, seguindo a identificação //APPL no arquivo .mdl, deverão ser exclusivas em uma empresa. O MIB Manager tenta respeitar esse requisito mas não pode saber se outros agentes estão em execução. Talvez seja necessário modificar o arquivo .mdl para assegurar que o nome do aplicativo seja exclusivo na empresa.
  4. No campo Escopo, use a lista suspensa para especificar se o MIB Manager gera saída somente de traps, somente de objetos ou de todos os OIDs.

    As opções disponíveis dependem do tipo de arquivo que estiver sendo exportado.

  5. Para limitar o número de objetos incluídos no arquivo gerado, selecione a caixa de seleção Somente Subárvore(s) Selecionada(s).
  6. Clique em Exportar para exportar os arquivos para o diretório de destino.
