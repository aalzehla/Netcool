내보내기 환경 설정 설정

이 절에서는 규칙 파일 내보내기 환경 설정의 설정 방법에 대해 설명합니다.

디렉토리 환경 설정을 설정하려면, 환경 설정을 클릭하거나 기본 메뉴에서 파일 > 환경 설정을 클릭한 후 환경 설정 창의 왼쪽 분할창에서 내보내기를 클릭하십시오.

  • 포함 경로 필드에서 NCiL(Netcool Include Library) 규칙 파일 패키지(설치된 경우)로 설치된 규칙의 위치를 지정하십시오. 이 옵션은 Netcool Include Library 파일 형식으로 데이터를 내보내는 경우 그리고 디렉토리 경로가 포함된 규칙을 가리켜야 하는 경우에만 필요합니다. 그러므로 새 규칙 파일은 직접 편집할 필요가 없습니다. 기본값은 NCHOME/omnibus/probes/arch입니다.
  • 기본 내보내기 파일 형식 필드에서는 출력 파일에 대해 Windows/DOS 형식(CRLF) 또는 Unix 형식(LF)을 지정하십시오. 이 특성을 사용하면 Windows 시스템에서 규칙을 작성하여 dos2unix와 같은 유틸리티를 실행하지 않고 UNIX 시스템에 복사할 수 있습니다.
    참고: 이 선택사항은 모든 출력 파일에 적용되며 규칙 파일에만 한정되지 않습니다. 기본값은 Unix 형식입니다.
  • 경보 유형 필드에서는 세부사항 보기에서 트랩의 @Type 필드를 편집할 경우 표시되는 드롭 다운 목록 옵션을 지정하십시오.

다음 옵션은 NCKL 내보내기 형식을 이전에 선택한 경우에만 사용할 수 있습니다.

  • varbind 설명 포함 선택란을 선택하면, varbind 설명이 주석 문서로 추가됩니다. 기본값은 사용입니다.
  • @AlertKey의 상위 값 계산 선택란이 선택되면, varbind 상위 오브젝트에서 @AlertKey 값이 생성됩니다. 기본값은 사용입니다.
  • 엔터프라이즈당 한 번 @Agent 설정 선택란이 선택되면, 각 엔터프라이즈에 대해 @Agent 필드가 설정됩니다. @Agent 필드는 규칙 파일에 대해 처리 중인 MIB 모듈의 이름을 기반으로 합니다. 기본값은 사용입니다.
  • @Class 템플리트 추가 선택란이 선택되면, @Class 필드가 설정됩니다. @Class 템플리트 추가 선택란이 선택되면, @Class 필드가 300으로 설정됩니다. 기본값은 사용입니다.
