Erweiterte Suchverfahren

Sie können Suchvorgänge effizienter gestalten, indem Sie die Suchergebnisse durch die Verwendung von Anführungszeichen, Platzhaltern und booleschen Operatoren eingrenzen.

  • Wenn Sie die Hilfefunktion durchsuchen, können Sie einzelne Wörter oder Ausdrücke als exakte Zeichenfolge eingeben, indem Sie sie in Anführungszeichen setzen.

    Beispiel: Wenn Sie im Feld Suchen die Zeichenfolge "log file" eingeben, suchen Sie nach allen Vorkommen der Zeichenfolge log file. Wenn Sie die Anführungszeichen nicht eingeben, liefert die Suche alle Themen, die sowohl den Begriff log als auch den Begriff file enthalten.

  • Mithilfe von Platzhalterzeichen wie einem Stern (*) und einem Fragezeichen (?) im Suchbegriff können Sie den Suchbereich erweitern.
    • Verwenden Sie den Stern (*) für mehrere unbekannte oder frei wählbare Zeichen in der Suchabfrage. Beispiel: Wenn Sie par* eingeben, werden alle Themen angezeigt, die partly, participate, partial oder andere Wörter mit par enthalten.
      Anmerkung: Bei dem Suchbegriff gilt eine Einschränkung: bei Verwendung eines Sterns (*) in einem Suchbegriff müssen dem Stern mindestens drei Zeichen vorangestellt sein. Dementsprechend muss ein Suchbegriff ohne ein Platzhalterzeichen mindestens drei zeichen beinhalten.
    • Verwenden Sie das Fragezeichen für ein einzelnes Zeichen in der Suchabfrage. Beispiel: Wenn Sie par? eingeben, werden alle Themen angezeigt, die part enthalten; es werden jedoch keine Themen angezeigt, die partial oder partly enthalten.
    Anmerkung: Bei Verwendung eines Sterns (*) in einem Suchbegriff müssen dem Stern mindestens drei Zeichen vorangestellt sein.
  • Sie können die Abfrage mithilfe der booleschen Operatoren AND, OR und NOT weiter eingrenzen. Beispiel:
    database AND "log file"
    Beschränkt das Suchergebnis auf Themen, die sowohl den Begriff database als auch den Ausdruck log file enthalten.
    database OR "log file"
    Dehnt das Suchergebnis auf Themen aus, die entweder den Begriff database oder den Ausdruck log file enthalten.
    database NOT "log file"
    Beschränkt das Suchergebnis auf Themen, die zwar den Begriff database, nicht aber den Ausdruck log file enthalten.
    database OR "data base" NOT "log file"
    Sucht nach Dokumenten, die entweder den Begriff database oder den Ausdruck data base, nicht aber den Ausdruck log file enthalten.
