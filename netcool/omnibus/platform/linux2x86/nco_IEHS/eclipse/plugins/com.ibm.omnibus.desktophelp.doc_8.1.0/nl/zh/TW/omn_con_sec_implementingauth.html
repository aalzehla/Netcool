使用群組及角色實作授權

許可權會控制對 ObjectServer 中物件及資料的存取。 透過將一項以上的許可權結合至角色，您可以快速且有效地管理存取。

每個使用者都會指派給一或多個群組。 您可以透過將一個以上角色授與該群組，來為每一個群組指派許可權，以對資料庫物件執行動作。您可以建立邏輯分組 (例如超級使用者或系統管理者)、實體分組 (例如倫敦或紐約 NOC) 或任何其他分組，以簡化安全設定。

例如，建立自動化需要瞭解 Tivoli Netcool/OMNIbus 作業及配置特定 ObjectServer 的方法。 通常，您不想讓所有使用者都建立或修改自動化。 一個解決方案是建立名為 AutoAdmin 的角色，其具有建立及修改觸發程式群組、檔案、SQL 與外部程序及信號的許可權。 然後，您可將該角色授與要建立及更新自動化的管理者群組。

security.sql Script 包含適用於不同類別使用者 (例如操作員及管理者) 的預設群組及角色。 您也可以使用此 Script 作為建立自己的群組及角色的範本。

使用 Netcool/OMNIbus Administrator 或 ObjectServer SQL 可以配置使用者、群組及角色。
