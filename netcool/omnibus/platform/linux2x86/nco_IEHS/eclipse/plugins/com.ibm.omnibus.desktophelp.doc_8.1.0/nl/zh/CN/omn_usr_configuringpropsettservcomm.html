IBM Tivoli Netcool/OMNIbus V7.4

配置属性、服务器通信和设置

可以从 UNIX 和 Windows Conductor 来配置 Tivoli Netcool/OMNIbus 组件的属性以及服务器组件的通信信息。在 Windows 上，可以在设置窗口内的组合任务中执行这两个功能。在 UNIX 上，还可以更改事件列表内的日期格式。

  • 配置属性
    您可以编辑以下各项的属性文件中所含的信息：ObjectServer、代理服务器、数据库初始化实用程序（nco_dbinit）、Confpack 实用程序（nco_confpack）、Netcool/OMNIbus Administrator（nco_config）、探测器和网关。属性文件具有 .props 扩展名。
  • 配置服务器通信信息
    必须维护 Tivoli Netcool/OMNIbus 服务器组件的服务器通信信息。 可以使用服务器编辑器来创建和修改服务器组件的通信详细信息，测试服务器活动，以及添加备份（或故障转移）服务器和侦听器。
  • 配置 Windows 上的 Tivoli Netcool/OMNIbus 设置
    可以在单个任务中配置 Windows 上的 Tivoli Netcool/OMNIbus 的属性和服务器通信信息。
  • 更改 UNIX 上的事件列表内的日期格式
    NCO_TIME 环境变量定义了事件列表内的日期格式。 缺省格式为 mm/dd/yy hh:mm:ss；例如，11/12/03 20:13:36。 POSIX strptime 功能确定了此格式。
