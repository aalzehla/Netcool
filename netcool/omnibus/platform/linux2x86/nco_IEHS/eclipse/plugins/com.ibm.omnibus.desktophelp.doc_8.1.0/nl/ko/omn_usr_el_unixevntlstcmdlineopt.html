IBM Tivoli Netcool/OMNIbus, 버전 7.4

UNIX 이벤트 목록 명령행 옵션

UNIX에서 nco_event 명령을 사용하여 이벤트 목록을 시작하는 경우, 다수의 명령행 옵션을 지정하여 구성을 수정할 수 있습니다. $NCHOME/omnibus/bin 디렉토리에서 이 명령을 실행할 수 있습니다.

다음 표에서는 UNIX 이벤트 목록을 시작하기 위한 명령행 옵션을 설명합니다.

표 1. UNIX 이벤트 목록(nco_event) 명령행 옵션
명령행 옵션 설명
-cmap 개인용 색상 맵을 지정합니다. 이 명령행 옵션은 줄일 수 없습니다.
-config string | URL

또는

-elc string | URL

 사용할 초기 구성 파일을 지정합니다. 여기서,

string은 .elc 파일의 경로를 지정합니다.

URL은 HTTP 또는 FTP를 사용하여 액세스할 수 있는 원격 서버에 있는 .elc 파일을 지정합니다. 다음 형식 중 하나를 사용하여 URL을 지정하십시오.

  • http://servername[:port]/path/filename.elc
  • http://username:password@servername[:port]/path/filename.elc
  • ftp://servername[:port]/path/filename.elc
  • ftp://username:password@servername[:port]/path/filename.elc

서버가 기본 포트를 사용하고 있지 않은 경우에만 선택적 포트 번호(:port)를 지정해야 합니다.

servername 값에 대해 IPv4 또는 IPv6 주소를 선택하여 지정할 수 있습니다. IPv6 주소를 지정하면 주소가 대괄호([]) 안에 묶입니다. 예제는 다음과 같습니다.

http://[2003:13f8:206:190:204:baef:fe6a:6af0]:2880/mydir/myfile.elc
-diagnostic 확장된 진단 메시지를 표시합니다.

이 옵션은 이벤트 목록 문제점을 발견하는 데에만 유용합니다. IBM Software Support에서 권고하지 않으면 이 명령행 옵션을 사용하지 마십시오.
-dualwrite 0 | 1 데스크탑 ObjectServer 아키텍처에서 작동할 때 이중 쓰기 모드를 사용할지 또는 사용하지 않을지 여부를 표시합니다. 올바른 옵션은 다음과 같습니다.
  • 0: 이중 쓰기 모드 사용 안함
  • 1: 이중 쓰기 모드 사용

-dualwrite 명령행 옵션은 데스크탑 ObjectServer의 master.national 테이블에 있는 DualWrite 필드 항목을 겹쳐씁니다.

-failbackpolltime integer 장애 조치로 인하여 백업 ObjectServer에 연결된 경우, 이벤트 목록은 주기적으로 기본 ObjectServer에 재연결을 시도합니다. 이 명령행 옵션은 이벤트 목록이 기본 ObjectServer의 리턴에 대해 폴링할 때의 빈도(초)를 지정합니다. 기본 ObjectServer가 다시 시작된 것이 발견된 경우, 이벤트 목록은 백업 ObjectServer로부터 연결을 끊고 기본 ObjectServer에 다시 연결합니다. 기본값은 30초입니다. 0으로 설정된 경우, 폴링이 발생하지 않습니다.

이벤트 목록이 ObjectServer에 연결된 경우, 이벤트 목록은 연결 중인 ObjectServer의 BackupObjectServer 특정 설정을 확인합니다. 폴링은 특성이 백업 ObjectServer를 나타내는 TRUE로 설정된 경우에만 발생합니다.
-help 이 명령에 대한 도움말을 표시하고 종료합니다.
-masterserver string 데스크탑 ObjectServer 아키텍처에서 사용할 마스터 ObjectServer를 표시합니다.

-masterserver 명령행 옵션은 데스크탑 ObjectServer의 master.national 테이블에서 MasterServer 필드 항목을 겹쳐씁니다.

-networktimeout integer 네트워크 장애가 발생할 경우 ObjectServer와의 연결 또는 로그인 시도가 제한시간 초과되는 시간(초)을 지정합니다. 지정된 제한시간 기간이 지난 후 이벤트 목록은 ObjectServer에 다시 연결하려고 시도합니다. 두 번째 제한시간 기간 동안 연결에 실패하면, 이벤트 목록이 백업 ObjectServer(사용 가능한 경우)에 연결하려고 시도합니다. 기본값은 20초입니다.
-password string 이벤트 목록에 로그인하는 데 사용할 비밀번호를 지정합니다.
-server string 연결 중인 ObjectServer 이름을 지정합니다.
-uneditable 이벤트 목록 구성 편집을 허용하지 않습니다. 이 옵션을 사용하는 경우, 필터 빌더 및 보기 빌더 단추와 저장 및 다른 이름으로 저장 메뉴 항목이 표시됩니다.
-username string 이벤트 목록에 로그인하기 위해 사용할 사용자 이름을 지정합니다.
-version 버전 정보를 표시하고 종료됩니다.
팁: 이벤트 목록에 대한 명령행 옵션을 가장 짧은 고유 약어로 줄일 수 있습니다. 예를 들어, -uneditable을 -un으로 줄일 수 있습니다.
상위 주제: 이벤트 목록 시작
관련 개념:

관련 태스크:
