IBM Tivoli Netcool/OMNIbus 7.4 版

伺服器編輯器

當您在 Tivoli Netcool/OMNIbus 系統中的任何主機上安裝或變更伺服器元件時，必須使用「伺服器編輯器」配置元件，使其與其他元件進行通訊。 使用此視窗，可以建立及修改伺服器元件的通訊明細、測試伺服器活動，以及新增備份 (或失效接手) 伺服器與接聽器。

伺服器元件包括 ObjectServer、閘道、程序控制代理程式及 Proxy 伺服器。 此功能僅可用於系統管理者及超級使用者。

您必須維護主機工作站上及每個需要連接至伺服器元件之工作站上的伺服器通訊資訊。 在配置通訊設定之後，「伺服器編輯器」會在連線資料檔中儲存結果。此檔案的名稱及位置如下所示：
UNIX 系統 $NCHOME/etc/omni.dat
Windows 系統 %NCHOME%\ini\sql.ini
如下所示完成此視窗：
伺服器清單
伺服器清單會顯示現有的伺服器元件及其設定：
伺服器
此直欄會顯示對此工作站定義的每個伺服器名稱。 預設伺服器名稱為：
  • NCOMS：ObjectServer 的預設名稱
  • NCO_PROXY：Proxy 伺服器的預設名稱
  • NCO_GATE：閘道的預設名稱
  • NCO_PA：程序代理程式的預設名稱
在 Windows 上，伺服器的主機工作站必須具有接聽器項目，且主機工作站及連接該伺服器的任何其他工作站必須具有用戶端項目。
備份 ObjectServer 顯示在「伺服器編輯器」中主要 ObjectServer 的下方。 若要隱藏備份 ObjectServer 的清單，請按兩下主要 ObjectServer 名稱或伺服器圖示。 系統會隱藏備份 ObjectServer，且字母 C 會顯示在伺服器圖示中。 再按兩下，以顯示備份 ObjectServer。 伺服器圖示會回到正常狀態。
主機名稱
此直欄會顯示安裝伺服器元件之工作站的主機名稱或 IP 位址。
註： 安裝之後，會針對所有伺服器將主機設為 omnihost（在 UNIX 上）。您必須將此項變更為伺服器主機工作站的名稱。 在 Windows 上，預設值為主機名稱。
埠
此直欄會顯示伺服器元件接聽未加密連線的埠。
SSL
在 UNIX 上，此直欄會顯示伺服器元件接聽已加密連線的埠。 在 UNIX 系統上，伺服器元件可以定義標準埠及/或 SSL 埠。
在 Windows 上，此直欄會針對已加密連線顯示單字是。 在 Windows 系統上，相同的埠號會同時用於已加密及未加密的連線。
伺服器登錄欄位
使用伺服器登錄欄位，可以輸入或編輯每個伺服器元件的詳細資料。 若您是在編輯現有伺服器元件的詳細資料，則您必須首先選取伺服器清單中的相關列，以便在伺服器登錄欄位中顯示詳細資料。
名稱
輸入新伺服器元件的名稱或編輯現有元件的名稱。 在 Windows 上，您也可以使用下拉清單來選取名稱。 在命名元件時，請使用下列字尾：_PROXY（用於 Proxy 伺服器）、_GATE（用於閘道）及 _PA（用於程序代理程式）。
註： 伺服器項目的名稱必須由 29 個以下大寫字母組成，且不得以整數開頭。
主機
輸入或編輯安裝伺服器元件之工作站的主機名稱或 IP 位址。 （對於 UNIX 上的新伺服器元件，名稱會預設為 omnihost，且必須變更為實際的主機名稱或 IP 位址。）
如果輸入 IP 位址，您就可以指定 IPv4 或 IPv6 位址。 例如：
  • 192.168.0.1
  • 2094:82a:2a6e:123:503:badd:fe43:f552
埠
在 UNIX 上，如果您要讓使用未加密連線的用戶端能夠連接至伺服器，請在此欄位中輸入有效的未用埠號。 若要禁止未加密連線，則不要設定埠。
在 Windows 上，在此欄位中輸入有效的未用埠號。
SSL
在 UNIX 上，如果您要讓使用已加密連線的用戶端能夠連接至伺服器，請在此欄位中輸入有效的未用埠號。 若要禁止已加密連線，則不要設定埠。
在 Windows 上，選取此勾選框，以指出埠接受使用 SSL 之用戶端的已加密連線。
註： 在 UNIX 系統上，伺服器元件可以定義標準埠及/或 SSL 埠。 在 Windows 系統上，相同的埠號可用於任意一種連線類型。
接聽器 (僅限 Windows)
若這是伺服器之主機工作站上的接聽器項目，請選取此勾選框。 若這是用戶端項目，請取消勾選接聽器勾選框。
新增
按一下此按鈕，以將新 (而且唯一的) 伺服器明細新增至伺服器清單。
移除/更新
此按鈕會根據您是否已針對目前選取的伺服器元件進行變更而改變。 如果要從伺服器清單中移除目前選取的伺服器元件，請按一下移除。 按一下更新，以使用現有伺服器元件的已更新明細重新整理伺服器清單。
測試
按一下此按鈕，以測試是否可以連接至伺服器清單中選取的伺服器。 「伺服器編輯器」會嘗試聯絡指定之主機及埠上的伺服器。 視窗會顯示測試指令的結果。
優先順序欄位
伺服器優先順序欄位可讓您提高或降低已配置為失效接手系統之伺服器元件的優先順序。 例如，假設使用名稱為 NCOMS_BAK 的備份配置名稱為 NCOMS_PRI 的 ObjectServer。 使用提高或降低按鈕，您可以將 NCOMS_BAK 的優先順序提高為主要 ObjectServer，而將 NCOMS_PRI 降低為備份。
提高
按一下此按鈕，以將所選伺服器元件的優先順序提高一個層次。
降低
按一下此按鈕，以將所選伺服器元件的優先順序降低一個層次。
產生全部 (僅限 UNIX)
按一下此按鈕，以針對所有 UNIX 作業系統產生介面檔。 當套用變更時 (使用套用按鈕)，這會產生名為 $NCHOME/etc/interfaces.arch 的介面檔，其中 arch 代表個別 UNIX 平台名稱；例如，interfaces.hpux11 及 interfaces.solaris2。
顯示群組 (僅限 Windows)
在 Windows 上，選取此勾選框，以在伺服器清單中依名稱對每個伺服器進行分組，在用戶端項目之後接著已識別的備用伺服器及接聽器。
套用 (僅限 UNIX)
按一下此按鈕，以將變更套用至介面檔。 任何已新增至伺服器清單、從伺服器清單中移除或已編輯的伺服器元件都會儲存到此檔案。
確定 (僅限 Windows)
按一下此按鈕，以儲存您的變更並關閉此視窗。
關閉 (UNIX)/取消 (Windows)
按一下此按鈕，以在使用套用按鈕儲存變更之後關閉此視窗，或關閉此視窗而不儲存變更。
匯入 (僅限 UNIX)
按一下此按鈕，以匯入通訊明細。
