Windows 事件列表命令行选项

在 Windows 上使用 NCOEvent.exe 命令启动事件列表时，可指定若干命令行选项来修改配置。可以从 %NCHOME%\omnibus\desktop 目录运行此命令。

下表描述了用于启动 Windows 事件列表的命令行选项。

表 1. Windows 事件列表（NCOEvent）命令行选项
命令行选项 描述
-config string | URL

或

-elc string | URL

 指定要使用的初始配置文件，其中：

string 指定 .elc 文件的路径。

URL 指定位于远程服务器上的 .elc 文件，该文件可使用 HTTP 或 FTP 进行访问。 通过使用以下某种格式指定 URL：

  • http://servername[:port]/path/filename.elc
  • http://username:password@servername[:port]/path/filename.elc
  • ftp://servername[:port]/path/filename.elc
  • ftp://username:password@servername[:port]/path/filename.elc

仅当服务器未使用缺省端口时才需要指定可选端口号 :port。

可以选择为 servername 值指定 IPv4 或 IPv6 地址。 指定 IPv6 地址时，使用方括号 [] 括起地址；例如：

http://[2003:13f8:206:190:204:baef:fe6a:6af0]:2880/mydir/myfile.elc

如果使用 FTP，那么路径必须是从根目录开始的完整路径，例如，/home/user/mydir/myconfig.elc。
-diagnostic 显示扩展诊断消息。

该选项仅用于查找事件列表中存在的问题。 除非 IBM 软件支持 建议你这样做，否则请勿使用此命令行选项。
-dualwrite 0 | 1 指示在桌面 ObjectServer 体系结构中操作时是启用还是禁用双重写方式。 有效选项如下所示：
  • 0：禁用双写入方式
  • 1：启用双重写方式

-dualwrite 命令行选项将覆盖桌面 ObjectServer master.national 表中的 DualWrite 字段项。
-failbackpolltime integer 如果因发生故障转移而连接至备份 ObjectServer，事件列表会定期尝试重新连接至主控 ObjectServer。该命令行选项指定了事件列表轮询以返回主控 ObjectServer 的频率（以秒为单位）。如果检测到主控 ObjectServer 已重新启动，那么事件列表将与备份 ObjectServer 断开连接，然后重新连接至主控 ObjectServer。缺省值为 30 秒。当该值设置为 0 时，不发生轮询。

当事件列表连接至 ObjectServer 时，该事件列表会检查其正在连接的 ObjectServer 的 BackupObjectServer 属性设置。仅当该属性设置为 TRUE（即指示备份 ObjectServer）时才发生轮询。
-help 显示该命令的帮助并退出
-masterserver string 指示桌面 ObjectServer 体系结构中要使用的主控 ObjectServer。

-masterserver 命令行选项将覆盖桌面 ObjectServer master.national 表中的 MasterServer 字段项。
-networktimeout integer 指定发生网络故障时，多少秒后登录尝试或连接 ObjectServer 将超时。 指定的超时时段后，事件列表将尝试重新连接 ObjectServer。 如果超时时段后的连接不成功，事件列表将尝试连接到备份 ObjectServer（如果可用）。 缺省值为 20 秒。
-password string 指定用于登录到事件列表的密码。
-server string 指定要连接的 ObjectServer 的名称。
-username string 指定用于登录事件列表的用户名。
-version 显示版本信息并退出。
