監視事件清單中的 TADDM 事件

如果在 IT 環境中偵測到配置變更，則會在 IBM Tivoli Application Dependency Discovery Manager 中產生 TADDM 事件。如果系統已配置為支援此功能，則您可以從 Tivoli Netcool/OMNIbus 事件清單監視 TADDM 事件。

如果已進行此配置，則會在事件清單中提供 TADDM 過濾器，用於過濾 TADDM 事件。 如果現行視圖中具有類別直欄，則您還可以透過對其排序來識別事件清單中的 TADDM 事件。依預設，此項目為 Tivoli Application Dependency Discovery Manager，但系統管理者可以變更此值。

依預設，TADDM 事件的嚴重性層次為「不確定」。

從事件清單中， 您可以啟動至 TADDM Java 主控台或 Web 用戶端， 以便檢視已產生警示的配置項目進一步相關資訊：

  • 若要從 TADDM Java 主控台中檢視配置項目的所有屬性詳細資料， 請用滑鼠右鍵按一下事件清單中的 TADDM 事件， 然後從蹦現功能表中按一下 TADDM > 配置項目詳細資料（主控台）。
  • 若要從 TADDM Web 用戶端中檢視配置項目的所有屬性詳細資料，請用滑鼠右鍵按一下事件清單中的 TADDM 事件，然後從蹦現功能表中按一下 TADDM > 配置項目詳細資料 (web)。
  • 若要從 TADDM Java 主控台中檢視配置項目的變更歷程報告，請用滑鼠右鍵按一下事件清單中的 TADDM 事件，然後從蹦現功能表中按一下 TADDM > 變更歷程（主控台）。
  • 若要從 TADDM Web 用戶端中檢視配置項目的變更歷程報告，請用滑鼠右鍵按一下事件清單中的 TADDM 事件，然後從蹦現功能表中按一下 TADDM > 變更歷程 (web)。

    您可以按兩下某個事件，以在「事件資訊」視窗中檢視其完整詳細資料。
