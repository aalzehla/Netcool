IBM Tivoli Netcool/OMNIbus 7.4 版

IPv6 配置

Tivoli Netcool/OMNIbus 提供 IPv4 及 IPv6 的支援。元件可以操作並同時存在於僅支援 IPv4、僅支援 IPv6 或支援雙重 IPv4 及 IPv6 配置的網路上。

Tivoli Netcool/OMNIbus 伺服器元件在 IPv6 及 IPv4 環境中會按如下所述操作：
  • ObjectServer 可以同時處理來自 IPv4 及 IPv6 網路的事件。 ObjectServer 在雙重堆疊的主機上執行時，傳回用戶端以回應指令的主機名稱，就是對應於用戶端正在執行之 IP 版本的伺服器主機名稱。 例如，執行 IPv4 的用戶端會接收到 ObjectServer 的 IPv4 主機名稱，而執行 IPv6 的用戶端會接收到 ObjectServer 的 IPv6 主機名稱。
  • 在雙重 IPv4 及 IPv6 環境中，單向及雙向的 ObjectServer 閘道都可以在通訊端上的兩個介面上接聽。
  • Proxy 伺服器可支援探測器與在 IPv4 和 IPv6 主機上執行的 ObjectServer 之間的連線。
  • 在雙重 IPv4 及 IPv6 環境中，程序代理程式可以在通訊端上的兩個介面上接聽。
可支援下列 IPv6 位址格式：
  • 八組四個十六進位字元 (以冒號分隔)；例如，ABCD:EF01:2345:6789:ABCD:EF01:2345:6789
  • 若所有 16 個位元都是零，則可以用冒號 (:) 取代該區段。

    例如，位址 1010:0000:0000:0000:ABCD:EF01:2345:6789 可以撰寫成 1010::ABCD:EF01:2345:6789。

  • 可以用 IPv6 位址表示 IPv4 位址。 例如：
    • 0:0:0:0:0:0:192.101.50.5
    • 0:0:0:0:0:FFFF:103.27.35.8

      這些位址也可以表示為：

    • ::192.101.50.5
    • ::FFFF:103.27.35.8
