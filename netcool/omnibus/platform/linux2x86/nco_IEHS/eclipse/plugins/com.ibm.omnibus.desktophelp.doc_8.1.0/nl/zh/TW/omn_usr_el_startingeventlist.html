IBM Tivoli Netcool/OMNIbus 7.4 版

啟動事件清單

若要監視及管理事件清單中的警示，您必須透過使用有效的鑑別認證，連接到執行中的 ObjectServer。 您可以從「管理者」、從指令行或從 Windows 開始功能表啟動事件清單。

若要啟動事件清單：
  1. 針對作業系統執行適當的動作：
    選項 敘述
    UNIX
    • 從「管理者」視窗中，按一下按鈕列中的事件清單。
    • 從命令提示字元中，輸入：

      $NCHOME/omnibus/bin/nco_event &
    Windows
    • 從「管理者」功能表中，選取事件清單。
    • 從 Windows 開始功能表中，選取程式集 > Netcool 套組 > 事件清單。
    • 從命令提示字元中，輸入：

      %NCHOME%\omnibus\desktop\NCOEvent.exe
    從指令行啟動事件清單時，您可以使用 nco_event 及 NCOEvent.exe 指令，指定其他指令行選項。
  2. 如果您從指令行或 Windows 開始功能表啟動事件清單，且先前未啟動「管理者」，則會提示您提供鑑別詳細資料，以連接到執行中的 ObjectServer。您必須完成事件清單的「登入」視窗，如下所示：
    使用者名稱
    輸入您的使用者名稱。
    密碼
    輸入您的密碼。 會在您輸入文字的位置顯示星號 (*) 字元。
    註： 如果是針對外部鑑別來源（而不是 ObjectServer）驗證您的使用者名稱及密碼，請檢查此來源是否支援多位元組字元。如果不支援多位元組字元，您必須使用 ASCII 字元指定使用者名稱及密碼。
    伺服器
    ObjectServer 名稱預設為 NCOMS（Netcool/OMNIbus 主要伺服器）。 如果已針對您的環境配置多個 ObjectServer，則所有可用的 ObjectServer 都會顯示在此下拉清單中。選取您要連接至的 ObjectServer。
    確定
    按一下此按鈕，以鑑別您的登入詳細資料。
    註： 允許您嘗試登入三次。 每次嘗試時，都會清除密碼欄位。 如果您在第三次嘗試中仍無法正確登入，程式即會終止。
    取消
    按一下此按鈕，以取消登入嘗試。

結果

如果事件清單 (nco_event) 的實例正在執行中，則 nco_event 指令會開啟新的事件清單視窗。 如果沒有執行中的 nco_event 實例，則 nco_event 指令會啟動一個實例，並顯示監視器框視窗。

當您啟動事件清單時，會開啟「事件清單」監視器框視窗。 從此視窗中，您可以在 ObjectServer 接收事件、警示、訊息或資料項目時，監視及管理所有警示。

一般而言，在您啟動事件清單時，預設事件清單配置 (default.elc) 會顯示在「事件清單」監視器框視窗中。 您還可以選擇開啟自訂的事件清單配置檔案。 可以在指令行中或使用檔案 > 開啟視窗指定自訂的配置。 這些自訂的配置可位於檔案系統上，或位於可使用 HTTP 或 FTP 存取的遠端伺服器上。

失效接手登入： 如果事件清單遺失與主要 ObjectServer 的連線，且存在次要或失效接手 ObjectServer，則會提示您登入次要 ObjectServer。 在主要 ObjectServer 重新可用之後，會顯示一則訊息，指出事件清單正在重新連接到主要 ObjectServer。 您無需重新登入主要 ObjectServer。
  • UNIX 事件清單指令行選項
    當您使用 nco_event 指令，在 UNIX 上啟動事件清單時，可以指定一些指令行選項來修改配置。您可以從 $NCHOME/omnibus/bin 目錄執行此指令。
  • Windows 事件清單指令行選項
    當您使用 NCOEvent.exe 指令，在 Windows 上啟動事件清單時，可以指定一些指令行選項來修改配置。您可以從 %NCHOME%\omnibus\desktop 目錄執行此指令。
