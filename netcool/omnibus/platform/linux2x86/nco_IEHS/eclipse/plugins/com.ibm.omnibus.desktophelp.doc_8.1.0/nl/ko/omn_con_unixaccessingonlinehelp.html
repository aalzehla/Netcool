IBM Tivoli Netcool/OMNIbus, 버전 7.4

UNIX의 온라인 도움말 액세스

온라인 도움말은 웹 애플리케이션인 IEHS(IBM® Eclipse Help System)를 사용하여 배치됩니다. 설치 및 구성 설정에 따라 독립형 모드 또는 정보 센터 모드 중 하나에서 온라인 도움말에 액세스할 수 있습니다.

온라인 도움말을 구성하고 액세스하려면 다음 표의 지시를 따르십시오.

표 1. 도움말 구성 설정
설정 설명
브라우저에 대한 환경 변수(UNIX) 브라우저의 디렉토리 위치를 다음 환경 변수에 추가하십시오(아직 추가되지 않은 경우).
  • PATH
  • LD_LIBRARY_PATH(Solaris 및 Linux), LIBPATH(AIX®) 또는 SHLIB_PATH(HP-UX)
  • 브라우저 특정의 환경 변수(예: MOZILLA_FIVE_HOME)
독립형 모드 독립형 모드에서 온라인 도움말을 사용하려면 Tivoli Netcool/OMNIbus 설치 중에 로컬 도움말 시스템 기능을 선택해야 합니다.
설치 후에는 기본 IEHS 포트 번호 8888이 다른 로컬 서비스에 사용되고 있는 경우에만 구성이 필요합니다. 이 경우 IEHS 구성 파일 $NCHOME/omnibus/etc/nco_IEHS.cfg를 다음과 같이 편집하십시오.
  • IEHSMode: 0
  • IEHSHost: 공백
  • IEHSPort: 사용하지 않은 포트 번호
참고: 구성 파일에서 포트 번호를 변경한 후 변경사항을 실행하기 위해서는 로컬 IEHS 서버를 종료해야 합니다. $NCHOME/omnibus/bin/help_end 명령을 실행하십시오. (브라우저에 대한 환경 변수 설정을 변경하는 경우에도 변경사항을 실행하기 위해 IEHS 서버를 종료해야 합니다.)
정보 센터 모드 정보 센터 모드에서 시스템 관리자는 모든 사용자가 온라인 도움말 시스템에 대해 하나의 IEHS 웹 서버를 공유함으로써 로컬 호스트 또는 워크스테이션의 로드를 줄일 수 있도록 설정을 구성합니다.

시스템 관리자가 정보 센터 모드에서 온라인 도움말을 액세스해야 한다고 알려준 경우 로컬 IEHS 구성 파일 $NCHOME/omnibus/etc/nco_IEHS.cfg를 다음과 같이 구성해야 합니다.

  • IEHSMode: 1
  • IEHSHost: IEHS 서버의 IP 주소 또는 호스트 이름
  • IEHSPort: IEHS 서버가 실행 중인 포트 번호

IEHSHost 및 IEHSPort 설정은 시스템 관리자가 제공합니다.

독립형 모드에서 온라인 도움말에 액세스하면 기본 도움말 브라우저 창이 관련 텍스트와 함께 자동으로 열립니다. 로컬 IEHS 서버는 처음 도움말이 요청되었을 때부터 서버가 종료될 때까지 계속해서 실행됩니다. 로컬 IEHS 서버를 종료하려면 다음 명령을 실행하십시오.

$NCHOME/omnibus/bin/help_end

일반적으로 help_end 명령을 실행한 후에는 반드시 컴퓨터를 재부팅하십시오.

정보 센터 모드에서 온라인 도움말에 액세스하기 전에 먼저 시스템 관리자가 지정된 원격 IEHS 서버를 시작해야 합니다.

참고: 독립형 모드의 경우, 로컬 컴퓨터에서 실행 중인 브라우저만 도움말 항목을 표시할 수 있습니다. 다른 호스트에서 실행 중인 브라우저가 열려 있는 경우, 도움말 시스템은 해당 브라우저에 온라인 도움말을 표시하려고 시도합니다. 그 결과 다음 오류 메시지가 나타납니다.
로컬 호스트:이름에 연결을 시도하는 중에 연결이 거부되었습니다.
올바른 브라우저에 도움말을 표시하려면 이 창을 닫으십시오.
