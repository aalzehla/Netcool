监视和管理事件列表中的事件

您可使用事件列表来监视和管理警报。 ObjectServer 收到事件、警报、消息或数据项时，将创建警报。 每个警报由 ObjectServer alerts.status 表的行中的信息列（或字段）组成。

可使用过滤器和视图来确定事件列表中显示的警报信息。 过滤器使您能够根据特定条件显示警报子集。 视图使您能够选择要显示的警报字段。

  • 启动事件列表
    要监视和管理事件列表内的警报，您必须使用有效认证凭证连接到正在运行的 ObjectServer。 您可以从 Conductor、命令行或 Windows 开始菜单启动事件列表。
  • 使用事件列表来管理事件
    使用事件列表来管理事件。
  • 对事件列表执行统一操作
    在“事件列表”监视器框窗口上，每个监视器框代表一个事件列表，这些列表包含与过滤器和视图的特定配置相匹配的事件。 可以单独访问每个事件列表并管理事件。 也可以对所有事件列表执行一些统一的操作。
  • 处理事件列表配置
    可以保存为“事件列表”监视器框窗口上的所有监视器框定义的过滤器、视图以及其他事件配置。 该信息保存在事件列表配置（.elc）文件中。
  • 事件列表中的因特网服务监视器
    IBM Tivoli Composite Application Manager for Internet Service Monitoring 与其余 Tivoli Netcool/OMNIbus 产品套件配合使用。 监视器和探测器生成的事件可发送到同一个 ObjectServer，并且您可使用同一个桌面处理这些事件。
  • 监视事件列表中的影响服务事件
    影响服务事件（SAE）是一种警报，它警告操作员一个关键客户服务已受到一个或多个网络事件的影响。影响服务事件在 IBM Tivoli Network Manager IP Edition 内生成。
  • 监视事件列表中的预测事件
    预测事件表示对即将发生的阈值违例的预测。这些事件不同于转发给 Tivoli Netcool/OMNIbus 的其他事件，因为它们表示尚未发生但很可能在近期发生的情况、故障或状态。
  • 监视事件列表中的 TADDM 事件
    在您的 IT 环境中检测到配置更改时，将在 IBM Tivoli Application Dependency Discovery Manager 内生成 TADDM 事件。如果您的系统已配置为支持该功能，那么可以通过 Tivoli Netcool/OMNIbus 事件列表监视 TADDM 事件。
  • 从事件列表中找出根本原因事件
    在虚拟化环境中，Tivoli Netcool/OMNIbus 可以关联事件来确定哪些事件是其他症状事件的根本原因事件。如果您的系统上启用了此功能，您可以在事件列表中找出症状事件后的根本原因事件。
  • 从命令行、脚本和工具运行瞬时事件列表
    您可直接从命令行、脚本或部分事件列表工具运行定制的瞬时事件列表。
