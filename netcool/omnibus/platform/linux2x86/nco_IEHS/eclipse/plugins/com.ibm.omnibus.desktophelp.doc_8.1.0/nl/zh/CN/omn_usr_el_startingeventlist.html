IBM Tivoli Netcool/OMNIbus V7.4

启动事件列表

要监视和管理事件列表内的警报，您必须使用有效认证凭证连接到正在运行的 ObjectServer。 您可以从 Conductor、命令行或 Windows 开始菜单启动事件列表。

要启动事件列表：
  1. 为操作系统执行相应的操作：
    选项 描述
    UNIX
    • 从 Conductor 窗口单击按钮栏内的事件列表。
    • 从命令提示符输入：

      $NCHOME/omnibus/bin/nco_event &
    Windows
    • 从 Conductor 菜单选择事件列表。
    • 从 Windows 开始菜单选择程序 > Netcool Suite > 事件列表。
    • 从命令提示符输入：

      %NCHOME%\omnibus\desktop\NCOEvent.exe
    从命令行启动事件列表时，可以使用 nco_event 和 NCOEvent.exe 命令来指定其他命令行选项。
  2. 如果从命令行或 Windows 开始菜单启动事件列表，并且之前未启动 Conductor，系统将提示您提供认证详细信息以连接到正在运行的 ObjectServer。必须按以下方法完成事件列表的“登录”窗口：
    用户名
    请输入用户名。
    密码
    请输入密码。 在您输入文本之处显示星号（*）。
    注： 如果是依据外部认证源（而不是 ObjectServer）验证用户名和密码，请检查该源是否支持多字节字符。 如果不支持多字节字符，那么您必须使用 ASCII 字符指定用户名和密码。
    服务器
    ObjectServer 名称缺省设置为 NCOMS（Netcool/OMNIbus 主控服务器）。 如果您的环境配置了多个 ObjectServer，那么所有可用的 ObjectServer 将显示在此下拉列表中。 选择想要连接到的 ObjectServer。
    确定
    单击此按钮可认证您的登录详细信息。
    注： 允许您进行三次登录尝试。 每次尝试时，都会清空密码字段。 如果在第三次尝试时您仍然未能正确登录，那么程序将终止。
    取消
    单击此按钮可取消您的登录尝试。

结果

如果事件列表的实例（nco_event）正在运行，那么 nco_event 命令将打开新事件列表窗口。 如果没有 nco_event 的实例正在运行，那么 nco_event 命令将启动一个实例，然后显示监视器框窗口。

启动事件列表时，将打开“事件列表”监视器框窗口。 从此窗口可以监视和管理在 ObjectServer 接收事件、警报、消息或数据项时生成的所有警报。

通常，当启动事件列表时，会在“事件列表”监视器框窗口内显示缺省事件列表配置（default.elc）。 也可以选择打开定制的事件列表配置文件。 可在命令行内或通过使用文件 > 打开窗口来指定定制的配置。 这些定制的配置可能在您的文件系统上，或位于可使用 HTTP 或 FTP 访问的远程服务器上。

故障转移登录： 如果事件列表丢失了到主控 ObjectServer 的连接，并且存在辅助或故障转移 ObjectServer，那么将提示您登录到辅助 ObjectServer。 当主控 ObjectServer 再次变为可用时，将出现指示事件列表正在重新连接到主控 ObjectServer 的消息。 不需要再次登录到主控 ObjectServer。
  • UNIX 事件列表命令行选项
    在 UNIX 上使用 nco_eve nt 命令启动事件列表时，可指定若干命令行选项来修改配置。您可从 $NCHOME/omnibus/bin 目录运行此命令。
  • Windows 事件列表命令行选项
    在 Windows 上使用 NCOEvent.exe 命令启动事件列表时，可指定若干命令行选项来修改配置。可以从 %NCHOME%\omnibus\desktop 目录运行此命令。
