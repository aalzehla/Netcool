IBM Tivoli Netcool/OMNIbus, 버전 7.4

이벤트 목록에서 TADDM 이벤트 모니터링

TADDM 이벤트는 IT 환경에서 구성 변경이 발견될 때 IBM Tivoli Application Dependency Discovery Manager 내에 생성됩니다. Tivoli Netcool/OMNIbus 이벤트 목록에서 TADDM 이벤트를 모니터할 수 있습니다(이 기능을 지원하도록 시스템이 구성된 경우).

구성된 경우, 이벤트 목록에서 TADDM 필터를 사용하여 TADDM 이벤트를 필터링할 수 있습니다. 또한 현재 보기에서 클래스 컬럼을 사용할 수 있는 경우 이 컬럼을 정렬하여 이벤트 목록에서 TADDM 이벤트를 식별할 수 있습니다. 기본적으로 해당 항목은 Tivoli Application Dependency Discovery Manager이지만 이는 시스템 관리자가 이 값을 변경할 수 있습니다.

TADDM 이벤트의 기본 심각도 레벨은 확인할 수 없음입니다.

이벤트 목록에서 TADDM Java™ 콘솔 또는 웹 클라이언트를 실행하여 경보가 발생한 구성 항목에 대한 추가 세부사항을 볼 수 있습니다.

  • TADDM Java 콘솔에서 구성 항목에 대한 모든 속성 세부사항을 보려면, 이벤트 목록에서 TADDM 이벤트를 마우스 오른쪽 단추로 클릭한 후 팝업 메뉴에서 TADDM > 구성 항목 세부사항(콘솔)을 클릭하십시오.
  • TADDM 웹 클라이언트에서 구성 항목에 대한 모든 속성 세부사항을 보려면, 이벤트 목록에서 TADDM 이벤트를 마우스 오른쪽 단추로 클릭한 후 팝업 메뉴에서 TADDM > 구성 항목 세부사항(웹)을 클릭하십시오.
  • TADDM Java 콘솔에서 구성 항목에 대한 변경 히스토리 보고서를 보려면, 이벤트 목록에서 TADDM 이벤트를 마우스 오른쪽 단추로 클릭한 후 팝업 메뉴에서 TADDM > 변경 히스토리(콘솔)를 클릭하십시오.
  • TADDM 웹 클라이언트에서 구성 항목에 대한 변경 히스토리 보고서를 보려면, 이벤트 목록에서 TADDM 이벤트를 마우스 오른쪽 단추로 클릭한 후 팝업 메뉴에서 TADDM > 변경 히스토리(웹)를 클릭하십시오.

    이벤트를 두 번 클릭하면 이벤트 정보 창에서 전체 세부사항을 볼 수 있습니다.
