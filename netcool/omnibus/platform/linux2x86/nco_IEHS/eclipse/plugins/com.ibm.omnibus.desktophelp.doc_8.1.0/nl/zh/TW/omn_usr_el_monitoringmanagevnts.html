監視及管理事件清單中的事件

您可以使用事件清單來監視及管理警示。 在 ObjectServer 接收事件、警示、訊息或資料項目時，會建立警示。 每個警示都是由 ObjectServer alerts.status 表格的列中保留之資訊的直欄 (或欄位) 所組成。

顯示在事件清單中的警示資訊是以過濾器及視圖的使用來判定。 過濾器可讓您根據特定的準則顯示警示子集。 視圖可讓您選擇要顯示的警示欄位。

  • 啟動事件清單
    若要監視及管理事件清單中的警示，您必須透過使用有效的鑑別認證，連接到執行中的 ObjectServer。 您可以從「管理者」、從指令行或從 Windows 開始功能表啟動事件清單。
  • 使用事件清單管理事件
    使用事件清單可以管理事件。
  • 對事件清單執行群體動作
    在「事件清單」監視器框視窗中，每個監視器框都代表包含符合特定過濾器及視圖配置之事件的事件清單。 您可以分別存取每個事件清單並管理事件。 您也可以對所有事件清單執行一些群體動作。
  • 使用事件清單配置
    您可以儲存針對「事件清單」監視器框視窗上所有監視器框定義的過濾器、視圖及其他事件配置。 此資訊儲存在事件清單配置檔案 (.elc) 中。
  • 事件清單中的網際網路服務監視器
    IBM Tivoli Composite Application Manager for Internet Service Monitoring 可與其餘 Tivoli Netcool/OMNIbus 產品組合一起使用。 監視器及探測器產生的事件都可以轉遞到相同的 ObjectServer，且您可以使用相同的桌面操作這些事件。
  • 監視事件清單中影響服務的事件
    影響服務的事件 (SAE) 是一則警示，警告操作員已有一個以上網路事件影響到重要客戶服務。 影響服務的事件是在 IBM Tivoli Network Manager IP Edition 內產生。
  • 監視事件清單中的預測事件
    預測事件代表對即將發生的臨界值違規的預測。這些事件不同於轉遞至 Tivoli Netcool/OMNIbus 的其他事件，因為它們代表尚未發生的事件、錯誤或狀況，但在近期很有可能會發生。
  • 監視事件清單中的 TADDM 事件
    如果在 IT 環境中偵測到配置變更，則會在 IBM Tivoli Application Dependency Discovery Manager 中產生 TADDM 事件。如果系統已配置為支援此功能，則您可以從 Tivoli Netcool/OMNIbus 事件清單監視 TADDM 事件。
  • 識別事件清單中的主要原因事件
    在虛擬化環境中，Tivoli Netcool/OMNIbus 可以使事件產生關聯，以判定哪些事件是其他症狀事件的主要原因。在清單事件中，您可以識別症狀事件背後的主要原因事件，如果您的系統上已啟用此功能的話。
  • 從指令行、Script 及工具執行暫時性事件清單
    您可以直接從指令行、在 Script 中或作為事件清單工具的一部分執行自訂的暫時性事件清單。
