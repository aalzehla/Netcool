IBM Tivoli Netcool/OMNIbus V7.4

在 Windows 上访问联机帮助

联机帮助是使用 IBM® Eclipse 帮助系统（IEHS）（一种 Web 应用程序）进行部署的。根据您的安装和配置设置，您能以单机或信息中心方式访问联机帮助。

要配置和访问联机帮助，请以下表所描述的方式继续操作。

表 1. 帮助配置设置
设置 描述
单机方式 要以单机方式使用联机帮助，您必须在 Tivoli Netcool/OMNIbus 安装期间已选择了本地帮助系统功能部件。
在安装后，只有在其他本地服务正在使用缺省 IEHS 端口号 8888 时才有必要进行配置。 如果正是这种情况，请以如下方式编辑 IEHS 配置文件 %NCHOME%\omnibus\ini\nco_IEHS.cfg：
  • IEHSMode：0
  • IEHSHost：空白
  • IEHSPort：未用端口号
注： 在更改配置文件中的端口号后，您必须关闭本地 IEHS 服务器以使您的更改生效。 运行 %NCHOME%\omnibus\bin\help_end.bat 命令。
信息中心方式 信息中心方式中，您的系统管理员会配置此设置，以便所有的用户可以共享联机帮助系统的同一个 IEHS Web 服务器，这样可减轻本地主机或工作站上的负载。

如果系统管理员通知您必须以信息中心方式访问联机帮助，那么您必须按下面的描述配置本地 IEHS 配置文件 %NCHOME%\omnibus\ini\nco_IEHS.cfg：

  • IEHSMode：1
  • IEHSHost：IEHS 服务器的 IP 地址或主机名
  • IEHSPort：运行 IEHS 服务器所用的端口号

系统管理员将提供 IEHSHost 和 IEHSPort 设置。

当您以单机方式访问联机帮助时，缺省帮助浏览器窗口会自动打开，并带有相关文本。 本地 IEHS 服务器将开始处理第一个帮助请求，并一直运行到您将其关闭为止。 要关闭本地 IEHS 服务器，请运行以下命令：

%NCHOME%\omnibus\bin\help_end.bat

通常，仅在重新引导计算机之前才运行 help_end.bat 命令。

在您能够以信息中心方式访问联机帮助之前，您的系统管理员必须启动指定的远程 IEHS 服务器。

注： 在 Windows 上，如果您重新引导系统或者停止了 IEHS 的 javaw 进程（使用任务管理器）而没有首先关闭 IEHS 服务器，那么将不会除去 IEHS 服务器生成的某些临时文件。这可能会导致访问联机帮助时遇到困难。 在单机方式和信息中心方式中，您都必须先运行 help_end.bat 或 IC_end.bat 关闭 IEHS 服务器，然后 才能重新引导系统，且不得手动停止 IEHS javaw 进程。 如果您无法以单机方式访问帮助，或者无法以信息中心方式启动 IEHS 服务器，那么请运行 help_end.bat 或 IC_end.bat 来清除临时文件，然后重试。
