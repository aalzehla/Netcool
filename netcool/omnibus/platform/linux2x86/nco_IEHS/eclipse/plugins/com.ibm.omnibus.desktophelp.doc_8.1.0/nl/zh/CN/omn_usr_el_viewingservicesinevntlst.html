查看事件列表内的服务

如果安装了 IBM Tivoli Composite Application Manager for Internet Service Monitoring，那么您就可以查看和处理来自因特网服务监视器的服务警报。

要查看服务信息，请执行以下任意一项操作：
  • 从 Conductor 打开事件列表。 当安装 Composite Application Manager for Internet Service Monitoring 时，只要从Conductor 访问监视器框窗口就会自动打开“服务”窗口。
  • 从“事件列表”监视器框窗口单击窗口 > 服务。
  • 从事件列表单击工具栏中的显示服务窗口。

结果

此时将打开“服务”窗口。

“服务”窗口中的每一行表示带有以下信息的不同服务：
带颜色的符号
每行左边的带颜色符号表示 ObjectServer 从该服务接收到的上一个警报的状态。 符号按照颜色编码，规则如下：
  • 红色：服务级别协议未满足。
  • 琥珀色：服务存在问题。
  • 绿色：服务没有问题。
  • 黑色：服务状态未知。
服务
此列显示正在监视的服务的名称。
上次更改时间
此列显示服务状态上次更改的日期和时间。
上次正常工作时间
此列显示服务上次状态良好的日期和时间。
上次故障时间
此列显示服务上次出现故障的日期和时间。
上次边缘状态时间
此列显示服务上次处于紧急状态的日期和时间。
上次报告时间
此列显示上次生成服务报告的的日期和时间。

双击任何行都可打开包含该服务类型警报列表的事件列表窗口。 您可按照与从探测器接收到的警报相同的方式来处理事件列表中的服务警报。

单击关闭可关闭服务窗口。
