IBM Tivoli Netcool/OMNIbus, 버전 7.4

이벤트 목록 데이터에 대한 필터 설정

필터는 모니터 상자 및 이벤트 목록에 사용자 정의 경보 정보를 표시하기 위해 ObjectServer alerts 데이터베이스 테이블에서 실행할 수 있는 SQL 조회를 빌드하는 데 사용됩니다.

필터는 다음과 같은 이벤트 목록에 표시되는 경보를 정의합니다.
  • 특정 필드의 특정 값이 있음
  • 특정 필드의 값이 어떤 값과 유사함
  • 두 가지 고정 시점 사이에 이벤트가 발생함
  • 특정 시간(분) 전까지 이벤트가 발생함
다수의 기본 필터가 Tivoli Netcool/OMNIbus와 함께 제공됩니다. 이 필터는 다음과 같습니다.
최근 10분
이 필터는 ProbeWatch, TSMWatch, ConnectionWatch를 제외한 모든 경보를 표시합니다. 최근 10분 필터가 마지막 10분에 발생한 경보만 표시하는 것을 제외하면 결과는 기본 필터와 동일합니다.
유지보수
이 필터는 운영자가 "유지보수"로 표시한 경보를 표시합니다.
에스컬레이션
이 필터는 운영자가 에스컬레이션한 경보를 표시합니다.
정보
이 필터는 정보 경보로 식별된 경보를 표시합니다. 정보 경보는 다음과 같습니다.
  • 프로브를 통해 정보 경보로 지정된 경보
  • ProbeWatch, TSMWatch 또는 ConnectionWatch가 아닌 경보
  • 에스컬레이션 레벨이 설정된 경보
태스크 목록
이 필터는 운영자의 태스크 목록에 추가된 경보를 표시합니다.
Netcool® 상태
이 필터는 ProbeWatch, TSMWatch, ConnectionWatch 경보만 표시합니다.
기본값
이 필터는 ProbeWatch, TSMWatch, ConnectionWatch를 제외한 모든 경보를 표시합니다.
모든 이벤트
이 필터는 모든 경보를 표시합니다.

사용자 정의 필터를 빌드할 수도 있습니다. 필터는 .elf 파일로 저장됩니다. 필터 빌더를 사용하여 필터를 작성하고 편집합니다.

  • 필터의 구성요소
    필터는 요소의 콜렉션으로 구성되어 있습니다. 사용 가능 요소 유형은 조건, 논리 및 하위 조회 요소를 포함합니다.
  • 필터 지표
    모든 필터는 연관되어 있는 지표를 포함할 수 있습니다. 지표는 유용한 계수를 계산하기 위해 필터에 적용할 수 있는 측정법입니다.
  • 필터에 대한 우수 사례
    필터 빌드 시 집중 테스트를 먼저 수행하십시오.
  • 필터 빌더 시작
    필터 빌더는 필터를 구성하는 SQL 문을 빌드 및 테스트할 수 있는 그래픽 인터페이스를 제공합니다.
  • UNIX에서 필터 작성 또는 편집
    UNIX 필터 빌더를 사용하면 이벤트 목록 데이터의 필터를 빌드할 수 있습니다.
  • Windows에서 필터 작성 또는 편집
    Windows 필터 빌더를 사용하면 이벤트 목록 데이터의 필터를 빌드할 수 있습니다.
  • 끌어놓기로 UNIX에서 필터 작성
    UNIX에서 끌어놓기 기능을 사용하여 필터를 작성할 수 있습니다.
  • 필터 열기
    해당 정의를 보거나 편집하기 위해 저장된 필터를 열 수 있습니다.
상위 주제: 필터 및 보기를 사용한 사용자 정의 이벤트 정보 표시
