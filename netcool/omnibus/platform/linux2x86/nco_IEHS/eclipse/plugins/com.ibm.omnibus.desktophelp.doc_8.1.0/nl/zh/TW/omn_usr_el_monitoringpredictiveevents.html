監視事件清單中的預測事件

預測事件代表對即將發生的臨界值違規的預測。這些事件不同於轉遞至 Tivoli Netcool/OMNIbus 的其他事件，因為它們代表尚未發生的事件、錯誤或狀況，但在近期很有可能會發生。

預測事件將在 IBM Tivoli Monitoring 中根據歷程資料的線性趨勢來產生。如果系統已配置為支援此功能，則您可以從 Tivoli Netcool/OMNIbus 事件清單監視預測事件。如果已進行此配置，則「事件清單」監視器框視窗中還會提供預測監視器框，且事件清單中會提供預測過濾器及預測視圖。預測視圖包含下列預設直欄，這些直欄從左到右顯示如下：Node、TrendDirection、Summary、FirstOccurrence、LastOccurrence、Count、PredictionTime。

直欄的排序優先順序和排序順序如下所示：
  1. 嚴重性以降冪排序
  2. LastOccurrence 以升冪排序
  3. PredictionTime 以升冪排序
在事件清單同時顯示實際事件及預測事件時，您可以使用預測過濾器及預測視圖來過濾預測事件，並對其排序。您也可以識別預測事件，如下所述：
  • 檢查摘要直欄中的文字。摘要文字通常為下列其中一種格式（重要預測事件的嚴重性一律為 5，警告預測事件嚴重性一律為 3）：
    • 對於線性趨勢事件：

      Node 上的預測事件比率將在 PredictionTime 天內達到臨界值。

    • 對於基準線事件：

      EventType 事件比率基準線狀況已針對節點 Node 激發，節點 situation_name 的事件計數為 EventCount。

    • 對於預測事件：

      將在 PredictionTime 日後超過 Node 上 resource 的 「重要」臨界值。（狀況：situation_name）。

      將在 PredictionTime 日後超過 Node 上 resource 的 「警告」臨界值。（狀況：situation_name）。

  • 如果現行視圖中具有類別直欄，請檢查此直欄中的文字。依預設，此項目為預測事件，但系統管理者可以變更此值。

從事件清單中，您可以對預測事件執行下列作業：

  • 決定預測的可信度。若要顯示符合某個預測可信度的預測事件：
    1. 用滑鼠右鍵按一下事件清單中的任何事件，然後從蹦現功能表中按一下顯示預測可信度。
    2. 從「內部指令參數」視窗的最小可信度百分比欄位中，指定僅限 0 至 100 之間的百分比值。 預測可信度會指出近似趨勢值的準確性，其中 0 表示不可信，100 表示幾乎完全可信。預設值為 50。
    3. 按一下確定。 畫面上會開啟個別事件清單，其中顯示可信度為指定等級及以上的預測事件。此事件清單會顯示「預設」視圖中的所有欄位。您還可以按兩下任何事件，以在「事件資訊」視窗中檢視其完整詳細資料。
  • 列出來自所預測的主機的所有預測事件及實際事件（如果有的話）。您可以使用顯示的結果，來協助識別預測趨勢接近警告或重要臨界值的可能原因。

    若要顯示這些事件，請用滑鼠右鍵按一下您要探索的預測事件，然後從蹦現功能表中按一下顯示預測事件臨界值。畫面上會開啟個別事件清單，其中顯示與所選預測事件處於相同主機上，並且在預測開始時間到計算的預測發生時間這段時間內發生的所有事件。（「預設」視圖將套用至此事件清單。）

    您可以按兩下每個事件，以在「事件資訊」視窗中檢視其完整詳細資料。
