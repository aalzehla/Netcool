使用过滤器和视图显示定制事件信息

警报数据库表存储了转发到 ObjectServer 的事件信息。 您可使用过滤器和视图来定义在事件列表中显示这些事件信息中的哪些信息。

过滤器控制哪些警报（或数据库记录）可见，视图控制这些警报的哪些列（或数据库字段）可见。 您可使用结构化查询语言（SQL）对数据库表执行信息请求。

要在事件列表中显示警报，需要对 alerts.status 表执行 SELECT 语句。 SQL 语句可划分为三个概念部分：
  • 使用在视图构建器中创建的视图选择要查看 alerts.status 表中的哪些列。
  • 使用在过滤器构建器中创建的过滤器选择一列或多列中包含特定数据值的事件。
  • 使用在视图构建器中创建的视图选择显示事件的顺序。

以下示例显示了对 alerts.status 表执行的 SELECT　语句： 

select Node as NODE, Summary as Detail from alerts.status
where Node ='link1'

SELECT 子句是在视图构建器中构造的，视图构建器用于创建和编辑视图。 WHERE 子句是在过滤器构建器中构造的，过滤器构建器用于创建和编辑过滤器。

  • 为事件列表数据设置过滤器
    过滤器用来构建可在 ObjectServer 警报数据库表上运行的 SQL 查询，以便在监视器框和事件列表中显示定制警报信息。
  • 为事件列表设置视图
    视图定义事件列表中所显示的列的选择，以及这些列中的信息格式。
