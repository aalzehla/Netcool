데스크탑 ObjectServer 아키텍처

데스크탑 ObjectServer 아키텍처를 사용하여 자주 과부하가 발생하는 ObjectServer의 성능을 개선할 수 있습니다.

데스크탑 ObjectServer 아키텍처:

  • 로드를 특별화된 데스크탑 ObjectServer로 이동하여 중앙 ObjectServer의 워크로드를 감축합니다.
  • 데스크탑 응답을 개선합니다. 즉, 데스크탑 운영자의 조치 및 데스크탑 GUI의 반영 간의 시간을 개선합니다.
  • 데스크탑 GUI 보류 가능성을 감축합니다.
  • 과부하된 표준 ObjectServer 구성의 종단간 지연 시간을 개선합니다.
  • 마스터 ObjectServer를 동시에 업데이트하여 높은 데이터 무결성과 일관성을 유지합니다.

데스크탑 ObjectServer 아키텍처는 단일 ObjectServer에서 정상적으로 수행된 의무를 공유하는 하나 이상의 데스크탑 ObjectServer와 마스터 ObjectServer로 구성됩니다. 데이터 작성 시 DSD(Dual server desktops)는 단일 마스터 ObjectServer에 연결하지만 데스크탑 ObjectServer에서 경보 데이터를 읽고 표시합니다. DSD의 기본 기능은 표준 데스크탑과 동일합니다. 조작의 경우, DSD는 표준 데스크탑과 동일하게 동작합니다.

데스크탑 ObjectServer 아키텍처가 다음 그림에 표시됩니다.

그림 1. 듀얼 서버 데스크탑 아키텍처 예제

DSD는 데스크탑 ObjectServer와 마스터 ObjectServer를 동시에 연결합니다. DSD에서 수행되는 운영자 조치(예: 도구나 저널 조치)가 단방향 SQL 연결을 통해 마스터 ObjectServer에 직접 전송됩니다.

마스터 ObjectServer의 경보가 데스크탑 ObjectServer를 통해 DSD에 전송됩니다. 단방향 ObjectServer 게이트웨이를 마스터 ObjectServer에서 데스크탑 ObjectServer로, IDUC 연결을 데스크탑 ObjectServer에서 DSD로 사용하여 아카이브됩니다.

다음 그림에서 표시된 대로 Dual-write 모드가 사용되면 업데이트는 다른 단방향 SQL 연결을 통해 데스크탑 ObjectServer에도 전송됩니다.
