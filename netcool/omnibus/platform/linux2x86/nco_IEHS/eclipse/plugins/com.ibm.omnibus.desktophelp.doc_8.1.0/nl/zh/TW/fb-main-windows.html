過濾器建置器

使用此視窗，可以定義用於過濾事件清單中警示的 SQL 查詢 (或 WHERE 子句)。 您可以使用「過濾器建置器」來設定及維護事件資料的這些過濾器。

「過濾器建置器」包含下列區域：
  • 過濾器設定區域 (頂端畫面)
  • 按鈕列 (過濾器設定區域下方的畫面)
  • 圖形顯示區域 (按鈕列下方的左上方畫面)
  • 元素定義區域 (按鈕列下方的右上方畫面)
  • SQL 顯示區域 (左下方畫面)
  • 度量區域 (右下方畫面)

過濾器設定區域

使用此區域，可以定義過濾器的名稱並設定其許可權。

如下所示完成此區域：

名稱
如果您正在建立新過濾器，請以過濾器的唯一名稱改寫現行名稱。 此名稱可用於選取事件清單功能表中的過濾器。 如果您正在編輯現有的過濾器，則不需要在此欄位中進行輸入。
可編輯
如果您擁有相關管理者權限，則可以選取此勾選框，讓其他使用者可編輯此過濾器，或者取消勾選該勾選框，以保護過濾器，使其免於修改。
限制： 只有在過濾器儲存為事件清單配置（.elc 檔）時， 才適用這項設定。

按鈕列

使用這些按鈕，可以定義過濾器元素。

如下所示完成此區域：

條件
按一下此按鈕，以建立條件元素。 條件元素可讓您在欄位與值之間進行比較。 條件按鈕會建立 WHERE 子句並容許指定值；例如：where Node = 'link1'
子查詢
按一下此按鈕，以建立條件內的查詢。 子查詢是更複雜的條件，可用於將值與其他 SELECT 陳述式的結果進行比對。
新增 AND 分支
按一下此按鈕，以將 AND 元素分支插入目前選取的元素之前。 您可以使用 AND 元素來合併兩個元素，以進行更複雜的比較。
新增 OR 分支
按一下此按鈕，以將 OR 元素分支插入目前選取的元素之前。 您可以使用 OR 元素來組合兩個元素，以進行更複雜的比較。
將 AND 變更為 OR
按一下此按鈕，以將目前選取的 AND 元素取代為 OR 元素。
將 OR 變更為 AND
按一下此按鈕，以將目前選取的 OR 元素取代為 AND 元素。
無效
按一下此按鈕，以將邏輯 NOT 運算子插入到目前選取的元素之前。
刪除樹狀結構項目
按一下此按鈕，以刪除現行元素及其所有子元素。
編輯 SQL 文字
按一下此按鈕，以開啟「過濾器 SQL 編輯」視窗來直接編輯 SQL。 當按一下確定以儲存並回到「過濾器建置器」時，會自動剖析 SQL 表示式，並將其新增至圖形顯示區域中的條件元素。 如果在「過濾器 SQL 編輯」視窗中輸入無效的 SQL 文字，則需要更正語法，然後才能結束視窗。
當您按一下上述任一按鈕時：
  • 圖形表示法會顯示在按鈕列下方的左上方畫面中的圖形顯示區域內。
  • SQL 語法會顯示在左下方清單框中的 SQL 顯示區域內。
  • 元素專用的欄位會出現在右方畫面的元素定義區域中。
註： 這些按鈕的可用性取決於目前選取的元素，以及對元素而言有效的作業。

圖形顯示區域

此區域會以顯示元素相互關係的圖形樹狀結構，來顯示現行過濾器的元素定義。 透過使用按鈕列中的按鈕，您可以將元素新增至樹狀結構。 會強調顯示目前選取的元素，且編輯與此元素相關。 當您選取此區域內的元素時，會顯示其詳細資料，且可在元素定義區域中進行編輯。

元素定義區域

此區域會顯示目前選取之元素的詳細資料，並可讓您指定一組不同的值。 選項會反映在圖形顯示區域及 SQL 顯示區域中。

如果已按一下條件按鈕來建立條件元素，請如下所示完成欄位：
類型
從此清單中選取下列任一個選項：
  • 簡式：選取此選項，可以在條件元素中使用簡式表示式。
  • 複式：選取此選項，可以使用複式表示式，其容許比較兩個資料庫直欄或兩個表示式。
直欄
從此清單中，選取您要在比較中使用的資料庫直欄。
依預設，清單會包含 alerts.status 資料庫表格中直欄的名稱。 「過濾器建置器」會自動決定可用的直欄。 在不同狀況下，此清單會有所不同。
運算子
從此清單中選取比較運算子。
可用的比較範圍由直欄下拉清單中的選項決定。 某些特定直欄無法使用某些比較。 例如，無法對數值直欄 (例如，「嚴重性」) 進行 LIKE 作業。
值
此處顯示的欄位取決於在類型、直欄及運算子清單中選取的選項。
如果類型為簡式，則顯示的值欄位取決於直欄清單中選取的資料庫直欄類型是字串、整數還是時間。
如果類型為複式，則會提供編輯按鈕，可讓您在「過濾器 SQL 編輯」視窗中直接編輯 SQL。 當按一下確定以儲存並回到「過濾器建置器」時，會自動剖析 SQL 表示式，並將其新增至圖形顯示區域中的條件元素。 如果在「過濾器 SQL 編輯」視窗中輸入無效的 SQL 文字，則需要更正語法，然後才能結束視窗。
如果已按一下子查詢按鈕來建立子查詢元素，請如下所示完成欄位：
直欄
從此清單中，選取要在搜尋中使用的資料庫直欄。
運算子
從清單中選取下列任一個選項：
  • 在：使用此選項，以搜尋直欄的內容。
  • 不在：使用此選項，以搜尋不存在的欄位內容。
選取
選取要在建置清單時使用，並針對其執行「在」或「不在」作業的直欄。 此清單中的選項由從清單中選取的選項決定。
從
選取資訊衍生自的資料庫表格。 這可以是 alerts.details、alerts.journal 或 alerts.status。

SQL 顯示區域

SQL 顯示畫面會在建置過濾器時顯示 SQL。 您可以按一下按鈕列中的編輯 SQL，以在「過濾器 SQL 編輯」視窗中直接編輯 SQL。

透過使用有效的 SQL 語法輸入過濾器查詢，然後按一下確定，以儲存您的輸入並回到「過濾器建置器」。 「過濾器建置器」會嘗試剖析 SQL 文字，並建置過濾器樹狀結構。 如果 SQL 無效，則會顯示一則錯誤訊息，而您必須在「過濾器 SQL 編輯」視窗中更正錯誤，然後才能繼續。

度量區域

使用度量按鈕，可以將測量套用至過濾器，以便它計算有用的數字。 例如，測量總和度量會提供 ObjectServer 中非重複警示的總數。

如下所示完成此區域：
度量
使用第一個清單，以選擇要使用的測量：
  • 選取平均值，以傳回符合過濾器之所有警示的選定欄位平均值。
  • 選取計數，以傳回符合過濾器的所有警示計數。 已選取的欄位並非用於此計算。
  • 選取總和，以傳回符合過濾器之所有警示的選定欄位總數。
  • 選取下限，以傳回符合過濾器之警示中的選定欄位最低值。
  • 選取上限，以傳回符合過濾器之警示中的選定欄位最高值。
使用第二個清單，以選取要套用測量的欄位。 只有警示中的整數及時間欄位才可用於度量計算。
提示： 度量值會顯示在「事件清單」監視器框視窗的監視器框中。

套用及儲存過濾器設定

定義或修改過濾器設定之後，您可以透過按一下相關按鈕來套用、儲存或取消所進行的變更，如下所示：
套用
按一下此按鈕，以將變更套用至過濾器，而不儲存它。
關閉
按一下此按鈕，以關閉視窗並捨棄變更。

您必須使用 .elf 副檔名儲存過濾器。 按一下檔案 > 儲存或檔案 > 另存新檔。
