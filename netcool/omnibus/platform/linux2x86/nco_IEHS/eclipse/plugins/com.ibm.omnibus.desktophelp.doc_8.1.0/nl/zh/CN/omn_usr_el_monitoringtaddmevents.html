监视事件列表中的 TADDM 事件

在您的 IT 环境中检测到配置更改时，将在 IBM Tivoli Application Dependency Discovery Manager 内生成 TADDM 事件。如果您的系统已配置为支持该功能，那么可以通过 Tivoli Netcool/OMNIbus 事件列表监视 TADDM 事件。

如果已配置，那么您的事件列表中将提供 TADDM 过滤器来过滤 TADDM 事件。您还可以通过对类列（如果当前视图中提供了该列）排序来识别事件列表中的 TADDM 事件。缺省情况下，条目为 Tivoli Application Dependency Discovery Manager，尽管该值可由您的系统管理员更改。

TADDM 事件缺省具有“不确定”严重性级别。

从事件列表，您可以启动到 TADDM Java 控制台或 Web 客户机，以查看关于已为其生成警报的配置项的更多详细信息：

  • 要在 TADDM Java 控制台中查看配置项的所有属性详细信息，右键单击事件列表中的 TADDM 事件，然后从弹出菜单中单击 TADDM > 配置项详细信息（控制台）。
  • 要在 TADDM Web 客户机中查看配置项的所有属性详细信息，右键单击事件列表中的 TADDM 事件，然后从弹出菜单单击 TADDM > 配置项详细信息（Web）。
  • 要在 TADDM Java 控制台中查看配置项的变更历史记录报告，右键单击事件列表中的 TADDM 事件，然后从弹出菜单中单击 TADDM > 变更历史记录（控制台）。
  • 要在 TADDM Web 客户机中查看配置项的变更历史记录报告，右键单击事件列表中的 TADDM 事件，然后从弹出菜单单击 TADDM > 变更历史记录（Web）。

    您可以双击某一事件，以在“事件信息”窗口中查看其完整详细信息。
