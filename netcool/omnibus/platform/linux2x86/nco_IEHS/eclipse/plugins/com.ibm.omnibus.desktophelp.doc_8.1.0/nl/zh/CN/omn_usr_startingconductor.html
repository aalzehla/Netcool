IBM Tivoli Netcool/OMNIbus V7.4

启动 Conductor

Conductor 向所有 Tivoli Netcool/OMNIbus 桌面工具提供了单点登录访问，以用于查看和管理警报。

要使用桌面工具，必须使用有效的认证凭证来连接到正在运行的 ObjectServer。

提示： 可以选择从 UNIX 内的命令行或从 Windows 开始菜单单独启动桌面工具。但是，每次以此方法启动桌面工具时，都需要输入用于认证和连接到选定 ObjectServer 的用户名和密码。

要启动 Conductor：

  1. 为操作系统执行相应的操作：
    选项 描述
    UNIX 从命令提示符输入：$NCHOME/omnibus/bin/nco &
    Windows
    • 从 Windows 开始菜单单击 Netcool Conductor。
    • 从命令提示符输入：

      %NCHOME%\omnibus\desktop\NCOConductor.exe
    此时将打开 Conductor 的“登录”窗口。
  2. 如下所示完成此窗口：
    用户名
    请输入用户名。
    密码
    请输入密码。 在您输入文本之处显示星号（*）。
    注： 如果是依据外部认证源（而不是 ObjectServer）验证用户名和密码，请检查该源是否支持多字节字符。 如果不支持多字节字符，那么您必须使用 ASCII 字符指定用户名和密码。
    服务器
    ObjectServer 名称缺省设置为 NCOMS（Netcool/OMNIbus 主控服务器）。 如果您的环境配置了多个 ObjectServer，那么所有可用的 ObjectServer 将显示在此下拉列表中。 选择想要连接到的 ObjectServer。
    确定
    单击此按钮可认证您的登录详细信息。
    注： 允许您进行三次登录尝试。 每次尝试时，都会清空密码字段。 如果在第三次尝试时您仍然未能正确登录，那么程序将终止。
    取消
    单击此按钮可取消您的登录尝试。

结果

成功登录后：
  • UNIX Conductor 将以窗口方式打开。
  • Windows Conductor 将显示为 Windows 系统托盘内的总线图标。右键单击此图标可查看 Windows Conductor 菜单。

    可以为每个正在运行的 ObjectServer 启动 Conductor。 在 Windows 上，在系统托盘内会分配给每个 Conductor 一个图标。
