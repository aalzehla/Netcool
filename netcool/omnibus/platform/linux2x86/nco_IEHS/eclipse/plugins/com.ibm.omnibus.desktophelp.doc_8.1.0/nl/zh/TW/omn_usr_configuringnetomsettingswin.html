IBM Tivoli Netcool/OMNIbus 7.4 版

在 Windows 上配置 Tivoli Netcool/OMNIbus 設定

在單一作業中，您可以配置 Windows 上 Tivoli Netcool/OMNIbus 的內容及伺服器通訊資訊。

若要在 Windows 上配置設定：
  1. 按一下開始 > 程式集 > Netcool 套組 > 系統公用程式 > OMNIbus 設定。 即會開啟「Netcool/OMNIbus 設定」視窗。
    提示： 如果您對其中一個標籤進行變更，然後嘗試存取另一個標籤而沒有儲存變更，則系統會提示您儲存或取消變更。 您也可以隨時按一下確定按鈕以儲存所有變更並關閉視窗，或按一下取消按鈕以關閉視窗而不儲存變更。 如果要儲存您對任何標籤所做的變更而不關閉視窗，請按一下套用按鈕。
  2. 使用概觀標籤，可以檢視 Tivoli Netcool/OMNIbus 伺服器元件的位置及內容相關資訊。
  3. 從伺服器標籤，可以建立及修改伺服器元件的通訊明細、測試伺服器活動，以及新增備份 (或失效接手) 伺服器與接聽器。 如下所示完成此標籤：
    伺服器清單
    伺服器清單會顯示現有的伺服器元件及其設定：
    伺服器
    此直欄會顯示對此工作站定義的每個伺服器名稱。 預設伺服器名稱為：
    • NCOMS：ObjectServer 的預設名稱
    • NCO_PROXY：Proxy 伺服器的預設名稱
    • NCO_GATE：閘道的預設名稱
    • NCO_PA：程序代理程式的預設名稱
    伺服器的主機工作站必須擁有接聽器項目，而連接至該伺服器的主機工作站及任何其他工作站都必須擁有用戶端項目。
    備份 ObjectServer 顯示在「伺服器編輯器」中主要 ObjectServer 的下方。 若要隱藏備份 ObjectServer 的清單，請按兩下主要 ObjectServer 名稱或伺服器圖示。 系統會隱藏備份 ObjectServer，且字母 C 會顯示在伺服器圖示中。 再按兩下，以顯示備份 ObjectServer。 伺服器圖示會回到正常狀態。
    主機
    此直欄會顯示安裝伺服器元件之工作站的主機名稱或 IP 位址。 預設值是主機名稱。
    埠
    此直欄會顯示伺服器元件接聽未加密連線的埠。
    SSL
    此直欄會對已加密連線顯示單字是。 相同的埠號可同時用於已加密和未加密連線。
    伺服器登錄欄位
    使用伺服器登錄欄位，可以輸入或編輯每個伺服器元件的詳細資料。 若您是在編輯現有伺服器元件的詳細資料，則您必須首先選取伺服器清單中的相關列，以便在伺服器登錄欄位中顯示詳細資料。
    名稱
    輸入新伺服器元件的名稱或編輯現有元件的名稱。 您也可以使用下拉清單來選取名稱。 在命名元件時，請使用下列字尾：_PROXY (用於 Proxy 伺服器)、_GATE (用於閘道) 及 _PA (用於程序控制代理程式)。
    註： 伺服器項目的名稱必須由 29 個以下大寫字母組成，且不得以整數開頭。
    主機
    輸入或編輯安裝伺服器元件之工作站的主機名稱或 IP 位址。
    如果輸入 IP 位址，您就可以指定 IPv4 或 IPv6 位址。 例如：
    • 192.168.0.1
    • 2094:82a:2a6e:123:503:badd:fe43:f552
    埠
    請輸入有效且未使用的埠號。
    接聽器
    若這是伺服器之主機工作站上的接聽器項目，請選取此勾選框。 若這是用戶端項目，請取消勾選接聽器勾選框。
    SSL
    選取此勾選框，指出埠接受來自使用 SSL 的用戶端已加密連線。 相同的埠號可用於已加密和未加密連線。
    新增
    按一下此按鈕，以將新 (而且唯一的) 伺服器明細新增至伺服器清單。
    移除/更新
    此按鈕會根據您是否已針對目前選取的伺服器元件進行變更而改變。 如果要從伺服器清單中移除目前選取的伺服器元件，請按一下移除。 按一下更新，以使用現有伺服器元件的已更新明細重新整理伺服器清單。
    測試
    按一下此按鈕，以測試是否可以連接至伺服器清單中選取的伺服器。 「伺服器編輯器」會嘗試聯絡指定之主機及埠上的伺服器。 視窗會顯示測試指令的結果。
    優先順序
    伺服器優先順序欄位可讓您提高或降低已配置為失效接手系統之伺服器元件的優先順序。 例如，假設使用名稱為 NCOMS_BAK 的備份配置名稱為 NCOMS_PRI 的 ObjectServer。 使用提高或降低按鈕，您可以將 NCOMS_BAK 的優先順序提高為主要 ObjectServer，而將 NCOMS_PRI 降低為備份。
    提高
    按一下此按鈕，以將所選伺服器元件的優先順序提高一個層次。
    降低
    按一下此按鈕，以將所選伺服器元件的優先順序降低一個層次。
    顯示群組
    選取此勾選框，以在伺服器清單中依名稱對每個伺服器進行分組，在用戶端項目之後接著已識別的備用伺服器及接聽器。
  4. 從內容標籤，編輯每個 ObjectServer、Proxy 伺服器、程序代理程式、探測器及閘道之內容檔中包含的資訊。 如下所示完成此標籤：
    類型
    選取下列任一選項：
    • 選取 ObjectServer 以編輯元件的內容檔，例如 ObjectServer 和 Proxy 伺服器。 名稱下拉清單會移入這些檔案。 這些檔案一般會儲存在 %NCHOME%\omnibus\etc 目錄中。
    • 選取探測器以編輯探測器內容檔。 名稱下拉清單會移入這些檔案。 這些檔案一般會儲存在 %NCHOME%\omnibus\probes\win32 目錄中。
    • 選取伺服器以新增內容檔。 這些檔案會儲存在 %NCHOME%\omnibus\ini 目錄中。 名稱下拉清單會移入文字 <New File>。
    名稱
    從此下拉清單選取您要編輯的內容檔。 基於從類型下拉清單進行的選取，資料會移入此清單。 預設值為：
    • nco_dbinit：資料庫起始設定開始時 (執行 nco_dbinit 以起始設定資料庫)，會讀取此內容檔。
    • NCO_PROXY：Proxy 伺服器內容檔 (Proxy 伺服器可用來減少與 ObjectServer 的探測器連線數目。)
    • nco_confpack：Confpack 內容檔 (nco_confpack 可用來匯入及匯出 ObjectServer 配置。)
    • nco_config：Netcool/OMNIbus Administrator內容檔
    • NCOMS：ObjectServer 內容檔
    請注意，畫面上沒有顯示 .props 副檔名，而且當您在名稱清單上選取檔案時，內容清單會自動移入可用的內容。
    內容清單
    此矩陣會顯示已新增至內容檔之任意內容的詳細資料。 如果您要變更內容，則可以選取此清單中的內容。 有一組已為探測器內容檔定義的預設內容組，且其無法移除。 這些內容會在內容清單中以灰色顯示。
    名稱
    此欄位會顯示已從內容清單中選取，要進行編輯的內容名稱。 您無法變更現有內容的名稱。 您也可以使用下拉清單來選取要編輯的內容。
    如果您要將新內容新增至選取的內容檔，請在此處輸入內容名稱。
    如果已從類型清單選取伺服器，以建立新的內容檔，請在此處輸入新內容名稱。
    值
    如果您正在使用現有且已選取的內容檔，請編輯所選取內容的值，或輸入您要新增之新內容的值。
    如果已從類型清單選取伺服器，以建立新的內容檔，請輸入新內容的值。
    新增/更新
    此按鈕會視您是否要新增或編輯現有的內容而改變。 如果輸入新的內容名稱及值，請按一下新增，以將值新增至內容清單。 如果變更內容清單中所選取內容的值，請按一下更新，以更新內容清單中的值。
    如果已從類型清單選取伺服器，以建立新的內容檔，請按一下新增以新增內容。
    移除
    按一下此按鈕，以從內容檔移除所選內容。 如果選取預設內容，則無法使用此按鈕。
    儲存
    按一下此按鈕，以將變更儲存至內容檔。
    如果已從類型清單選取伺服器，以建立新的內容檔，請在將內容新增至內容清單之後，按一下儲存。 在「儲存新內容」視窗上，輸入內容檔的名稱，但請省略 .props 副檔名；這會自動為您新增。 按一下儲存以將檔案儲存至 %NCHOME%\omnibus\ini 目錄中。 內容檔名稱會顯示在名稱欄位中 (這個欄位與類型欄位相鄰)，然後您可以繼續將內容新增至此檔案。 請注意，您需要編輯此檔案，以將引號新增至針對內容指定的字串值。
