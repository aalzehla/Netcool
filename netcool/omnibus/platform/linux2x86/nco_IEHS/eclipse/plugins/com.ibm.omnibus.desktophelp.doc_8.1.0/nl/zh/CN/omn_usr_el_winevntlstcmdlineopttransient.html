瞬时事件列表的 Windows 命令行选项

在 Windows 上运行 NCOelct.exe 命令启动瞬时事件列表时，必须指定若干命令行选项来定制事件列表。可以从 %NCHOME%\omnibus\desktop 目录运行此命令。

下表描述了用于在 Windows 上启动瞬时事件列表的命令行选项。

表 1. Windows 瞬时事件列表（NCOelct.exe）命令行选项
命令行选项 描述
-diagnostic 显示扩展诊断消息。
-dualwrite 0 | 1 指示在桌面 ObjectServer 体系结构中操作时是启用还是禁用双重写方式。 有效选项如下所示：
  • 0：禁用双写入方式
  • 1：启用双重写方式

-dualwrite 命令行选项将覆盖桌面 ObjectServer master.national 表中的 DualWrite 字段项。
-elf string | URL

这是必需选项。

指定要用于瞬时事件列表的过滤器文件，其中：

string 指定 .elf 文件的路径。

URL 指定远程服务器上可使用 HTTP 或 FTP 进行访问的 .elf 文件。 通过使用以下某种格式指定 URL：

  • http://servername[:port]/path/filename.elf
  • http://username:password@servername[:port]/path/filename.elf
  • ftp://servername[:port]/path/filename.elf
  • ftp://username:password@servername[:port]/path/filename.elf

仅当服务器未使用缺省端口时才需要指定可选端口号 :port。

如果使用 FTP，那么路径必须是从根目录开始的完整路径，例如，/home/user/mydir/myfilter.elf。

除了在 .elf 文件中指定的过滤器参数之外，还可以使用 -params 命令行选项来指定过滤器参数。 关于用法的详细信息，请参阅位于此表内的 -params 描述。
-elv string | URL 指定用于瞬时事件列表的事件列表视图文件，其中：

string 指定 .elv 文件的路径。

URL 指定远程服务器上可使用 HTTP 或 FTP 访问的 .elv 文件。 通过使用以下某种格式指定 URL：

  • http://servername[:port]/path/filename.elv
  • http://username:password@servername[:port]/path/filename.elv
  • ftp://servername[:port]/path/filename.elv
  • ftp://username:password@servername[:port]/path/filename.elv

仅当服务器未使用缺省端口时才需要指定可选端口号 :port。

如果使用 FTP，路径必须是从根目录开始的完整路径；例如，/home/user/mydir/myview.elv。
-help 显示该命令的帮助并退出
-masterserver string 指示桌面 ObjectServer 体系结构中要使用的主控 ObjectServer。

-masterserver 命令行选项将覆盖桌面 ObjectServer master.national 表中的 MasterServer 字段项。
-networktimeout integer 指定发生网络故障时，多少秒后登录尝试或连接 ObjectServer 将超时。 指定的超时时段后，事件列表将尝试重新连接 ObjectServer。 如果超时时段后的连接不成功，事件列表将尝试连接到备份 ObjectServer（如果可用）。 缺省值为 20 秒。
-params string 指定字符串，用于替换过滤器文件的 filter_text 参数内的单个占位符。 此占位符必须以 @ 符号开头（例如，@FilterText）。
-password string 指定所选用户名的密码。 NCOelct 实用程序仅接受密码内包含字母数字字符。 密码中不允许出现 #、/、>、@ 和 ! 等特殊字符。
-server string 指定要连接的 ObjectServer 的名称。
-username string 指定用于登录事件列表的用户名。
-version 显示版本信息并退出。
