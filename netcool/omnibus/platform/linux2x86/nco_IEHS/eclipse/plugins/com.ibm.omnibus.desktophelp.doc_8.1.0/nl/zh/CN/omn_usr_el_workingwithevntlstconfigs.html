处理事件列表配置

可以保存为“事件列表”监视器框窗口上的所有监视器框定义的过滤器、视图以及其他事件配置。 该信息保存在事件列表配置（.elc）文件中。

可以根据需要创建尽可能多的配置文件，并且可以随时将不同的配置文件装入“事件列表”监视器框窗口。

可以打开在本地保存的或在远程服务器上保存的事件列表配置。 也可以本地保存事件列表配置，或将事件列表配置保存到可通过使用 HTTP 或 FTP 访问的远程服务器上。

  • 保存本地事件列表配置
    从“事件列表”监视器框窗口可以保存为当前监视器框组定义的过滤器、视图以及其他警报配置，之后将此配置应用于事件列表。 如果您已修改了任何为当前监视器框配置的过滤器，那么您会发现保存是有用的。
  • 打开本地事件列表配置
    如果之前已本地保存过一个或多个事件列表配置，那么可以打开任何相关联的事件列表配置文件，以便监视和管理与已定义的过滤器和视图相匹配的事件。
  • 打开远程事件列表配置
    可以打开远程事件列表配置文件，以便打开一个或多个位于远程服务器上的事件列表。 服务器必须可以使用 HTTP 或 FTP 进行访问。
  • 保存远程事件列表配置
    如果已从网络上的远程服务器打开了事件列表配置，那么将不允许将其保存到该服务器上。 必须将文件保存到不同的位置。 服务器必须可通过使用 HTTP 或 FTP 进行访问。
  • 在 UNIX 上创建和编辑事件列表配置
    可以创建或编辑事件列表配置来保存“事件列表”监视器框窗口内出现的监视器框的过滤器和视图定义。
  • 在 Windows 上创建和编辑事件列表配置
    可以创建或编辑事件列表配置来保存“事件列表”监视器框窗口内出现的监视器框的过滤器和视图定义。
