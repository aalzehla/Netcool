IBM Tivoli Netcool/OMNIbus, 버전 7.4

서버 편집기

사용자의 Tivoli Netcool/OMNIbus 시스템의 호스트에서 서버 구성요소를 설치 또는 변경하는 경우, 서버 편집기를 사용하여 다른 구성요소와 통신하도록 구성요소를 구성해야 합니다. 서버 구성요소에 대한 통신 세부사항을 작성 및 수정하고 서버 활동을 테스트하며, 백업(또는 장애 조치) 서버 및 리스너를 추가하려면 이 창을 사용하십시오.

서버 구성요소에는 ObjectServer, 게이트웨이, 프로세스 제어 에이전트, 프록시 서버가 있습니다. 시스템 관리자 및 수퍼유저만 이 기능을 사용할 수 있습니다.

호스트 워크스테이션 및 서버 구성요소에 연결해야 하는 모든 워크스테이션에서 서버 통신 정보를 유지보수해야 합니다. 통신 설정을 구성하면 서버 편집기가 연결 데이터 파일에 결과를 저장합니다. 이 파일의 이름과 위치는 다음과 같습니다.
UNIX 시스템 $NCHOME/etc/omni.dat
Windows 시스템 %NCHOME%\ini\sql.ini
다음과 같이 이 창을 완료하십시오.
서버 목록
서버 목록은 기존 서버 구성요소 및 해당 설정을 표시합니다.
서버
이 컬럼은 이 워크스테이션에 대해 정의되는 각 서버의 이름을 표시합니다. 기본 서버 이름은 다음과 같습니다.
  • NCOMS: ObjectServer의 기본 이름
  • NCO_PROXY: 프록시 서버의 기본 이름
  • NCO_GATE: 게이트웨이의 기본 이름
  • NCO_PA: 프로세스 에이전트의 기본 이름
Windows에서는 서버의 호스트 워크스테이션에 리스너 항목이 있어야 하며, 호스트 워크스테이션 및 해당 서버에 연결하는 기타 워크스테이션에 클라이언트 항목이 있어야 합니다.
백업 ObjectServer는 서버 편집기의 기본 ObjectServer 아래에 표시됩니다. 백업 ObjectServer의 목록을 숨기려면 기본 ObjectServer 이름 또는 서버 아이콘을 두 번 클릭하십시오. 백업 ObjectServer가 숨겨지고 문자 C가 서버 아이콘에 나타납니다. 백업 ObjectServer를 표시하려면 다시 두 번 클릭하십시오. 서버 아이콘이 정상으로 돌아옵니다.
호스트 이름
이 컬럼은 서버 구성요소가 설치된 워크스테이션의 호스트 이름 또는 IP 주소를 표시합니다.
참고: 설치 후 호스트는 모든 서버에 대해 omnihost로 설정됩니다(UNIX). 이를 서버 호스트 워크스테이션의 이름으로 변경해야 합니다. Windows에서 기본값은 호스트 이름입니다.
포트
이 컬럼은 서버 구성요소가 암호화되지 않은 연결을 인식하는 포트를 표시합니다.
SSL
UNIX에서 이 컬럼은 서버 구성요소가 암호화된 연결을 인식하는 포트를 표시합니다. UNIX 시스템에서는 서버 구성요소에 표준 포트, SSL 포트 또는 두 포트가 모두 정의되어 있을 수 있습니다.
Windows에서 이 컬럼은 암호화된 연결에 대해 yes라는 단어를 표시합니다. Windows 시스템에서는 암호화된 연결과 암호화되지 않은 연결 모두에 동일한 포트 번호가 사용됩니다.
서버 항목 필드
서버 항목 필드를 사용하여 각 서버 구성요소의 세부사항을 입력하거나 편집하십시오. 기존 서버 구성요소의 세부사항을 편집할 경우 맨 먼저 서버 목록에서 해당 행을 선택하여 세부사항이 서버 항목 필드 안에 표시되도록 해야 합니다.
이름
새 서버 구성요소의 이름을 입력하거나 기존 구성요소의 이름을 편집하십시오. Windows에서는 드롭 다운 목록을 사용하여 이름을 선택할 수도 있습니다. 구성요소 이름을 지정할 때 프록시 서버에는 _PROXY, 게이트웨이에는 _GATE, 프로세스 에이전트에는 _PA 접미어를 사용하십시오.
참고: 서버 항목의 이름은 29자 이하의 대문자로 구성되어야 하며 정수 값으로 시작할 수 없습니다.
호스트
서버 구성요소가 설치된 워크스테이션의 호스트 이름 또는 IP 주소를 입력하거나 편집하십시오(UNIX에서 새 서버 구성요소의 경우, 이름은 기본적으로 omnihost로 설정되며 실제 호스트 이름 또는 IP 주소로 변경해야 함).
IP 주소를 입력하는 경우 IPv4 또는 IPv6 주소를 지정할 수 있습니다. 예:
  • 192.168.0.1
  • 2094:82a:2a6e:123:503:badd:fe43:f552
포트
UNIX에서는 암호화되지 않은 연결을 사용하는 클라이언트가 서버에 연결할 수 있게 하려면 이 필드에 사용하지 않은 올바른 포트 번호를 입력하십시오. 암호화되지 않은 연결을 허용하지 않으려면 포트를 설정하지 마십시오.
Windows에서는 이 필드에 사용하지 않은 올바른 포트 번호를 입력하십시오.
SSL
UNIX에서는 암호화된 연결을 사용하는 클라이언트가 서버에 연결할 수 있게 하려면 이 필드에 사용하지 않은 올바른 포트 번호를 입력하십시오. 암호화된 연결을 허용하지 않으려면 포트를 설정하지 마십시오.
Windows에서 포트가 SSL을 사용하는 클라이언트로부터 암호화된 연결을 허용하는 것을 표시하려면 이 선택란을 선택하십시오.
참고: UNIX 시스템에서는 서버 구성요소에 표준 포트, SSL 포트 또는 두 포트가 모두 정의되어 있을 수 있습니다. Windows 시스템에서는 어느 연결 유형이든 동일한 포트 번호가 사용됩니다.
리스너(Windows 전용)
서버 호스트 워크스테이션의 리스너 항목인 경우 이 선택란을 선택하십시오. 클라이언트 항목이면 리스너 선택란을 선택 취소하십시오.
추가
서버 목록에 새 고유한 서버 세부사항을 추가하려면 이 단추를 클릭하십시오.
제거/업데이트
이 단추는 현재 선택된 서버 구성요소를 변경했는지 여부에 따라 변경됩니다. 서버 목록에서 현재 선택된 서버 구성요소를 제거하려면 제거를 클릭하십시오. 기존 서버 구성요소의 업데이트된 세부사항으로 서버 목록을 새로 고치려면 업데이트를 클릭하십시오.
테스트
서버 목록에서 선택된 서버에 연결할 수 있는지 테스트하려면 이 단추를 클릭하십시오. 서버 편집기가 지정된 호스트와 포트의 서버에 연결하려고 시도합니다. 테스트 명령 결과가 창에 표시됩니다.
우선순위 필드
서버 우선순위 필드를 사용하면 오류 복구 시스템으로 구성되어 있는 서버 구성요소의 우선순위를 높이거나 낮출 수 있습니다. 예를 들어, NCOMS_PRI라는 ObjectServer는 NCOMS_BAK이라는 백업으로 구성된다고 가정합니다. 높임 또는 낮춤 단추를 사용하여 NCOMS_BAK의 우선순위를 기본 ObjectServer 및 NCOMS_PRI 백업이 되도록 높일 수 있습니다.
높임
선택된 서버 구성요소의 우선순위를 한 레벨씩 올리려면 이 단추를 클릭하십시오.
낮춤
선택된 서버 구성요소의 우선순위를 한 레벨씩 내리려면 이 단추를 클릭하십시오.
모두 생성(UNIX 전용)
모든 UNIX 운영 체제에 대해 인터페이스 파일을 생성하려면 이 단추를 클릭하십시오. 적용 단추를 사용하여 변경사항을 적용하면, $NCHOME/etc/interfaces.arch라는 인터페이스 파일이 생성됩니다. 여기서 arch는 개별 UNIX 플랫폼 이름을 나타냅니다(예: interfaces.hpux11 및 interfaces.solaris2).
그룹 표시(Windows 전용)
Windows에서, 클라이언트 항목 다음에 백업 서버 및 리스너가 식별된 상태에서 서버 목록 내에서 이름순으로 각 서버를 그룹화하려면 이 선택란을 선택하십시오.
적용(UNIX 전용)
인터페이스 파일에 변경사항을 저장하려면 이 단추를 클릭하십시오. 서버 목록에 추가, 서버 목록에서 제거 또는 편집된 서버 구성요소가 이 파일에 저장됩니다.
확인(Windows 전용)
변경사항을 저장하고 이 창을 닫으려면 이 단추를 클릭하십시오.
닫기(UNIX)/취소(Windows)
적용 단추를 사용하여 변경사항을 저장한 후 이 창을 닫거나 변경사항을 저장하지 않고 이 창을 닫으려면 이 단추를 클릭하십시오.
가져오기(UNIX 전용)
통신 세부사항을 가져오려면 이 단추를 클릭하십시오.
관련 태스크:
서버 통신 정보 구성
