监视事件列表中的预测事件

预测事件表示对即将发生的阈值违例的预测。这些事件不同于转发给 Tivoli Netcool/OMNIbus 的其他事件，因为它们表示尚未发生但很可能在近期发生的情况、故障或状态。

预测事件是根据历史数据的线性趋势在 IBM Tivoli Monitoring 中生成的。如果您的系统已配置为支持该功能，那么可以通过 Tivoli Netcool/OMNIbus 事件列表监视预测事件。 如果还进行了配置，那么将在“事件列表”监视器框窗口中提供预测监视器框，并在事件列表中提供预测过滤器和预测视图。预测视图包含下列缺省列（按从左到右顺序排列）：Node 、TrendDirection、Summary、FirstOccurrence、LastOccurrence、Count 和 PredictionTime。

列的排序优先级和排序顺序如下所示：
  1. Severity 按照降序排序
  2. LastOccurrence 按照升序排序
  3. PredictionTime 按照升序排序
当事件列表同时显示实际事件和预测事件时，您可以使用预测过滤器和预测视图对预测事件进行过滤和排序。您还可以如下确定预测事件：
  • 检查摘要列中的文本。摘要文本通常具有下列其中一种格式（紧急预测性事件的严重性始终为 5，警告预测性事件的严重性始终为 3）：
    • 对于线性趋势事件：

      Node 上的预测事件发生率将在 PredictionTime 天内达到阈值。

    • 对于基准事件：

      已对节点 Node 触发 EventType 事件速率基准情境，节点 situation_name 的事件计数为 EventCount。

    • 对于预测事件：

      节点上资源的临界阈值将在 PredictionTime 天内违规。（状态：situation_name）。

      节点上资源的警告阈值将在 PredictionTime 天内违规。（状态：situation_name）。

  • 如果您的当前视图中有类列，那么请检查该列中的文本。缺省情况下，条目为预测事件，尽管该值可由您的系统管理员更改。

从事件列表，可以对预测事件执行以下操作：

  • 确定预测的置信水平。要显示符合某个预测置信水平的预测事件：
    1. 右键单击事件列表中的任一事件，然后从弹出菜单单击显示预测置信度。
    2. 从“内部命令参数”窗口，在最低置信度百分比字段中仅指定 0 到 100 之间的百分比值。 预测置信度表示预计趋势值的准确度，其中 0 表示不可信，而 100 表示准确估计。缺省值为 50。
    3. 单击确定。 此时将打开独立的事件列表，显示具有指定的置信水平或更高的置信水平的预测事件。该事件列表显示缺省视图中的所有字段。您还可以双击任一事件，以在“事件信息”窗口中查看其完整详细信息。
  • 列出对其进行预测的主机所产生的所有预测事件和实际事件（如有）。您可以使用显示的结果来帮助识别走向警告阈值或临界阈值的预测趋势的可能原因。

    要显示事件，请右键单击要调查的预测事件，然后单击弹出菜单中的显示预测事件阈值。此时将打开独立的事件列表，显示与选定的预测事件位于同一主机上的所有事件，并显示哪些事件是在开始预测到计算预测要发生的这个时段内发生的。（缺省视图适用于该事件列表。）

    您可以双击每个事件，以在“事件信息”窗口中查看其完整详细信息。
