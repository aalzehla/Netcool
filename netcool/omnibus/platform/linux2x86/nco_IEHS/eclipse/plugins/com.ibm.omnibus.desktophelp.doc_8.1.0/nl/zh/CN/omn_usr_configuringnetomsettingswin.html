IBM Tivoli Netcool/OMNIbus V7.4

配置 Windows 上的 Tivoli Netcool/OMNIbus 设置

可以在单个任务中配置 Windows 上的 Tivoli Netcool/OMNIbus 的属性和服务器通信信息。

要配置 Windows 上的设置：
  1. 单击开始 > 程序 > Netcool Suite > 系统实用程序 > OMNIbus 设置。 将打开“Netcool/OMNIbus 设置”窗口。
    提示： 如果您对其中某个选项卡做出了更改，随即尝试访问其他选项卡而未保存更改，那么将提示您保存或取消更改。 在任何时刻，您都可以单击确定按钮保存所有更改并关闭窗口，或单击取消按钮关闭窗口而不保存更改。 如果要保存对任意选项卡所作的更改而不关闭窗口，单击应用按钮。
  2. 使用概览选项卡以查看关于 Tivoli Netcool/OMNIbus 服务器组件的位置和属性的信息。
  3. 从服务器选项卡，创建和修改服务器组件的通信详细信息，测试服务器活动，以及添加备份（或故障转移）服务器和侦听器。 以如下方式完成此选项卡：
    服务器列表
    服务器列表显示现有的服务器组件及其设置：
    Server
    此列显示为此工作站定义的每个服务器的名称。 缺省服务器名称为：
    • NCOMS：ObjectServer 的缺省名称
    • NCO_PROXY：代理服务器的缺省名称
    • NCO_GATE：网关的缺省名称
    • NCO_PA：进程代理程序的缺省名称
    服务器的主机工作站必须具有侦听器条目，而主机工作站和任何其他连接到该服务器的工作站必须具有客户机条目。
    备份 ObjectServer 在服务器编辑器中显示在主控 ObjectServer 之下。 要隐藏备份 ObjectServer 列表，请双击主控 ObjectServer 名称或服务器图标。 备份 ObjectServer 将被隐藏，服务器图标中将显示 C 字母。 再次双击可显示备份 ObjectServer。 服务器图标返回到正常状态。
    主机
    此列显示安装了服务器组件的工作站的主机名或 IP 地址。 缺省值为主机名。
    端口
    此列显示服务器组件侦听未加密连接的端口。
    SSL
    此列对加密连接显示是。 加密和未加密连接使用相同的端口号。
    服务器项字段
    使用服务器项字段可输入或编辑每个服务器组件的详细信息。 如果您要编辑现有服务器组件的详细情况，您必须首先在服务器列表中选择相应的行，以便在服务器项字段中显示详细情况。
    名称
    输入新服务器组件的名称或编辑现有组件的名称。 您也可以使用下拉列表选择名称。 命名组件时请使用以下后缀：_PROXY（代理服务器）、_GATE（网关）和 _PA（进程控制代理）。
    注： 服务器项的名称必须包含不超过 29 个大写字母，且不能以整数开头。
    主机
    输入或编辑安装了服务器组件的工作站的主机名或 IP 地址。
    如果输入 IP 地址，那么可以指定 IPv4 或 IPv6 地址。 For example:
    • 192.168.0.1
    • 2094:82a:2a6e:123:503:badd:fe43:f552
    端口
    输入有效的、未使用过的端口号。
    侦听器
    如果这是服务器主机工作站上的侦听器项，请选此复选框。 如果这是客户机项，请清除侦听器复选框。
    SSL
    选此复选框表示端口接受从客户机发出的使用 SSL 的加密连接。 加密和未加密连接使用相同的端口号。
    添加
    单击此按钮可向服务器列表添加新的（且唯一的）服务器详细信息。
    除去/更新
    此按钮将根据您是否对当前选定的服务器组件做出了更改而发生变化。 如果要从服务器列表除去当前选定的服务器组件，请单击除去。 单击更新可使用现有服务器组件的更新详细信息来刷新服务器列表。
    测试
    单击此按钮可测试您是否能连接到在服务器列表中选择的服务器。 服务器编辑器将尝试连接指定主机和端口上的服务器。 测试命令的结果将在窗口中显示。
    优先级
    服务器优先级字段可使您能够提高或降低已配置为故障转移系统的服务器组件的优先级。 例如，假设名为 NCOMS_PRI 的 ObjectServer 是使用名为 NCOMS_BAK 的备份配置的。 通过使用提高或降低按钮，您可提高 NCOMS_BAK 的优先级使之成为主控 ObjectServer，并降低 NCOMS_PRI 的优先级使之成为备份 ObjectServer。
    提高
    单击此按钮可将所选服务器组件的优先级提高一级。
    降低
    单击此按钮可将所选服务器组件的优先级降低一级。
    显示组
    选此复选框可在服务器列表中按名称对每个服务器分组，同时在客户机条目后标识备份服务器和侦听器。
  4. 从属性选项卡编辑每个 ObjectServer、代理服务器、进程代理程序、探测器和网关的属性文件内包含的信息。 以如下方式完成此选项卡：
    类型
    请选择以下任一选项：
    • 选择 ObjectServer 可编辑类似 ObjectServer 和代理服务器的组件的属性文件。 这些文件将出现在名称下拉列表中。 这些文件通常存储在 %NCHOME%\omnibus\etc 目录中。
    • 选择探测器可编辑探测器属性文件。 这些文件将出现在名称下拉列表中。 这些文件通常存储在 %NCHOME%\omnibus\probes\win32 目录中。
    • 选择服务器可添加新的属性文件。 这些文件保存在 %NCHOME%\omnibus\ini 目录中。 名称下拉列表将使用文本 <New File> 进行填充。
    名称
    从下拉列表中选择要编辑的属性文件。 显示的列表取决于从类型下拉列表中所作的选择。 缺省值为：
    • nco_dbinit：数据库初始化开始时（运行 nco_dbinit 以初始化数据库）要读取的属性文件。
    • NCO_PROXY：代理服务器属性文件（代理服务器可用于减少 ObjectServer 上的探测器连接数量）。
    • nco_confpack：Confpack 属性文件（nco_confpack 用于导入和导出 ObjectServer 配置）。
    • nco_config：Netcool/OMNIbus Administrator 属性文件
    • NCOMS：ObjectServer 属性文件
    请注意，不显示 .props 文件扩展名，当您在名称列表中选择了文件时将自动使用可用属性填充属性列表。
    属性列表
    此矩阵显示添加到属性文件的任何属性的详细信息。 要更改某个属性，可以在此列表中选择该属性。 已为探测器属性文件定义了一组缺省属性，用户不能除去这些属性。 这些属性将以灰色显示在属性列表中。
    名称
    此字段显示了您从属性列表中选定进行编辑的属性的名称。 不能更改已有属性的名称。 您也可以使用下拉列表来选择要编辑的属性。
    如果要向选定的属性文件添加新的属性，请在此输入属性的名称。
    如果您从类型列表选择了服务器以新建属性文件，请在此输入新属性的名称。
    值
    如果要使用现有的选定属性文件，请编辑选定属性的值，或为要添加的新属性输入值。
    如果您从类型列表选择了服务器以新建属性文件，请输入新属性的值。
    添加/更新
    此按钮将根据您要添加还是编辑现有属性而发生变化。 如果输入了新的属性名和值，单击添加可向属性列表添加该值。 如果更改了属性列表中选择的属性的值，单击更新可更新属性列表中的该值。
    如果您从类型列表选择了服务器以新建属性文件，单击添加可添加新的属性。
    除去
    单击此按钮可从属性文件中除去所选属性。 如果选择了缺省属性，那么不能使用此按钮。
    保存
    单击此按钮可保存属性文件的更改。
    如果您从类型列表选择了服务器来新建属性文件，请在向属性列表添加新的属性后单击保存 。 在保存新属性窗口中输入属性文件的名称，但请省略 .props 扩展名；系统将自动为您添加该扩展名。 单击保存将文件保存到 %NCHOME%\omnibus\ini 目录中。 属性文件名将显示在与类型字段相连的名称字段中，您可以继续向该文件添加属性。 请注意，将要求您编辑此文件以向您为属性指定的字符串值添加引号。
