IPv6 構成

Tivoli Netcool/OMNIbus は、IPv4 と IPv6 を両方ともサポートします。 IPv4 のみの構成、IPv6 のみの構成、または IPv4 と IPv6 のデュアル構成をサポートするネットワーク上にコンポーネントを共存させ、動作させることができます。

Tivoli Netcool/OMNIbus サーバー・コンポーネントは、IPv6 環境および IPv4 環境で以下のように動作します。
  • ObjectServer は、IPv4 と IPv6 の両方のネットワークから発生したイベントを処理できます。 ObjectServer がデュアル・スタック・ホスト上で実行されている場合、コマンドへの応答としてクライアントに戻されるホスト名は、クライアントが実行している IP バージョンに対応するサーバーのホスト名です。 例えば、IPv4 を実行しているクライアントは ObjectServer の IPv4 ホスト名を受け取り、IPv6 を実行しているクライアントは ObjectServer の IPv6 ホスト名を受け取ります。
  • IPv4 と IPv6 のデュアル環境では、単一方向および双方向の ObjectServer ゲートウェイが、通信ソケット上で両方のインターフェースを listen できます。
  • プロキシー・サーバーは、IPv4 と IPv6 のホスト上で実行されているプローブと ObjectServer の間の接続をサポートできます。
  • IPv4 と IPv6 のデュアル環境では、プロセス・エージェントが、通信ソケット上で両方のインターフェースを listen できます。
以下の IPv6 アドレス・フォーマットがサポートされます。
  • 4 桁の 16 進文字のグループ 8 個を、コロンで区切ります。例えば、ABCD:EF01:2345:6789:ABCD:EF01:2345:6789 です。
  • 16 ビットすべてがゼロである場合は、そのセグメントをコロン (:) に置き換えることができます。

    例えば、アドレス 1010:0000:0000:0000:ABCD:EF01:2345:6789 は 1010::ABCD:EF01:2345:6789 に書き換えることができます。

  • IPv4 アドレスは IPv6 アドレスとして表すことができます。 例えば、以下のようになります。
    • 0:0:0:0:0:0:192.101.50.5
    • 0:0:0:0:0:FFFF:103.27.35.8

      これらのアドレスは、以下のように表すこともできます。

    • ::192.101.50.5
    • ::FFFF:103.27.35.8
