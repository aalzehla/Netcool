IBM Tivoli Netcool/OMNIbus V7.4

为事件列表数据设置过滤器

过滤器用来构建可在 ObjectServer 警报数据库表上运行的 SQL 查询，以便在监视器框和事件列表中显示定制警报信息。

过滤器定义在事件列表中显示具有如下特征的警报：
  • 特定字段具有特定值
  • 特定字段具有与某值相似的值
  • 事件发生在两个固定的时间点之间
  • 事件发生在最多特定分钟数之前
随 Tivoli Netcool/OMNIbus 提供了多个缺省过滤器。 这些过滤器如下：
前 10 分钟。
该过滤器显示除 ProbeWatch、TSMWatch 和 ConnectionWatch 之外的所有警报。 结果与缺省过滤器的结果基本相同，区别在于最后 10 分钟过滤器仅显示最后 10 分钟内发生的警报。
正在维护
该过滤器显示已由操作员指示为“正在维护”的警报。
已升级
该过滤器显示已由操作员升级的警报。
信息
该过滤器显示已标识为“信息”警报的警报。 “信息”警报就是：
  • 已由探测器指定为“信息”警报的警报
  • 除 ProbeWatch、TSMWatch 或 ConnectionWatch 警报之外的警报
  • 已设置升级等级的警报
任务列表
该过滤器显示已添加到操作员任务列表的警报。
Netcool® 状态
该过滤器只显示 ProbeWatch、TSMWatch 和 ConnectionWatch 警报。
缺省值
该过滤器显示除 ProbeWatch、TSMWatch 和 ConnectionWatch 之外的所有警报。
所有事件
该过滤器显示所有警报。

另外，您还可以构建定制过滤器。 过滤器保存为 .elf 文件。 使用过滤器构建器可创建和编辑过滤器。

  • 过滤器组件
    过滤器由一组 元素 组成。 可用元素类型包括条件、逻辑和子查询元素。
  • 过滤器度量
    所有过滤器都有关联度量。 度量是对过滤器应用的度量方法，可用于计算有用的数据。
  • 过滤器的最佳实践
    构建过滤器时，请先执行最小集中测试。
  • 启动过滤器构建器
    过滤器构建器提供了图形界面，可在其中构建和测试组成过滤器的 SQL 语句。
  • 在 UNIX 上创建或编辑过滤器
    使用 UNIX 过滤器构建器可构建事件列表数据的过滤器。
  • 在 Windows 上创建或编辑过滤器
    使用 Windows 过滤器构建器可构建事件列表数据的过滤器。
  • 在 UNIX 上通过拖放创建过滤器
    在 UNIX 上，可以使用拖放功能来创建过滤器。
  • 打开过滤器
    可以打开已保存的过滤器，以便查看或编辑其定义。
