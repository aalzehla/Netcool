IBM Tivoli Netcool/OMNIbus V7.4

处理事件

要处理事件列表内的事件，必须先选择事件。

可以使用事件列表来执行以下任务：
  • 应答、取消应答、划分优先级、删除以及解决一个或多个选定事件
  • 将所有权分配给一个或多个选定事件
  • 刷新、冻结以及打印事件列表
  • 搜索特定事件的事件列表
  • 应用过滤器和视图
  • 启动过滤器构建器并编辑过滤器，或启动视图构建器并编辑视图
  • 限制显示的行数
  • 选择和取消选择事件
    要处理事件列表内的一个或多个事件，您必须首先选择事件。 然后您可以使用警报菜单中的选项来操作事件。 您还可以从选定事件的列表中取消选择一个或多个事件。
  • 应答和取消应答事件
    可以在事件列表内应答和取消应答事件，但仅当您有许可权时才能进行此操作。
  • 划分事件的优先级
    可以通过更改事件严重性，升级或或取消升级事件，禁止事件，隐藏事件以及指示事件正在“维护中”来划分事件列表中事件的优先级。
  • 将所有权分配给事件
    事件从探测器到达时，除非已将事件的所有权分配给了特定用户，否则此事件的所有者为 nobody 用户。 具有相应特权的用户可以在整个问题生命周期内取得事件的所有权，或将事件分配给特定的用户或组。
  • 刷新事件列表
    事件列表将定期自动刷新，以显示所有传入警报。 可以选择在配置的时间间隔之间刷新事件列表，以及时查看当前时刻的所有最新警报。
  • 冻结事件列表
    可以冻结任何已打开的事件列表，这样就不会显示 ObjectServer 中的更新。
  • 解决事件
    从事件列表可显示已知解决方案的列表，它们由事件的类确定。
  • 删除事件
    仅当您有许可权时才可删除事件。
  • 完整地显示事件信息
    从事件列表可以查看任何选定事件的完整详细信息。 存储于 alerts.status、alerts.details 以及 alerts.journal 数据库表内的事件信息将会显示。
  • 将事件添加至任务列表
    可以使用任务列表来确定需要特别注意的事件。 将事件添加至与“任务列表”事件列表过滤器结合使用的任务列表。
  • 查看相关事件
    事件列表包含了可帮助您在网络上查找相关事件的工具。 您可以查看与选定事件相关的各类事件。
  • 限制显示的行数
    系统管理员可以限制您能在事件列表内看到的行数，并使您能够在定义范围内更改显示的行数。 这些许可权是在视图构建器内配置的。
  • 维护事件的日志
    Tivoli Netcool/OMNIbus 提供了日志工具，可以在其中输入和存储事件历史信息。 可以维护事件列表内任何事件的日志。
  • 搜索事件列表数据
    可以通过多种方法来搜索事件列表数据。 可以输入要搜索的特定文本，使用过滤工具来快速查找数据匹配，以及按严重性来过滤事件列表数据。
  • 自动查看对事件列表的更新
    更新发生时可控制事件列表的滚动方式。 “跳转至顶部”功能可帮助您快速查看已更新的事件。
  • 从事件列表中复制数据
    可以将事件数据从事件列表复制到剪贴板，以便在其他应用程序内使用。
  • 更改显示的事件信息
    可以通过编辑事件列表视图，或通过选择不同的视图来更改在当前事件列表内显示的列。 也可以编辑当前事件列表所使用的过滤器条件，或为事件列表选择不同的过滤器。
  • 打印事件列表的内容
    可以打印事件列表的当前视图。 UNIX 和 Windows 有不同的打印选项。 在 UNIX 上，您可以控制输出的格式。
  • 使用工具来管理事件
    系统管理员可以定义用于处理事件列表中事件的工具，并可将这些工具添加到 Tivoli Netcool/OMNIbus 桌面菜单以供您使用。 可以使用这些工具对 ObjectServer 运行 SQL 命令，并可运行诸如本地应用程序批处理文件或脚本之类的外部命令。
