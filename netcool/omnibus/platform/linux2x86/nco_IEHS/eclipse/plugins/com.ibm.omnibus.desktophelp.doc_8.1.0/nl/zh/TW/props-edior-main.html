IBM Tivoli Netcool/OMNIbus 7.4 版

內容編輯器

使用此視窗可以編輯 ObjectServer、Proxy 伺服器、資料庫起始設定公用程式 (nco_dbinit)、Confpack 公用程式 (nco_confpack)、Netcool/OMNIbus Administrator (nco_config)、探測器及閘道的內容檔中所含的配置資訊。您也可以在 Windows 上建立新的內容檔。此功能僅可用於超級使用者。

如下所示完成此視窗：
類型
在 UNIX 上，選取下列任一選項：
  • 選取一般，以編輯探測器以外的元件內容檔。 檔案下拉清單會移入這些檔案。
  • 選取探測器以編輯探測器內容檔。 檔案下拉清單會移入這些檔案。
在 Windows 上，選取下列任何選項：
  • 選取 ObjectServer 以編輯元件的內容檔，例如 ObjectServer 和 Proxy 伺服器。 名稱下拉清單會移入這些檔案。 這些檔案一般會儲存在 %NCHOME%\omnibus\etc 目錄中。
  • 選取探測器以編輯探測器內容檔。 名稱下拉清單會移入這些檔案。 這些檔案一般會儲存在 %NCHOME%\omnibus\probes\win32 目錄中。
  • 選取伺服器以新增內容檔。 這些檔案會儲存在 %NCHOME%\omnibus\ini 目錄中。 名稱下拉清單會移入文字 <New File>。
檔案 (UNIX)/名稱 (Windows)
從此下拉清單選取您要編輯的內容檔。 基於從類型下拉清單進行的選取，資料會移入此清單。 預設值為：
  • nco_dbinit.props：資料庫起始設定開始時要讀取的內容檔 (執行 nco_dbinit 以起始設定資料庫。)
  • NCO_PROXY.props：Proxy 伺服器內容檔 (Proxy 伺服器可用於減少與 ObjectServer 的探測器連線數目。)
  • nco_confpack.props：Confpack 內容檔 (nco_confpack 用於匯入及匯出 ObjectServer 配置。)
  • nco_config.props：Netcool/OMNIbus Administrator內容檔
  • NCOMS.props：ObjectServer 內容檔
在 Windows 上，不會顯示 .props 副檔名，而且當您在名稱清單中選取檔案時，內容清單中會自動移入可用的內容。
開啟 (僅限 UNIX)
在 UNIX 上，於選取類型及檔名之後，按一下此按鈕以開啟內容檔。
內容清單
此矩陣會顯示已新增至內容檔之任意內容的詳細資料。 如果您要變更內容，則可以選取此清單中的內容。 有一組已為探測器內容檔定義的預設內容組，且其無法移除。 這些內容會在內容清單中以灰色顯示。
名稱
此欄位會顯示已從內容清單中選取，要進行編輯的內容名稱。 您無法變更現有內容的名稱。 在 Windows 上，您也可以使用下拉清單來選取要編輯的內容。
如果您要將新內容新增至選取的內容檔，請在此處輸入內容名稱。
如果已從類型清單選取伺服器，以建立新的內容檔，請在此處輸入新內容名稱。
值
如果您正在使用現有且已選取的內容檔，請編輯所選取內容的值，或輸入您要新增之新內容的值。
如果已從類型清單選取伺服器，以建立新的內容檔，請輸入新內容的值。
新增/更新
此按鈕會視您是否要新增或編輯現有的內容而改變。 如果輸入新的內容名稱及值，請按一下新增，以將值新增至內容清單。 如果變更內容清單中所選取內容的值，請按一下更新，以更新內容清單中的值。
如果已從類型清單選取伺服器，以建立新的內容檔，請按一下新增以新增內容。
移除
按一下此按鈕，以從內容檔移除所選內容。 如果選取預設內容，則無法使用此按鈕。
儲存
按一下此按鈕，以將變更儲存至內容檔。
如果已從類型清單選取伺服器，以建立新的內容檔，請在將內容新增至內容清單之後，按一下儲存。 在「儲存新內容」視窗上，輸入內容檔的名稱，但請省略 .props 副檔名；這會自動為您新增。 按一下儲存以將檔案儲存至 %NCHOME%\omnibus\ini 目錄中。 內容檔名稱會顯示在名稱欄位中 (這個欄位與類型欄位相鄰)，然後您可以繼續將內容新增至此檔案。 請注意，您需要編輯此檔案，以將引號新增至針對內容指定的字串值。
關閉 (UNIX)
按一下此按鈕，以在儲存變更之後關閉此視窗，或關閉此視窗而不儲存變更。
確定 (Windows)
按一下此按鈕，以在儲存變更之後關閉此視窗。
取消 (Windows)
按一下此按鈕，以關閉此視窗而不儲存變更。
