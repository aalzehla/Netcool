Windows 事件清單指令行選項

當您使用 NCOEvent.exe 指令，在 Windows 上啟動事件清單時，可以指定一些指令行選項來修改配置。您可以從 %NCHOME%\omnibus\desktop 目錄執行此指令。

下表說明用於啟動 Windows 事件清單的指令行選項。

表 1. Windows 事件清單 (NCOEvent) 指令行選項
指令行選項 說明
-config string | URL

或

-elc string | URL

 指定要使用的起始配置檔，其中：

string 指定 .elc 檔案的路徑。

URL 指定位於遠端伺服器上的 .elc 檔案，其可使用 HTTP 或 FTP 進行存取。 透過使用下列其中一種格式指定 URL：

  • http://servername[:port]/path/filename.elc
  • http://username:password@servername[:port]/path/filename.elc
  • ftp://servername[:port]/path/filename.elc
  • ftp://username:password@servername[:port]/path/filename.elc

僅當伺服器未使用預設埠時，才需要指定選用埠號 :port。

您可以針對 servername 值選擇性地指定 IPv4 或 IPv6 位址。 當指定 IPv6 位址時，請將位址含括在方括弧 [ ] 內；例如：

http://[2003:13f8:206:190:204:baef:fe6a:6af0]:2880/mydir/myfile.elc

如果使用 FTP，路徑必須是從根目錄開始的完整路徑；例如，/home/user/mydir/myconfig.elc。
-diagnostic 顯示延伸的診斷訊息。

此選項僅用於偵測事件清單的問題。 除非 IBM 軟體支援中心 建議，否則請勿使用此指令行選項。
-dualwrite 0 | 1 指出在桌面 ObjectServer 架構中運作時，是啟用還是停用雙重寫入模式。 有效的選項如下所示：
  • 0：停用雙重寫入模式
  • 1：啟用雙重寫入模式

-dualwrite 指令行選項會置換桌面 ObjectServer 之 master.national 表格中的 DualWrite 欄位項目。
-failbackpolltime integer 如果因發生失效接手而連接至備份 ObjectServer，事件清單會定期嘗試重新連接至主要 ObjectServer。 此指令行選項可指定事件清單輪詢主要 ObjectServer 傳回結果的頻率（以秒為單位）。如果偵測到主要 ObjectServer 已重新啟動，事件清單會從備份 ObjectServer 中斷連線，並重新連接至主要 ObjectServer。預設值為 30 秒。 設定為 0 時，將不會進行輪詢。

事件清單在連接至 ObjectServer 時，會檢查正在連接的 ObjectServer 的 BackupObjectServer 內容設定。僅在此內容設定為 TRUE，表示此為備份 ObjectServer 時，才會進行輪詢。
-help 顯示此指令的說明並結束。
-masterserver string 指出要在桌面 ObjectServer 架構中使用的主要 ObjectServer。

-masterserver 指令行選項會置換桌面 ObjectServer 之 master.national 表格中的 MasterServer 欄位項目。
-networktimeout integer 指定以秒為單位的時間，如果發生網路失敗，則在這個時間之後，ObjectServer 的登入嘗試或連線會逾時。 在指定的逾時期間之後，事件清單會嘗試重新連接至 ObjectServer。 如果連線在第二個逾時期間之後仍不成功，則事件清單會嘗試連接至備份 ObjectServer (如果可用)。 預設值為 20 秒。
-password string 指定要用於登入事件清單的密碼。
-server string 指定正連接至的 ObjectServer 名稱。
-username string 指定要用於登入事件清單的使用者名稱。
-version 顯示版本資訊並結束。
