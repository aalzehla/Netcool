IBM Tivoli Netcool/OMNIbus, 버전 7.4

필터 빌더

이 창에서 이벤트 목록의 경보를 필터링하기 위해 SQL 쿼리(또는 WHERE 절)를 정의하십시오. 필터 빌더를 사용하여 이벤트 데이터에 대해 이 필터를 설정하고 유지보수할 수 있습니다.

필터 빌더는 다음 영역으로 구성됩니다.
  • 필터 설정 영역(맨 위 패널)
  • 단추 막대(필터 설정 영역 아래의 패널)
  • 그래픽 디스플레이 영역(단추 막대 아래의 맨 위 왼쪽 패널)
  • 요소 정의 영역(단추 막대 아래의 맨 위 오른쪽 패널)
  • SQL 디스플레이 영역(맨 아래 왼쪽 패널)
  • 지표 영역(맨 아래 오른쪽 패널)

필터 설정 영역

이 영역을 사용하여 이름을 정의하고 필터에 대한 권한을 설정하십시오.

다음과 같이 이 영역을 완료하십시오.

이름
필터를 새로 작성할 경우 현재 이름을 필터의 고유 이름으로 겹쳐쓰기합니다. 이 이름은 이벤트 목록 메뉴에서 필터를 선택하기 위해 사용됩니다. 기존 필터를 편집할 경우 이 필드에 항목을 작성할 필요가 없습니다.
편집 가능
해당 관리자 권한이 있는 경우 기타 사용자가 이 필터를 편집하도록 허용하려면 이 선택란을 체크하고 필터가 수정되지 못하게 하려면 이 선택란을 체크하지 마십시오.
제한사항: 이 설정은 필터가 이벤트 목록 구성(.elc 파일)의 일부로 저장된 경우에만 적용됩니다.

단추 막대

이 단추를 사용하여 필터 요소를 정의하십시오.

다음과 같이 이 영역을 완료하십시오.

조건
조건 요소를 작성하려면 이 단추를 클릭하십시오. 조건 요소를 사용하여 필드와 값을 비교할 수 있습니다. 조건 단추는 WHERE 절을 작성하고 값 스펙을 허용합니다(예: where Node = 'link1').
부정
현재 선택되어 있는 요소 앞에 논리 NOT 연산자를 삽입하려면 이 단추를 클릭하십시오.
선행 논리적
현재 선택되어 있는 요소 앞에 논리 요소를 삽입하려면 이 단추를 클릭하십시오. 자세한 복합 비교를 위해 논리 요소를 사용하여 두 요소를 결합할 수 있습니다.
후미 논리적
현재 선택되어 있는 요소 다음에 논리 요소를 삽입하려면 이 단추를 클릭하십시오.
하위 조회
조건 범위 내에서 조회를 작성하려면 이 단추를 클릭하십시오. 하위 조회는 값을 다른 SELECT 문의 결과와 일치시키기 위해 사용할 수 있는 추가 복합 조건입니다.
요소 삭제
현재 선택되어 있는 요소를 삭제하려면 이 단추를 클릭하십시오.
트리 삭제
현재 요소 및 해당 하위 요소를 모두 삭제하려면 이 단추를 클릭하십시오.
SQL 편집
필터 SQL 편집 창을 열어 SQL을 직접 편집하려면 이 단추를 클릭하십시오. 확인을 클릭하여 저장하고 필터 빌더로 돌아가면 SQL 표현식이 자동으로 구문 분석되고 그래픽 디스플레이 영역의 조건 요소에 추가됩니다. 필터 SQL 편집 창에 올바르지 않은 SQL 텍스트를 입력하는 경우 창을 종료하기 전에 구문을 정정해야 합니다.
이 단추 중 어느 하나를 클릭하면 다음과 같습니다.
  • 그래픽 표시가 단추 막대 아래에 있는 맨 위 왼쪽 패널의 그래픽 디스플레이 영역에 표시됩니다.
  • SQL 구문이 맨 아래 왼쪽 목록 상자의 SQL 디스플레이 영역에 표시됩니다.
  • 요소 특정의 필드가 오른쪽 패널의 요소 정의 영역에 표시됩니다.
참고: 이 단추의 가용성은 현재 선택되어 있는 요소 및 해당 요소에 대한 올바른 조작에 따라 결정됩니다.

그래픽 디스플레이 영역

이 영역은 요소의 관련성이 표시된 그래픽 트리 구조에서 현재 필터에 대한 요소 정의를 표시합니다. 단추 막대에 있는 단추를 사용하여 트리에 요소를 추가할 수 있습니다. 현재 선택되어 있는 요소는 주위에 적색 경계가 표시됩니다. 편집은 현재 요소만 가능합니다. 이 영역 내에서 요소를 선택하면 관련 세부사항이 표시되고 요소 정의 영역에서 편집할 수 있습니다.

요소 정의 영역

이 영역은 현재 선택되어 있는 요소의 세부사항을 표시하고 다른 값 세트를 지정할 수 있게 합니다. 선택사항은 그래픽 디스플레이 영역 및 SQL 디스플레이 영역에 반영됩니다.

조건 단추를 클릭하여 조건 요소를 작성했으면 다음을 참조하여 필드를 완료하십시오.
표현식 유형
이 목록에서 다음 옵션 중 하나를 선택하십시오.
  • 단순: 조건 요소의 단순 표현식을 사용하려면 이 옵션을 선택하십시오.
  • 복합: 두 개의 데이터베이스 컬럼 또는 두 개의 표현식을 비교할 수 있는 복합 표현식을 사용하려면 이 옵션을 선택하십시오.
  • 자유 형식: ObjectServer SQL의 고급 기능 사용을 허용하는 표현식을 지정하려면 이 옵션을 선택하십시오.
컬럼
이 목록에서 비교에 사용하려는 데이터베이스 컬럼을 선택하십시오.
목록에는 alerts.status 데이터베이스 테이블의 열 이름이 기본 값으로 포함됩니다. 필터 빌더는 사용 가능한 컬럼을 자동으로 판별하며, 이 목록은 상황에 따라 다릅니다.
참고: 표현식 유형 목록에서 자유 형식 옵션을 선택한 경우 컬럼 목록이 표시되지 않습니다.
연산자
이 목록에서 비교 연산자를 선택하십시오.
사용 가능한 비교의 범위는 컬럼 드롭 다운 목록의 선택사항에 따라 결정됩니다. 특정 컬럼에 대해 사용 불가능한 비교도 있습니다. 예를 들어, 숫자 컬럼(예: 심각도)에 대한 LIKE 연산 수행은 가능하지 않습니다.
참고: 표현식 유형 목록에서 자유 형식 옵션을 선택한 경우 연산자 목록이 표시되지 않습니다.
값 데이터 유형 필드
여기에 표시되는 필드는 표현식 유형 목록에서 선택한 옵션에 따라 다릅니다.
표현식 유형이 단순인 경우 표시되는 값 필드는 컬럼 목록에서 선택된 데이터베이스 컬럼이 문자열, 정수 또는 시간 중 어느 유형인지에 따라 결정됩니다.
표현식 유형이 복합 또는 자유 형식인 경우 편집 단추가 필터 SQL 편집 창에서 SQL을 직접 편집할 수 있게 하기 위해 제공됩니다. 확인을 클릭하여 저장하고 필터 빌더로 돌아가면 SQL 표현식이 자동으로 구문 분석되고 그래픽 디스플레이 영역의 조건 요소에 추가됩니다. 필터 SQL 편집 창에 올바르지 않은 SQL 텍스트를 입력하는 경우 창을 종료하기 전에 구문을 정정해야 합니다.
선행 논리적 또는 후미 논리적 단추를 클릭하여 논리 요소를 작성했으면 다음을 참조하여 필드를 완료하십시오.
논리적 AND
AND 연산자를 필터 정의에 추가하려면 이 옵션을 선택하십시오. 이것은 기본 옵션입니다.
논리적 OR
OR 연산자를 필터 정의에 추가하려면 이 옵션을 선택하십시오.
하위 조회 단추를 클릭하여 하위 조회 요소를 작성했으면 다음을 참조하여 필드를 완료하십시오.
컬럼
이 목록에서 검색에 사용할 데이터베이스 컬럼을 선택하십시오.
연산자
목록에서 다음 옵션 중 하나를 선택하십시오.
  • 속함: 컬럼의 내용을 검색하려면 이 옵션을 사용하십시오.
  • 속하지 않음: 내용이 없는 필드를 검색하려면 이 옵션을 사용하십시오.
선택
목록 빌드 시 속함 또는 속하지 않음 작업을 수행할 때 대조용으로 사용할 컬럼을 선택하십시오. 이 목록의 옵션은 원본 목록에서 선택되는 옵션에 따라 판별됩니다.
원본
정보를 얻을 데이터베이스 테이블을 선택하십시오. 이는 alerts.details, alerts.journal 또는 alerts.status가 될 수 있습니다.

SQL 디스플레이 영역

SQL 디스플레이 영역은 필터를 빌드함에 따라 SQL을 표시합니다. 단추 막대에서 SQL 편집을 클릭하여 필터 SQL 편집 창에서 SQL을 직접 편집할 수 있습니다.

올바른 SQL 구문을 사용하여 필터 조회를 입력하고 확인을 눌러서 입력을 저장한 후 필터 빌더로 돌아가십시오. 필터 빌더는 SQL 텍스트를 구문 분석하여 필터 트리를 빌드하려고 시도합니다. SQL이 올바르지 않은 경우 오류 메시지가 표시되며, 계속하려면 우선 필터 SQL 편집 창에서 오류를 정정해야 합니다.

지표 영역

필터에 측정법을 적용하여 유용한 계수를 계산하게 하려면 지표 단추를 사용하십시오. 예를 들어, Sum of Tally 지표는 ObjectServer의 총 비중복 경보 수를 제공합니다.

다음과 같이 이 영역을 완료하십시오.
지표
첫 번째 목록을 사용하여 사용할 측정법을 선택하십시오.
  • 필터와 일치하는 모든 경보에 대해 선택된 필드의 평균 값을 리턴하려면 평균을 선택하십시오.
  • 필터와 일치하는 모든 경보의 수를 리턴하려면 수를 선택하십시오. 선택된 필드는 이 계산에 사용되지 않습니다.
  • 필터와 일치하는 모든 경보에 대해 선택된 필드의 합계를 리턴하려면 합계를 선택하십시오.
  • 필터와 일치하는 경보에서 선택된 필드의 가장 낮은 값을 리턴하려면 최소를 선택하십시오.
  • 필터와 일치하는 경보에서 선택된 필드의 가장 높은 값을 리턴하려면 최대를 선택하십시오.
두 번째 목록을 사용하여 측정법이 적용되는 필드를 선택하십시오. 경보에서 정수 및 시간 필드만 지표 계산에 사용 가능합니다.
팁: 지표 값은 이벤트 목록 모니터 상자 창의 모니터 상자에 표시됩니다.

필터 설정 적용 및 저장

필터 설정을 정의 또는 수정한 후 다음과 같이 해당 단추를 클릭해 작성된 변경사항을 적용, 저장 또는 취소할 수 있습니다.
적용
변경사항을 저장하지 않고 필터에 적용하려면 이 단추를 클릭하십시오.
닫기
창을 닫고 변경사항을 버리려면 이 단추를 클릭하십시오.
필터는 .elf 파일 확장자로 저장해야 합니다.
참고: UNIX에서 필터 빌더가 컨덕터에서 실행 중인 경우 적용 단추가 표시되지 않습니다. 이 경우에는 UNIX 필터 빌더에서 파일 > 저장 또는 파일 > 다른 이름으로 저장을 선택하여 필터를 모두 저장해야 합니다.
관련 태스크:
UNIX에서 필터 작성 또는 편집
필터 조건에 대한 연산자 지정
필터 조건에 대한 비교 값 지정
