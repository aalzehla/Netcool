フィルターおよびビューを使用したカスタム・イベント情報の表示

alerts データベース表には、ObjectServer に転送されるイベント情報が保持されます。 フィルターおよびビューを使用して、イベント・リストに表示されるこのイベント情報を 定義することができます。

フィルターは表示されるアラート (または、データベース・レコード) を制御し、 ビューはこれらのアラートに表示される列 (または、データベース・フィールド) を制御します。 構造化照会言語 (SQL) を使用して、データベース表の情報要求を実行することができます。

イベント・リスト内にアラートを表示するには、alerts.status 表上で SELECT ステートメントを実行する必要があります。 SQL ステートメントは、以下の 3 つの 概念部分に分けられます。
  • ビュー・ビルダー内で作成されたビューを使用して、表示する alerts.status 表内の列を選択する。
  • フィルター・ビルダー内で作成されたフィルターを使用して、これらの 1 つ以上の列内にある固有のデータ値を含むイベントを選択する。
  • ビュー・ビルダー内で作成されたビューを使用して、イベントを表示する 順序を選択する。

alerts.status 表上で実行される SELECT ステートメントの例を以下に示します。 

select Node as NODE, Summary as Detail from alerts.status
where Node ='link1'

SELECT 節はビュー・ビルダー内で構成され、ビューの作成および編集に 使用されます。 WHERE 節はフィルター・ビルダー内で構成され、フィルターの作成および編集に 使用されます。

  • イベント・リスト・データのフィルターのセットアップ
    フィルターは、モニター・ボックスおよびイベント・リストにカスタム・アラート情報を表示するために、ObjectServer alerts データベース表上で実行することができる SQL 照会を構築するのに使用されます。
  • イベント・リストのビュー設定
    ビューでは、イベント・リストに表示される列の選択およびこれらの列内の情報のフォーマットが定義されます。
