IBM Tivoli Netcool/OMNIbus V7.4

配置服务器通信信息

必须维护 Tivoli Netcool/OMNIbus 服务器组件的服务器通信信息。 可以使用服务器编辑器来创建和修改服务器组件的通信详细信息，测试服务器活动，以及添加备份（或故障转移）服务器和侦听器。

要配置服务器通信信息：
  1. 为操作系统执行相应的操作：
    操作系统 操作
    UNIX 执行以下任一操作：
    • 单击 UNIX Conductor 上的界面按钮。
    • 在命令行上输入 $NCHOME/omnibus/bin/nco_xigen。
    Windows 执行以下任一操作：
    • 单击 开始 > 程序 > Netcool Suite > 系统实用程序 > 服务器编辑器。
    • 通过右键单击桌面托盘内的总线图标打开 Conductor 菜单，然后单击服务器。
    将打开服务器编辑器。
  2. 如下所示完成此窗口：
    服务器列表
    服务器列表显示现有的服务器组件及其设置：
    Server
    此列显示为此工作站定义的每个服务器的名称。 缺省服务器名称为：
    • NCOMS：ObjectServer 的缺省名称
    • NCO_PROXY：代理服务器的缺省名称
    • NCO_GATE：网关的缺省名称
    • NCO_PA：进程代理程序的缺省名称
    在 Windows 上，服务器的主机工作站必须具有侦听器项，主机工作站和任何其他连接到服务器的工作站必须具有客户机项。
    备份 ObjectServer 在服务器编辑器中显示在主控 ObjectServer 之下。 要隐藏备份 ObjectServer 列表，请双击主控 ObjectServer 名称或服务器图标。 备份 ObjectServer 将被隐藏，服务器图标中将显示 C 字母。 再次双击可显示备份 ObjectServer。 服务器图标返回到正常状态。
    主机名
    此列显示安装了服务器组件的工作站的主机名或 IP 地址。
    注： 安装后，对于（UNIX 上的）所有服务器，主机都将设置为 omnihost。 您必须将其更改为服务器主机工作站的名称。 在 Windows 上，缺省值为主机名。
    端口
    此列显示服务器组件侦听未加密连接的端口。
    SSL
    在 UNIX 上， 此列显示服务器组件侦听加密连接的端口。 在 UNIX 系统上，可定义服务器组件的标准端口和/或 SSL 端口。
    在 Windows 上，此列对加密连接显示是。 在 Windows 系统上，加密和未加密连接使用相同的端口号。
    服务器项字段
    使用服务器项字段可输入或编辑每个服务器组件的详细信息。 如果您要编辑现有服务器组件的详细情况，您必须首先在服务器列表中选择相应的行，以便在服务器项字段中显示详细情况。
    名称
    输入新服务器组件的名称或编辑现有组件的名称。 在 Windows 上，您也可以使用下拉列表选择名称。 为组件命名时使用以下后缀：_PROXY（代理服务器）、_GATE（网关）和 _PA（进程代理程序）。
    注： 服务器项的名称必须包含不超过 29 个大写字母，且不能以整数开头。
    主机
    输入或编辑安装了服务器组件的工作站的主机名或 IP 地址。 （对于 UNIX 上的新服务器组件，缺省情况下名称将设置为 omnihost，并且必须更改为实际的主机名或 IP 地址。）
    如果输入 IP 地址，那么可以指定 IPv4 或 IPv6 地址。 For example:
    • 192.168.0.1
    • 2094:82a:2a6e:123:503:badd:fe43:f552
    端口
    在 UNIX 上， 要让使用未加密连接的客户机能够连接到服务器上，请在此字段中输入有效的、未使用过的端口号。 要禁用未加密连接，请勿设置此端口。
    在 Windows 上，在此字段中输入有效的、未使用过的端口号。
    SSL
    在 UNIX 上， 要让使用加密连接的客户机能够连接到服务器上，请在此字段中输入有效的、未使用过的端口号。 要禁用加密连接，请勿设置此端口。
    在 Windows 上，选此复选框表示端口接受从客户机发出的使用 SSL 的加密连接。
    注： 在 UNIX 系统上，服务器组件可能具有已定义的标准端口或/和 SSL 端口。 在 Windows 系统上，两种连接使用相同的端口号。
    侦听器（仅限 Windows）
    如果这是服务器主机工作站上的侦听器项，请选此复选框。 如果这是客户机项，请清除侦听器复选框。
    添加
    单击此按钮可向服务器列表添加新的（且唯一的）服务器详细信息。
    除去/更新
    此按钮将根据您是否对当前选定的服务器组件做出了更改而发生变化。 如果要从服务器列表除去当前选定的服务器组件，请单击除去。 单击更新可使用现有服务器组件的更新详细信息来刷新服务器列表。
    测试
    单击此按钮可测试您是否能连接到在服务器列表中选择的服务器。 服务器编辑器将尝试连接指定主机和端口上的服务器。 测试命令的结果将在窗口中显示。
    优先级字段
    服务器优先级字段可使您能够提高或降低已配置为故障转移系统的服务器组件的优先级。 例如，假设名为 NCOMS_PRI 的 ObjectServer 是使用名为 NCOMS_BAK 的备份配置的。 通过使用提高或降低按钮，您可提高 NCOMS_BAK 的优先级使之成为主控 ObjectServer，并降低 NCOMS_PRI 的优先级使之成为备份 ObjectServer。
    提高
    单击此按钮可将所选服务器组件的优先级提高一级。
    降低
    单击此按钮可将所选服务器组件的优先级降低一级。
    生成全部（限 UNIX）
    单击此按钮可为所有 UNIX 操作系统生成接口文件。 当你应用更改（使用应用按钮）时，这将生成名为 $NCHOME/etc/interfaces.arch 的接口文件，其中 arch 代表各个 UNIX 平台名称；如，interfaces.hpux11 和 interfaces.solaris2。
    显示组（仅限 Windows）
    在 Windows 上，选此复选框可在服务器列表中按名称对每个服务器分组，同时在客户机项后标识备份服务器和侦听器。
    应用（仅限 UNIX）
    单击此按钮对接口文件应用更改。 任何向服务器列表添加、从服务器列表除去或被编辑的服务器组件将保存到此文件中。
    确定（仅限 Windows）
    单击此按钮可保存更改并关闭本窗口。
    关闭（仅限 UNIX）/取消（仅限 Windows）
    单击此按钮可在使用应用按钮保存更改后关闭窗口，或不保存更改就关闭窗口。
    导入（仅限 UNIX）
    单击此按钮可导入通信详细情况。
