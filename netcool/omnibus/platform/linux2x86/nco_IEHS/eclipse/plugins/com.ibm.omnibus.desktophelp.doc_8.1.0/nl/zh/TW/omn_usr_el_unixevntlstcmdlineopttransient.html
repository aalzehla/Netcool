暫時性事件清單的 UNIX 指令行選項

當您執行 nco_elct 指令，在 UNIX 上啟動暫時性事件清單時，必須指定一些指令行選項來自訂事件清單。您可以從 $NCHOME/omnibus/bin 目錄執行此指令。

下表說明用於啟動 UNIX 上暫時性事件清單的指令行選項。

表 1. UNIX 暫時性事件清單 (nco_elct) 指令行選項
指令行選項 說明
-cmap 指定專用色彩對映。 您無法縮寫此指令行選項。
-diagnostic 顯示延伸的診斷訊息。
-dualwrite 1 | 0 指出在桌面 ObjectServer 架構中運作時，是啟用還是停用雙重寫入模式。 有效的選項如下所示：
  • 0：停用雙重寫入模式
  • 1：啟用雙重寫入模式

-dualwrite 指令行選項會置換桌面 ObjectServer 之 master.national 表格中的 DualWrite 欄位項目。
-ffile string | URL 指定要用於暫時性事件清單的過濾器檔案。 事件清單過濾器的副檔名為 .elf。
註： 您也可以使用 HTTP 及 FTP 來擷取遠端 .elf 檔案。 在執行 nco_event 指令以啟動 UNIX 事件清單時指定 -config 指令行選項，即可完成此作業。
-fmetric string 指定用於暫時性事件清單的過濾器度量。 如果您已使用 -ffile 指令行選項指定現有的事件清單過濾器檔案，則可忽略此設定。
-fname string 指定過濾器的名稱。 如果您已使用 -ffile 指令行選項指定現有的事件清單過濾器檔案，則可忽略此設定。
-ftext string 指定要用於過濾事件的 SQL 陳述式。 如果您已使用 -ffile 指令行選項指定現有的事件清單過濾器檔案，則可忽略此設定。
-help 顯示此指令的說明並結束。
-masterserver string 指出要在桌面 ObjectServer 架構中使用的主要 ObjectServer。

-masterserver 指令行選項會置換桌面 ObjectServer 之 master.national 表格中的 MasterServer 欄位項目。
-networktimeout integer 指定以秒為單位的時間，如果發生網路失敗，則在這個時間之後，ObjectServer 的登入嘗試或連線會逾時。 在指定的逾時期間之後，事件清單會嘗試重新連接至 ObjectServer。 如果連線在第二個逾時期間之後仍不成功，則事件清單會嘗試連接至備份 ObjectServer (如果可用)。 預設值為 20 秒。
-password string 指定所選使用者名稱的密碼。 nco_elct 公用程式僅接受在密碼中使用英數字元。 特殊字元（如 #、/、>、@ 及 !）不允許在密碼中使用。
-server string 指定正連接至的 ObjectServer 名稱。
-username string 指定要用於登入事件清單的使用者名稱。
-version 顯示版本資訊並結束。
-vfile string | URL 透過使用指定的事件清單視圖，開啟暫時性事件清單。 事件清單視圖的副檔名為 .elv。
註： 您也可以使用 HTTP 及 FTP 來擷取遠端 .elv 檔案。 在執行 nco_event 指令以啟動 UNIX 事件清單時指定 -config 指令行選項，即可完成此作業。
提示： 您可以將暫時性事件清單的指令行選項縮寫為其最短的唯一縮寫。 例如，-ffile 可縮寫為 -ff。
