IBM Tivoli Netcool/OMNIbus, 버전 7.4

이벤트 목록에서 예측 이벤트 모니터링

예측 이벤트는 임박한 임계값 위반에 대한 예측을 나타냅니다. 이 이벤트는 발생, 결함 또는 아직 발생하지 않았지만 가까운 미래에 발생 가능성이 높은 상황을 나타내기 때문에 Tivoli Netcool/OMNIbus로 전달되는 기타 이벤트와 다릅니다.

예측 이벤트는 히스토리 데이터 선형추세를 기반으로 IBM Tivoli Monitoring에서 생성됩니다. Tivoli Netcool/OMNIbus 이벤트 목록에서 예측 이벤트를 모니터할 수 있습니다(이 기능을 지원하도록 시스템이 구성된 경우). 구성된 경우, 이벤트 목록 모니터 상자 창에서 예측 모니터 상자를 사용할 수 있고 이벤트 목록에서 예측 필터 및 예측 보기를 사용할 수 있습니다. 예측 보기에는 다음 기본 컬럼이 있으며 이들은 왼쪽에서 오른쪽으로 노드, TrendDirection, 요약, FirstOccurrence, LastOccurrence, 계수, PredictionTime과 같이 표시됩니다.

컬럼의 정렬 우선순위 및 정렬 순서는 다음과 같습니다.
  1. 심각도 컬럼은 내림차순으로 정렬됨
  2. LastOccurrence 컬럼은 오름차순으로 정렬됨
  3. PredictionTime 컬럼은 오름차순으로 정렬됨
이벤트 목록이 실제 이벤트와 예측 이벤트를 둘 다 표시하는 경우, 예측 필터 및 예측 보기를 사용하여 예측 이벤트를 필터링하고 정렬할 수 있습니다. 또한 다음과 같이 예측 이벤트를 식별할 수 있습니다.
  • 요약 컬럼의 텍스트를 확인하십시오. 요약 텍스트는 일반적으로 다음 형식 중 하나입니다(치명적 예측 이벤트의 심각도는 항상 5이고, 경고 예측 이벤트의 심각도는 항상 3임).
    • 선형추세 이벤트의 경우:

      Predicted event rate on Node will hit threshold within PredictionTime days.

    • 기준선 설정 이벤트의 경우:

      EventType event rate baseline situation has fired for node Node, the event count is EventCount for node situation_name.

    • 예측 이벤트의 경우:

      Critical threshold for resource on Node will be breached in PredictionTime day(s). (Situation: situation_name).

      Warning threshold for resource on Node will be breached in PredictionTime day(s). (Situation: situation_name).

  • 현재 보기에 클래스 컬럼이 있으면 이 컬럼의 텍스트를 확인하십시오. 기본적으로 해당 항목은 Predictive Events이지만 시스템 관리자가 이 값을 변경할 수 있습니다.

이벤트 목록에서 예측 이벤트에 대해 다음 조작을 수행할 수 있습니다.

  • 예측 신뢰수준을 판별하십시오. 특정 예측 신뢰수준을 충족시키는 예측 이벤트를 표시하려면 다음을 수행하십시오.
    1. 이벤트 목록에서 이벤트를 마우스 오른쪽 단추로 클릭한 후 팝업 메뉴에서 예측 신뢰도 표시를 클릭하십시오.
    2. 내부 명령 매개변수 창의 최소 신뢰도 백분율 필드에 0 - 100 사이의 백분율 값만 지정하십시오. 예측 신뢰도는 대략적인 경향 값이 어느 정도 정확한지 표시합니다(여기서 0은 신뢰도가 없고 100은 완벽에 가까움). 기본값은 50입니다.
    3. 확인을 클릭하십시오. 별도의 이벤트 목록이 열려 지정된 신뢰수준 이상의 예측 이벤트를 표시합니다. 이 이벤트 목록은 기본 보기의 모든 필드를 표시합니다. 이벤트를 추가로 두 번 클릭하면 이벤트 정보 창에서 전체 세부사항 세트를 볼 수 있습니다.
  • 예측이 수행된 호스트에서 시작된 예측 이벤트와 실제 이벤트(사용 가능한 경우)를 모두 나열하십시오. 표시된 결과를 사용하여 경고 또는 치명적 임계값에 가까운 예측 경향의 가능한 원인을 쉽게 식별할 수 있습니다.

    이벤트를 표시하려면 조사할 예측 이벤트를 마우스 오른쪽 단추로 클릭한 후 팝업 메뉴에서 예측 이벤트 임계값 표시를 클릭하십시오. 별도의 이벤트 목록이 열려 선택된 예측 이벤트와 동일한 호스트에 있고 예측이 수행된 시간부터 예측이 발생할 것으로 계산된 시간까지 발생한 이벤트를 모두 표시합니다(기본 보기가 이 이벤트 목록에 적용됨).

    각 이벤트를 두 번 클릭하면 이벤트 정보 창에서 전체 세부사항 세트를 볼 수 있습니다.
