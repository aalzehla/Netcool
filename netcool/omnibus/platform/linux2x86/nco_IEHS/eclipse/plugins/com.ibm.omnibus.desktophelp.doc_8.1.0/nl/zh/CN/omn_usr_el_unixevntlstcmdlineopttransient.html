瞬时事件列表的 UNIX 命令行选项

在 UNIX 上运行 nco_elct 命令启动瞬时事件列表时，必须指定若干命令行选项来定制事件列表。您可从 $NCHOME/omnibus/bin 目录运行此命令。

下表描述了用于在 UNIX 上启动瞬时事件列表的命令行选项。

表 1. UNIX 瞬时事件列表（nco_elct）命令行选项
命令行选项 描述
-cmap 指定专用颜色映射图。 不能缩写此命令行选项。
-diagnostic 显示扩展诊断消息。
-dualwrite 1 | 0 指示在桌面 ObjectServer 体系结构中操作时是启用还是禁用双重写方式。 有效选项如下所示：
  • 0：禁用双写入方式
  • 1：启用双重写方式

-dualwrite 命令行选项将覆盖桌面 ObjectServer master.national 表中的 DualWrite 字段项。
-ffile string | URL 指定要对瞬时事件列表使用的过滤器文件。 事件列表过滤器具有 .elf 扩展名。
注： 也可以通过使用 HTTP 和 FTP 来检索远程 .elf 文件。 可通过在运行 nco_event 命令启动 UNIX 事件列表时指定 -config 命令行选项来执行此操作。
-fmetric string 指定瞬时事件列表的过滤器度量。 如果您使用 -ffile 命令行选项指定了一个现有事件列表过滤器文件，将忽略此设置。
-fname string 指定过滤器名称。 如果您使用 -ffile 命令行选项指定了一个现有事件列表过滤器文件，将忽略此设置。
-ftext string 指定要用来过滤事件的 SQL 语句。 如果您使用 -ffile 命令行选项指定了一个现有事件列表过滤器文件，将忽略此设置。
-help 显示该命令的帮助并退出
-masterserver string 指示桌面 ObjectServer 体系结构中要使用的主控 ObjectServer。

-masterserver 命令行选项将覆盖桌面 ObjectServer master.national 表中的 MasterServer 字段项。
-networktimeout integer 指定发生网络故障时，多少秒后登录尝试或连接 ObjectServer 将超时。 指定的超时时段后，事件列表将尝试重新连接 ObjectServer。 如果超时时段后的连接不成功，事件列表将尝试连接到备份 ObjectServer（如果可用）。 缺省值为 20 秒。
-password string 指定所选用户名的密码。 nco_elct 实用程序仅接受密码中包含字母数字字符。 密码中不允许出现 #、/、>、@ 和 ! 等特殊字符。
-server string 指定要连接的 ObjectServer 的名称。
-username string 指定用于登录事件列表的用户名。
-version 显示版本信息并退出。
-vfile string | URL 使用指定的事件列表视图打开瞬时事件列表。 事件列表视图具有 .elv 文件扩展名。
注： 您也可以使用 HTTP 和 FTP 检索远程 .elv 文件。 可通过在运行 nco_event 命令启动 UNIX 事件列表时指定 -config 命令行选项来执行此操作。
提示： 您可将瞬时事件列表的命令行选项缩写为唯一的最短缩写。 例如，-ffile 可缩写为 -ff。
