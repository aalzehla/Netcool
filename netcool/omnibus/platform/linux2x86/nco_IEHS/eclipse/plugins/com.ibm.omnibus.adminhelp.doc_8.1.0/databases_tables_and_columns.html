Databases, Tables and Columns

Use this pane to view details of all databases and tables that are set up in the selected ObjectServer, and to set up and maintain databases and tables.

Within this pane, all the ObjectServer databases are depicted in a tree structure, and can be identified with a database symbol. System databases have a lock symbol next to them to indicate that you cannot make changes to such databases. To view the tables defined for each database, double-click the database name. You can also click the round symbol (on UNIX) or the plus (+) symbol (on Windows) that is shown to the left of the database name. Tables are depicted with a table symbol. You can collapse a list of expanded tables by clicking the round symbol (on UNIX) or the minus (-) symbol (on Windows) that is shown to the left of the database name.

You can view and configure table and column information for each non-system database listed.

Use the Column Definitions tab to view information about the columns in any selected table. This information includes:
  • The unique name assigned to the column
  • The data type
  • The maximum length of characters allowed
  • An indication of whether the column is a primary key
  • An indication of whether the value of this column must be specified in the initial INSERT command
  • An indication of whether the value of this column can be changed after the initial INSERT command
  • An indication of whether the column is hidden

Use the Data View tab to view data contained in the selected table. The table columns are depicted as headers, and the data is shown in the rows.

Tip: To refresh any displayed information, click the symbol for the currently-selected database or table, or click Refresh in the toolbar.
