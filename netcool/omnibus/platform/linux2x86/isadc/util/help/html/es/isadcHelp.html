Introducción

La herramienta IBM® Support Assistant Data Collector se proporciona como ayuda para resolver los problemas de los productos de software de IBM. La herramienta se centra en la recogida automática de datos de problemas. También proporciona soporte de análisis de síntomas para las distintas categorías de los problemas encontrados por los productos de software de IBM. Se recoge y analiza la información pertinente de un tipo de problema para facilitar la identificación del origen del problema investigado. La herramienta ayuda a los clientes reduciendo la cantidad de tiempo necesario para reproducir un problema con los niveles de rastreo adecuados establecidos, así como el esfuerzo necesario para enviar la información de registro apropiada al soporte de IBM. En esta guía de ayuda, se le indica cómo iniciar y ejecutar la herramienta IBM Support Assistant Data Collector.


Accesibilidad

IBM® se esfuerza por proporcionar productos que puedan ser utilizados por todas las personas, cualquiera que sea su edad o capacidad. La interfaz de IBM ISA Data Collector Console da soporte a operaciones de sólo teclado y también se puede utilizar con aplicaciones de lectura de pantalla.


Paso a paso

En la mayoría de los casos, la siguiente secuencia de pasos le guiará para iniciar y ejecutar la herramienta. Si se experimenta problemas, o si desea más información sobre cualquiera de estos pasos, puede hacer referencia a las secciones siguientes que siguen a ésta.

  1. Instale la herramienta extrayendo los archivos desde el archivo de archivado que ha descargado.
    • Extraiga la herramienta en cualquier directorio de su elección.
    • Consulte las secciones siguientes para obtener detalles sobre cómo realizar las extracciones.
  2. Ejecute la herramienta en la modalidad de GUI o bien en la modalidad de consola de línea de mandatos.
    • Si se ejecuta en modalidad de consola de línea de mandatos, siga el procedimiento descrito a continuación para definir la variable de entorno JAVA_HOME. Una vez haya hecho esto, podrá ejecutar el script de lanzamiento.
    • Una vez que haya iniciado la herramienta, siga las instrucciones siguientes para interactuar con la misma cuando realiza una recogida de datos.

Instalación de herramientas

Instalación de la herramienta

En todos los casos, la instalación de la herramienta IBM Support Assistant Data Collector es una simple cuestión de extraer los archivos desde el archivador que ha descargado. Los archivos se pueden extraer en cualquier ubicación del sistema que elija en el sistema donde ejecutará la herramienta. Esto creará un subdirectorio \isadc en este directorio de destino.

Utilización de la herramienta

Definición de la variable de entorno JAVA_HOME

Si se ejecuta la herramienta en modalidad de consola de línea de mandatos, invoque el script de lanzamiento adecuado desde una línea de mandatos. En el caso de un sistema Windows, estos scripts de lanzamiento son archivos de proceso por lotes. Para los demás entornos, se trata de scripts de shell.

Puesto que la herramienta se implementa como una aplicación Java, es necesario definir la variable de entorno JAVA_HOME antes de que se pueda iniciar la herramienta. Si no está disponible Java en la PATH, tendrá que definir la variable de entorno JAVA_HOME manualmente. La herramienta IBM Support Assistant Data Collector requiere un JRE en el nivel 1.5 o superior, por lo que, en primer lugar, debe asegurarse de que está instalado un JRE apropiado en el sistema donde se ejecutará la herramienta. En caso afirmativo, deberá emitir un mandato específico del sistema operativo para definir la variable JAVA_HOME para indicar este JRE. El Microsoft JVM/JDK y gij (GNU libgcj) no están soportados. Por ejemplo, si tiene jdk1.5 instalado en c:\jre1.5 en una plataforma Windows, deberá definir JAVA_HOME utilizando el mandato siguiente:

SET JAVA_HOME=c:\jre1.5

En una plataforma Linux, AIX, HP-UX o Solaris, la sintaxis exacta del mandato para definir JAVA_HOME varían en función del shell que se utiliza. Por ejemplo, si utiliza un shell bash y tiene JDK instalado en /opt/jre15, deberá definir JAVA_HOME mediante el mandato siguiente:

export JAVA_HOME=/opt/jre15

Inicio de la herramienta en modalidad GUI

Abra el archivo index.html ubicado en el directorio de instalación isadc en un navegador web.

Inicio de la herramienta en la modalidad de consola de línea de mandatos

Si no está disponible una GUI, inicie la herramienta en modalidad de línea de comandos. Ejecute el script de lanzamiento desde la línea de mandatos.

  1. En un entorno Windows, ejecute isadc.bat en el directorio \isadc.
  2. En un entorno Linux, AIX, HP-UX o Solaris, ejecute isadc.sh desde el directorio /isadc.

Ubicación de archivos creados durante la ejecución

De forma predeterminada, el directorio de instalación isadc se utiliza para almacenar los archivos creados durante la ejecución. En algunos sistemas, el directorio de instalación isadc será de sólo lectura. En este caso, utilice el parámetro -useHome (sólo en modalidad de consola). El parámetro provocará que los archivos temporales se escriban en el directorio temporal de los sistemas y que los archivos persistentes se escriban en el directorio de inicio del usuario.

Interacción con la herramienta en modalidad GUI

Una vez que se ha iniciado la herramienta IBM Support Assistant Data Collector en un navegador web, se mostrará la interfaz gráfica.

Antes de que se pueda utilizar la herramienta para realizar una recopilación y análisis de datos, primero debe seleccionar un tipo de problema en la página principal.

Una vez seleccionado el tipo de problema, el siguiente paso es pulsar el botón Iniciar. Durante el proceso, el script de recopilación de datos le pedirá cualquier información adicional que necesite para completar sus actividades de recopilación de datos. Un script también le podría solicitar información de configuración, información sobre la secuencia de sucesos que genera el problema con el que está trabajando, o las preferencias respecto a cómo se debe completar la recogida de datos. Una vez que disponga de toda la información necesaria, el script continuará con el resto de la recogida.

Cuando la herramienta completa la recopilación de datos, puede enviar los resultados al servicio de soporte de IBM. Puede elegir entre FTP, que está sin cifrar, y HTTPS, que está cifrado, para la transferencia de archivos. El nombre del archivo zip que la herramienta envía al servicio de soporte de IBM se mostrará en el área de estado de la recopilación.

Interacción con la herramienta en la modalidad de consola de línea de mandatos

Si IBM Support Assistant Data Collector se ejecuta en la modalidad de consola de línea de mandatos, no hay ninguna lista de selección, ni ningún campo de entrada para la entrada de usuario. En lugar de ello, se presentan opciones disponibles en forma de listas numeradas, y el usuario especifica el número de la selección seguido de la tecla Intro. Los campos de entrada se transforman en solicitudes, en las que se especifica la respuesta y se pulsa Intro.

Para detener la herramienta de recopilador, escriba la opción quit en modalidad de consola.

La herramienta le solicitará los nombres de archivos. En las plataformas Unix, el uso de "~" como designación del directorio HOME del usuario no está soportado. Si se utiliza "~", se hará referencia a un subdirectorio debajo del directorio de trabajo actual por el nombre de "~".

Determinación de la información de versión

De forma predeterminada, la versión de la herramienta (y sus distintos subcomponentes) se imprime en la consola desde la que se ha iniciado. Al realizar la ejecución en la modalidad de GUI, la información de versión también se puede encontrar utilizando la opción del menú Ayuda->Acerca de. Si Java no está disponible, o la aplicación Java no se puede iniciar, también puede obtener la versión de la herramienta ejecutando los scripts de lanzamiento con la opción "-version".

Al utilizar la opción "-version", los scripts de lanzamiento imprimirán la información de versión sin invocar realmente la herramienta.

Soporte de proxy para las subidas HTTPS

De forma predeterminada, las subidas utilizarán un proxy definido por el sistema. En windows, el JRE utiliza la configuración de proxy de Internet Explorer. En Unix, JRE utiliza la configuración de proxy de GNOME 2. Para sobrescribir este valor y especificar otro proxy, utilice la opción "-promptForProxy" (modalidad de sólo consola)
