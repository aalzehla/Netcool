Introduzione

Lo strumento IBM® Support Assistant Data Collector offre un ausilio per la risoluzione dei problemi relativi ai prodotti software IBM. Lo strumento si concentra sulla raccolta automatica dei dati relativi al problema. Inoltre, offre il supporto per l'analisi dei sintomi per diverse categorie di problemi rilevati dai prodotti software IBM. Le informazioni relative ad un tipo di problema vengono raccolte e analizzate per aiutare l'utente ad identificare l'origine del problema in questione. Lo strumento assiste i clienti, riducendo il tempo impiegato per riprodurre un problema con i livelli di traccia appropriati impostati e riducendo lo sforzo richiesto per inviare le informazioni di log appropriate al supporto IBM. In questa guida viene illustrato come iniziare ad utilizzare lo strumento IBM Support Assistant Data Collector.


Accesso facilitato

IBM® è impegnata a fornire prodotti che possano essere utilizzati da tutti, indipendentemente dall'età o dall'abilità di ognuno. L'interfaccia della console di IBM ISA Data Collector supporta il funzionamento solo tramite la tastiera e può anche essere utilizzata con applicazioni di lettura dello schermo.


Istruzioni dettagliate

Nella maggior parte dei casi, la seguente sequenza di operazioni consentirà all'utente di iniziare ad utilizzare lo strumento. In caso di problemi o se si desidera acquisire ulteriori informazioni su tali operazioni, è possibile consultare le sezioni riportate di seguito.

  1. Installare lo strumento, estraendo i file dal file di archivio che è stato scaricato.
    • Estrarre lo strumento in una directory a propria scelta.
    • Consultare le sezioni riportate di seguito per i dettagli su come eseguire le estrazioni.
  2. Eseguire lo strumento in modalità GUI o in modalità console della riga comandi.
    • Se si sceglie l'esecuzione in modalità console della riga comandi, attenersi alla procedura descritta di seguito per impostare la variabile di ambiente JAVA_HOME. Dopo avere effettuato tale operazione, è possibile eseguire lo script di avvio.
    • Dopo avere avviato lo strumento, attenersi alle istruzioni riportate di seguito per interagire con esso mentre esegue una raccolta.

Installazione dello strumento

Installazione dello strumento

In tutti i casi, l'installazione dello strumento IBM Support Assistant Data Collector consiste nell'estrarre i file dall'archivio che è stato scaricato. I file possono essere estratti in un'ubicazione qualsiasi del file system scelta dall'utente, sul sistema su cui verrà eseguito lo strumento. In tal modo, verrà creata una directory secondaria \isadc in questa directory di destinazione.

Utilizzo dello strumento

Impostazione della variabile di ambiente JAVA_HOME

Se lo strumento viene eseguito in modalità console della riga comandi, richiamare lo script di avvio appropriato da una riga comandi. Nel caso di un sistema Windows, questi script di avvio sono i file batch. Per gli altri ambienti, si tratta degli script della shell.

Poiché lo strumento viene implementato come un'applicazione Java, per poterlo avviare è necessario impostare la variabile di ambiente JAVA_HOME. Se Java non è disponibile sul percorso, sarà necessario impostare manualmente la variabile di ambiente JAVA_HOME. Lo strumento IBM Support Assistant Data Collector richiede JRE livello 1.5 o un livello successivo, quindi è necessario verificare che sul sistema su cui verrà eseguito lo strumento sia installato un livello JRE appropriato. In caso affermativo, sarà necessario eseguire un comando specifico del sistema operativo per impostare la variabile JAVA_HOME, in modo che punti a questo JRE. Microsoft JVM/JDK e gij (GNU libgcj) non sono supportati. Ad esempio, se jdk1.5 è installato in c:\jre1.5 su una piattaforma Windows, impostare JAVA_HOME utilizzando il seguente comando:

SET JAVA_HOME=c:\jre1.5

Sulle piattaforme Linux, AIX, HP-UX o Solaris, la sintassi esatta del comando per impostare JAVA_HOME varia in base alla shell in uso. Ad esempio, se si utilizza la shell bash e JDK è installato in /opt/jre15, impostare JAVA_HOME utilizzando il seguente comando:

export JAVA_HOME=/opt/jre15

Avvio dello strumento in modalità GUI

È necessario aprire il file index.html, contenuto nella directory di installazione isadc, in un browser Web.

Avvio dello strumento in modalità console della riga comandi

Se non è disponibile una GUI, è necessario avviare lo strumento in modalità riga comandi. Eseguire lo script di avvio dalla riga comandi.

  1. In ambiente Windows, eseguire isadc.bat dalla directory \isadc.
  2. Negli ambienti Linux, AIX, HP-UX o Solaris, eseguire isadc.sh dalla directory /isadc.

Ubicazione dei file creati durante l'esecuzione

Per impostazione predefinita, la directory di installazione isadc viene utilizzata per memorizzare i file creati durante l'esecuzione. Su alcuni sistemi la directory di installazione isadc sarà in sola lettura. In questo caso, utilizzare il parametro -useHome (solo modalità console). Utilizzando tale parametro, i file temporanei verranno scritti nella directory temporanea dei sistemi e i file persistenti verranno scritti nella directory home dell'utente.

Interazione con lo strumento in modalità GUI

Dopo avere avviato lo strumento IBM Support Assistant Data Collector in un browser Web, verrà visualizzata l'interfaccia grafica.

Prima di utilizzare lo strumento per eseguire la raccolta e l'analisi dei dati, è necessario selezionare un tipo di problema dalla pagina principale.

Una volta selezionato il tipo di problema, premere il pulsante Avvia. Durante l'esecuzione, lo script di raccolta richiederà all'utente altre informazioni, necessarie per completare le attività di raccolta. Uno script può anche richiedere all'utente le informazioni di configurazione, le informazioni sulla sequenza di eventi da cui è scaturito il problema in questione oppure come preferisce che venga completata la raccolta. Dopo avere acquisito tutte le informazioni necessarie, lo script procederà con la parte rimanente della raccolta.

Quando lo strumento ha terminato la raccolta dei dati, è possibile inviare i risultati al supporto IBM. Per il trasferimento file è possibile scegliere tra FTP, senza codifica, ed HTTPS, con codifica. Il nome del file zip che lo strumento invia al supporto IBM verrà visualizzato nell'area di stato della raccolta.

Interazione con lo strumento in modalità console della riga comandi

Quando IBM Support Assistant Data Collector viene eseguito in modalità console della riga comandi, non sono disponibili elenchi di selezione o campi di immissione per l'input da parte dell'utente. Invece, le opzioni disponibili vengono presentate come elenchi numerati e l'utente immette il numero della selezione, quindi preme il tasto Invio. I campi di immissione vengono trasformati in prompt, in cui l'utente immette la risposta e preme Invio.

Per arrestare lo strumento Programma di raccolta, digitare l'opzione di uscita (quit) in modalità console.

Lo strumento richiederà i nomi dei file. Sulle piattaforme Unix, l'utilizzo di "~" per designare la directory HOME dell'utente non è supportato. Se si utilizza "~", verrà indicata una directory secondaria nella directory di lavoro attuale denominata "~".

Determinazione delle informazioni sulla versione

Per impostazione predefinita, la versione dello strumento (e dei suoi componenti secondari) viene indicata sulla console da cui è stato avviato lo strumento. Quando si sceglie l'esecuzione in modalità GUI, le informazioni sulla versione possono essere visualizzate anche selezionando l'opzione di menu Guida -> Informazioni su. Se Java non è disponibile oppure se non è possibile avviare l'applicazione Java, la versione dello strumento può essere determinata anche eseguendo gli script di avvio con l'opzione "-version".

Quando si utilizza l'opzione "-version", gli script di avvio mostreranno le informazioni sulla versione senza richiamare effettivamente lo strumento.

Supporto proxy per i caricamenti HTTPS

Per impostazione predefinita, i caricamenti utilizzano un proxy definito dal sistema. In Windows, JRE utilizza la configurazione proxy di Internet Explorer. In Unix, JRE utilizza la configurazione proxy di GNOME 2. Se si desidera specificare un proxy differente, utilizzare l'opzione "-promptForProxy" (solo modalità console).
