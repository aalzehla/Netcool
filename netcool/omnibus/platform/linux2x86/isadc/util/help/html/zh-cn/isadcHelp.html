简介

IBM® Support Assistant Data Collector 工具旨在协助您针对 IBM 软件产品的问题进行故障诊断。该工具的主要用途是自动收集问题数据。 它还针对 IBM 软件产品遇到的各类问题提供症状分析支持。 通过收集并分析与某种类型的问题有关的信息，有助于通过调查研究来找出问题的起源。 该工具通过设置正确的跟踪级别来缩短再现某个问题所花的时间，并且减少向 IBM 支持人员发送适当的日志信息所需的工作，为客户提供帮助。在此帮助指南中，我们将告诉您如何准备和运行 IBM Support Assistant Data Collector 工具。


辅助功能

IBM® 力求为不同年龄或能力的所有人提供可访问的产品。IBM ISA Data Collector 控制台界面支持仅使用键盘操作，并且还可与屏幕阅读应用程序配合使用。

  1. 通过从下载的归档文件解压缩文件来安装该工具。
    • 将该工具解压缩到您选择的任何目录。

  2. 以 GUI 方式或命令行控制台方式运行该工具。
    • 如果在命令行控制台方式下运行，那么遵循以下所述过程以设置 JAVA_HOME 环境变量。完成此过程之后，就可以开始执行启动脚本。
    • 一旦启动该工具之后，遵循以下指示信息以在其执行收集时与其交互。

工具安装

安装该工具

在任何情况下，安装 IBM Support Assistant Data Collector 工具都只需要将文件从您下载的归档中解压缩出来即可。可以将这些文件解压缩到您在将运行此工具的系统上选择的任何文件系统位置。这将在此目标目录中创建一个子目录 \isadc。

工具使用

设置 JAVA_HOME 环境变量

如果在命令行控制台方式下运行该工具，那么从命令行调用相应的启动脚本。在 Windows 系统中，这些启动脚本是批处理文件。 而在其他环境中，这些启动脚本是 shell 脚本。

由于此工具作为 Java 应用程序实现，因此，必须先设置 JAVA_HOME 环境变量才能启动此工具。 如果 PATH 中未提供 Java，那么必须手动设置 JAVA_HOME 环境变量。 IBM Support Assistant Data Collector 工具需要 1.5 或更高级别的 JRE，因此必须确保在将运行该工具的系统上已安装适合的 JRE。如果已满足此条件，那么您将需要发出一个特定于操作系统的命令将 JAVA_HOME 变量设置为指向此 JRE。 Microsoft JVM/JDK 和 gij (GNU libgcj) 不受支持。 例如，如果您在 Windows 平台的 c:\jre1.5 位置安装了 jdk1.5，那么应使用以下命令来设置 JAVA_HOME：

SET JAVA_HOME=c:\jre1.5

在 Linux、AIX、HP-UX 或 Solaris 平台上，用于设置 JAVA_HOME 变量的确切命令语法取决于您使用的 shell。 例如，如果使用 bash shell 且已在 /opt/jre15 位置安装了 JDK，那么应使用以下命令来设置 JAVA_HOME：

export JAVA_HOME=/opt/jre15

以 GUI 方式启动该工具

您将需要在 Web 浏览器中打开 isadc 安装目录下的 index.html 文件。

以命令行控制台方式启动该工具

如果 GUI 不可用，那么应以命令行方式启动该工具。只需从命令行执行启动脚本。

  1. 在 Windows 环境中，运行 \isadc 目录中的 isadc.bat。
  2. 在 Linux、AIX、HP-UX 或 Solaris 环境中，运行 /isadc 目录下的 isadc.sh。

在执行期间创建的文件的位置

缺省情况下，isadc 安装目录用来存储在执行期间创建的文件。在某些系统上，isadc 安装目录将为只读。在这种情况下，请使用 -useHome 参数（仅适用于控制台方式）。该参数会将临时文件写入系统临时目录，而将持久文件写入用户主目录。

以 GUI 方式与该工具交互

一旦在 Web 浏览器中启动 IBM Support Assistant Data Collector 之后，将显示图形界面。

必须先从首页中选择问题类型，然后才可将该工具用于执行数据收集和分析。

选择问题类型之后，下一个步骤是按“启动”按钮。随着收集过程的进行，收集脚本将要求您提供它完成收集活动所需的更多信息。 某个脚本可能还会要求您提供配置信息、有关导致您正在处理的问题的事件序列的信息，或者提供与其如何完成收集有关的首选项。一旦它获得了它需要的所有信息，脚本就会继续完成收集活动的其余任务。

当该工具完成数据收集时，可将结果发送给 IBM 支持机构。您可以选择使用 FTP（不加密），也可以选择 HTTPS（加密）进行文件传输。 该工具发送给 IBM 支持机构的 zip 文件的名称将显示在收集状态区域中。

以命令行控制台方式与该工具交互

当 IBM Support Assistant Data Collector 以命令行控制台方式运行时，没有用于用户输入的选择列表或输入字段。可用的选项是以编号列表形式提供的，在输入您所作选择的编号之后按 Enter 键。 输入字段将转换为提示，此时您需要在输入字段中输入回答并按 Enter 键。

要停止收集器工具，请在控制台方式下输入退出选项。

该工具会提示输入文件名。在 Unix 平台上，不支持将“~”指定为用户的主目录。如果使用了“~”，那么将引用当前工作目录下由“~”表示的子目录。

确定版本信息

缺省情况下，该工具的版本（及其各种子组件）会在启动它的控制台上显示。当在 GUI 方式下运行时，也可通过使用“帮助 -> 关于”菜单选项找到版本信息。如果 Java 不可用，或者 Java 应用程序无法启动，那么也可以通过运行带有“-version”选项的脚本来获取该工具的版本。

使用“-version”选项时，启动脚本将显示版本信息，但不实际调用该工具。

HTTPS 上载的代理支持

缺省情况下，上载将使用系统定义的代理。在 Windows 上，JRE 使用 Internet Explorer 的代理配置。在 Unix 上，JRE 使用 GNOME 2 的代理配置。要覆盖此配置并输入其他代理，请使用“-promptForProxy”选项（仅适用于控制台方式）。
