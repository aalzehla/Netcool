Einführung

IBM® Support Assistant Data Collector wird zur Unterstützung bei der Fehlersuche und -behebung bei IBM Softwareprodukten bereitgestellt. Das Tool dient schwerpunktmäßig der automatischen Erfassung problemrelevanter Daten. Außerdem stellt es Unterstützung für die Symptomanalyse der einzelnen Kategorien von Problemen und Fehler bereit, die bei IBM Softwareprodukten auftreten können. Die zu einem Fehler oder Problem zugehörigen Informationen werden erfasst und analysiert, um Aufschluss zum Ursprung des Problems zu liefern, das gerade untersucht wird. Das Tool verringert den Zeitaufwand zum Reproduzieren eines Problems mit den korrekten Traceebenen und reduziert den Aufwand, der für das Senden der geeigneten, für die IBM Unterstützungsfunktion erforderlichen Protokollinformationen anfällt. In diesem Hilfeleitfaden erfahren Sie, wie Sie das Tool IBM Support Assistant Data Collector in Betrieb nehmen und bedienen.

Schrittweise Anleitung

In den meisten Fällen können Sie das Tool in den folgenden Schritten installieren und einsatzbereit machen. Wenn Probleme auftreten oder weitere Informationen zu einem dieser Schritte gewünscht werden, können Sie die Abschnitte lesen, die auf diesen Abschnitt folgen.

  1. Installieren Sie das Tool, indem Sie die Dateien aus dem heruntergeladenen Archiv extrahieren, .
    • Extrahieren Sie das Tool in ein beliebiges Verzeichnis, das Sie auswählen.
    • Die nachfolgenden Abschnitte enthalten detaillierte Informationen dazu, wie die Extraktion ausgeführt wird.
  2. Führen Sie das Tool entweder im GUI-Modus oder im Befehlszeilenkonsolmodus aus.
    • Wenn Sie im Befehlszeilenkonsolmodus führen Sie die im Folgenden beschriebene Prozedur aus, um die Umgebungsvariable JAVA_HOME festzulegen. Anschließend können Sie das Startscript ausführen.
    • Folgen Sie, nachdem das Tool gestartet worden ist, den unten beschriebenen Anweisungen zur Interaktion mit dem Tool während der Ausführung einer Erfassung.

Installation des Tools

Tool installieren

In allen Fällen besteht die Installation des Tools IBM Support Assistant Data Collector lediglich darin, die Dateien aus dem heruntergeladenen Archiv zu extrahieren. Die Dateien können an eine beliebige Dateisystemposition extrahiert werden, die Sie auf dem System auswählen, auf dem das Tool ausgeführt werden soll. Daraufhin wird ein Unterverzeichnis \isadc in diesem Zielverzeichnis erstellt.

Verwendung des Tools

Umgebungsvariable JAVA_HOME festlegen

Wenn Sie das Tool im Befehlszeilenkonsolmodus ausführen, rufen Sie das entsprechende Startscript in einer Befehlszeile auf. Bei einem Windows-System sind diese Startscripts Batchdateien. In anderen Umgebungen sind sie Shell-Scripts.

Da das Tool als Java-Anwendung implementiert ist, muss die Umgebungsvariable JAVA_HOME festgelegt werden, damit das Tool überhaupt gestartet werden kann. Wenn Java in der Variablen PATH nicht verfügbar ist, müssen Sie die Umgebungsvariable JAVA_HOME manuell festlegen. Das Tool IBM Support Assistant Data Collector erfordert Java Runtime Environment (JRE) 1.5 oder eine höhere Version. Daher müssen Sie zunächst sicherstellen, dass eine entsprechende JRE auf dem System installiert ist, auf dem das Tool ausgeführt werden soll. Ist eine solche JRE vorhanden, müssen Sie mithilfe eines betriebssystemspezifischen Befehls die Variable JAVA_HOME so definieren, dass sie auf diese JRE verweist. Die JVM/JDK von Microsoft sowie gij ('GNU libgcj') werden nicht unterstützt. Beispiel: Wenn 'jdk1.5' im Verzeichnis 'c:\jre1.5' auf einer Windows-Plattform installiert ist, muss die Variable JAVA_HOME mit dem folgenden Befehl festgelegt werden:

SET JAVA_HOME=c:\jre1.5

Auf Linux-, AIX-, HP-UX- Solaris-Plattformen hängt die genaue Befehlssyntax zum Festlegen von JAVA_HOME je von der verwendeten Shell ab. Wenn Sie zum Beispiel Bash-Shell verwenden und die JDK im Verzeichnis '/opt/jre15' installiert ist, müssten Sie die Variable JAVA_HOME mit folgendem Befehl festlegen:

export JAVA_HOME=/opt/jre15

Tool im GUI-Modus starten

Sie müssen die Datei index.html im Installationsverzeichnis isadc in einem Web-Browser öffnen.

Tool im Befehlszeilenkonsolmodus starten

Wenn keine GUI verfügbar ist, muss das Tool im Befehlszeilenmodus gestartet werden. Führen Sie einfach das Startscript über die Befehlszeile aus.

  1. Führen Sie in einer Windows-Umgebung die Datei isadc.bat im Verzeichnis \isadc aus.
  2. Führen Sie in einer Linux-, AIX-, HP-UX- oder Solaris-Umgebung die Datei isadc.sh im Verzeichnis /isadc aus.

Position von Dateien, die während der Ausführung erstellt wurden

Standardmäßig wird das Installationsverzeichnis isadc zum Speichern von Dateien verwendet, die während der Ausführung erstellt wurden. Auf manchen Systemen ist das Installationsverzeichnis isadc schreibgeschützt. In diesem Fall müssen Sie den Parameter '-useHome' verwenden (nur im Konsolmodus). Dieser Parameter bewirkt, dass temporäre Dateien in das temporäre Verzeichnis des Systems und als persistent definierte Dateien in das Ausgangsverzeichnis des Benutzers geschrieben werden.

Mit dem Tool in GUI-Modus interagieren

Nachdem Sie das Tool IBM Support Assistant Data Collector in einem Web-Browser gestartet haben, wird die grafische Schnittstelle angezeigt.

Damit das Tool zum Ausführen einer Datenerfassung und -analyse verwendet werden kann, müssen Sie zunächst auf der Hauptseite einen Problemtyp auswählen.

Nach der Auswahl des Problemtyps klicken Sie auf die Schaltfläche Starten. Bei der Verarbeitung werden Sie vom Erfassungsscript zur Angabe aller zusätzlichen Informationen aufgefordert, die für die vollständige Ausführung der Erfassungsaktivitäten erforderlich sind. Ein Script kann Sie auch auffordern, Konfigurationsinformationen bereitzustellen, Informationen zu der Folge von Ereignissen anzugeben, die dem gegenwärtig vorliegenden Problem vorangegangen sind, oder Einstellungen zur Durchführung und Beendigung der Erfassung festzulegen. Nachdem alle benötigten Informationen bereitgestellt wurden, fährt das Script mit dem Rest der Datenerfassung fort.

Nachdem das Tool die Datenerfassung abgeschlossen hat, können Sie die Ergebnisse an die IBM Unterstützungsfunktion senden. Bei der Dateiübertragung können Sie zwischen der unverschlüsselten FTP-Übertragung und der verschlüsselten HTTPS-Übertragung wählen. Der Name der ZIP-Datei, die das Tool an den IBM Support sendet, wird im Statusbereich für die Erfassung angezeigt.

Mit dem Tool im Befehlszeilenkonsolmodus interagieren

Wenn IBM Support Assistant Data Collector im Befehlszeilenkonsolmodus ausgeführt wird, sind keine Auswahllisten oder Eingabefelder für Benutzereingaben verfügbar. Stattdessen werden die verfügbaren Auswahlmöglichkeiten in Form von nummerierten Listen angezeigt. Um eine Option auszuwählen, geben Sie die Nummer für die entsprechende Auswahlmöglichkeit ein und drücken anschließend die Eingabetaste. Eingabefelder werden in Eingabeaufforderungen umgesetzt, in die Sie Ihre Antwort eingeben und anschließend die Eingabetaste drücken.

Zum Stoppen des Collector-Tools geben Sie im Konsolmodus die Option quit ein.

Das Tool fordert zur Angabe von Dateinamen auf. Auf UNIX-Plattformen wird die Verwendung von "~" als Bezeichnung für das Ausgangsverzeichnis (HOME) des Benutzers nicht unterstützt. Wird "~" verwendet, so wird ein Unterverzeichnis unter dem gegenwärtigen Arbeitsverzeichnis namens "~" referenziert.

Versionsinformationen ermitteln

Standardmäßig wird die Version des Tools (und seiner diversen Unterkomponenten) auf der Konsole ausgegeben, auf der es gestartet wurde. Wenn die Ausführung im GUI-Modus erfolgt, so können die Versionsinformationen auch im Hilfemenü mit der Option für die Produktinformationen abgefragt werden. Ist Java nicht verfügbar oder sollte die Java-Anwendung nicht gestartet werden können, können Sie die Version des Tools auch erfahren, indem Sie die Startscripts mit der Option "-version" ausführen.

Bei Verwendung der Option "-version" geben die Startscripts die Versionsinformationen aus, ohne dass jedoch das Tool tatsächlich aufgerufen wird.

Proxy-Unterstützung für HTTPS-Uploads

Standardmäßig wird bei Uploads ein systemdefinierter Proxy verwendet. Unter Windows verwendet die JRE die Proxykonfiguration von Internet Explorer. Unter UNIX verwendet die JRE die Proxy-Konfiguration von GNOME 2. Wenn Sie diese Einstellung überschreiben und einen anderen Proxy eingeben möchten, verwenden Sie die Option "-promptForProxy" (nur Konsolmodus).
