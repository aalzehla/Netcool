소개

IBM® Support Assistant Data Collector 도구는 IBM 소프트웨어 제품의 문제점 해결을 돕는 지원 기능을 제공합니다. 이 도구는 문제점 데이터의 자동 콜렉션에 초점을 둡니다. 또한 이 도구는 IBM 소프트웨어 제품에 발생하는 다양한 유형의 문제점에 대한 증상 분석 지원을 제공합니다. 문제점 유형에 관련된 정보가 콜렉션 및 분석되어 조사 중인 문제점의 근본 원인을 식별하게 됩니다. 이 도구를 통해 고객은 적합한 추적 레벨 세트로 문제점을 재현하는 데 걸리는 시간을 줄이는 동시에 IBM Support에 적절한 로그 정보를 전송하기 위해 필요한 수고도 줄일 수 있습니다. 이 도움말 안내서에서는 IBM Support Assistant Data Collector 도구를 시작하여 실행하는 방법에 대해 설명합니다.


내게 필요한 옵션

IBM®은 나이와 능력에 상관없이 누구나 액세스할 수 있는 제품을 제공하려고 노력하고 있습니다. IBM ISA Data Collector 콘솔 인터페이스는 키보드 전용 조작을 지원하며 화면 판독 애플리케이션과 함께 사용할 수도 있습니다.


단계별 안내

대부분의 경우에는 다음과 같은 일련의 단계를 수행하여 도구를 시작하고 실행합니다. 문제가 발생하거나 이러한 단계에 대한 자세한 정보를 원하는 경우에는 아래 섹션을 참조하십시오.

  1. 다운로드한 아카이브 파일에서 파일 압축을 풀어서 도구를 설치합니다.
    • 선택한 디렉토리에 도구의 압축을 풉니다.
    • 압축을 푸는 방법에 대한 세부사항은 아래 섹션을 참조하십시오.
  2. GUI 모드 또는 명령행 콘솔 모드에서 도구를 실행합니다.
    • 명령행 콘솔 모드에서 실행하는 경우, 아래에 설명된 프로시저에 따라서 JAVA_HOME 환경 변수를 설정하십시오. 이 프로시저를 수행하면 실행 스크립트를 실행할 수 있습니다.
    • 도구를 시작한 경우 콜렉션을 수행하는 경우와 마찬가지로 상호작용을 위해 아래 지시사항을 따르십시오.

도구 설치

도구 설치

IBM Support Assistant Data Collector 도구를 설치하려면 다운로드한 아카이브에서 압축된 파일을 풀기만 하면 됩니다. 이 파일은 도구를 실행할 시스템에서 선택한 파일 시스템 위치로 압축을 풀 수 있습니다. 그러면 이 대상 디렉토리에 서브디렉토리 \isadc가 작성됩니다.

도구 사용법

JAVA_HOME 환경 변수 설정

명령행 콘솔 모드에서 도구를 실행하는 경우, 명령행에서 적절한 실행 스크립트를 호출하십시오. Windows 시스템의 경우 이러한 실행 스크립트는 일괄처리 파일입니다. 기타 환경의 경우에는 쉘 스크립트입니다.

이 도구는 Java 애플리케이션으로 구현되므로 도구를 시작하려면 먼저 JAVA_HOME 환경 변수를 설정해야 합니다. PATH에서 Java를 사용할 수 없는 경우 직접 JAVA_HOME 환경 변수를 설정해야 합니다. IBM Support Assistant Data Collector 도구를 사용하려면 JRE 레벨 1.5 이상이 필요하므로 먼저 도구를 실행할 시스템에 적절한 JRE가 설치되었는지 확인해야 합니다. 적절한 JRE가 설치된 경우 운영 체제에 특정한 명령을 실행하여 JAVA_HOME 변수가 이 JRE를 가리키도록 설정해야 합니다. Microsoft JVM/JDK 및 gij(GNU libgcj)는 지원되지 않습니다. 예를 들어 Windows 플랫폼에서 jdk1.5가 c:\jre1.5에 설치된 경우, 다음 명령을 사용하여 JAVA_HOME을 설정합니다.

SET JAVA_HOME=c:\jre1.5

Linux, AIX, HP-UX 또는 Solaris 플랫폼의 경우 JAVA_HOME을 설정하는 정확한 명령 구문은 사용 중인 쉘에 따라 다릅니다. 예를 들어, bash 셀을 사용하는 경우 JDK가 /opt/jre15에 설치되어 있으면 다음 명령을 사용하여 JAVA_HOME을 설정합니다.

export JAVA_HOME=/opt/jre15

GUI 모드에서 도구 시작

isadc 설치 디렉토리에 있는 index.html 파일을 웹 브라우저에 열어야 합니다.

명령행 콘솔 모드에서 도구 시작

GUI를 사용할 수 없는 경우 도구를 명령행 모드에서 시작해야 합니다. 명령행에서 실행 스크립트를 실행하십시오.

  1. Windows 환경의 경우 \isadc 디렉토리에서 isadc.bat을 실행하십시오.
  2. Linux, AIX, HP-UX 또는 Solaris 환경의 경우 /isadc 디렉토리에서 isadc.sh를 실행하십시오.

실행 중에 작성된 파일의 위치

기본적으로 isadc 설치 디렉토리는 실행 중에 작성된 파일을 저장하는 데 사용됩니다. 일부 시스템에서는 isadc 설치 디렉토리가 읽기 전용입니다. 이 경우에는 -useHome 매개변수를 사용합니다(콘솔 모드 전용). 이 매개변수를 사용하면 임시 파일은 시스템 임시 디렉토리에 쓰여지고 지속 파일은 사용자 홈 디렉토리에 쓰여집니다.

GUI 모드에서 도구와 상호작용

IBM Support Assistant Data Collector 도구가 웹 브라우저에서 시작되면 그래픽 인터페이스가 표시됩니다.

데이터 콜렉션 및 분석을 위해 도구를 사용하려면 먼저 기본 페이지에서 문제점 유형을 선택해야 합니다.

문제점 유형이 선택되면 다음 단계는 시작 단추를 누르는 것입니다. 이 단계가 진행되면 콜렉션 스크립트는 콜렉션 활동을 완료하기 위해 필요한 추가 정보를 사용자에게 요청합니다. 스크립트도 구성 정보, 처리하는 문제점의 발생 과정에 있는 일련의 이벤트에 대한 정보 또는 사용자가 선호하는 콜렉션 완료 방법을 요청할 수 있습니다. 필요한 정보가 모두 제공되면 스크립트는 나머지 콜렉션을 진행합니다.

도구가 데이터 콜렉션을 완료하면 결과를 IBM Support에 전송할 수 있습니다. 파일 전송을 위해 암호화되지 않은 FTP와 암호화된 HTTPS 사이에서 선택할 수 있습니다. 도구가 IBM 지원 센터에 전송하는 Zip 파일의 이름은 콜렉션 상태 영역에 표시됩니다.

명령행 콘솔 모드에서 도구와 상호작용

IBM Support Assistant Data Collector가 명령행 콘솔 모드에서 실행되면 사용자 입력을 위한 입력 필드나 선택사항 목록이 없습니다. 대신, 번호가 있는 목록으로 사용 가능한 선택사항이 제시되므로 해당 번호를 입력한 후 Enter 키를 눌러 선택할 수 있습니다. 입력 필드는 프롬프트로 변환되고, 여기서 사용자 응답을 입력한 후 Enter 키를 누를 수 있습니다.

콜렉터 도구를 중지하려면 콘솔 모드에서 종료 옵션을 입력하십시오.

파일 이름을 입력하라는 메시지가 표시됩니다. Unix 플랫폼에서는 "~"를 사용자의 HOME 디렉토리로 지정할 수 없습니다. "~"가 사용된 경우에는 이름이 "~"인 현재 작업 중인 디렉토리 아래의 서브디렉토리가 참조됩니다.

버전 정보 판별

기본적으로 도구의 버전(및 다양한 서브컴포넌트)은 실행된 콘솔에 인쇄됩니다. GUI 모드에서 실행하는 경우에는 도움말 -> 정보 메뉴 옵션을 사용하여 버전 정보를 찾을 수도 있습니다. Java를 사용할 수 없거나 Java 애플리케이션이 시작될 수 없는 경우에는 "-version" 옵션을 사용하여 실행 스크립트를 실행하여 도구의 버전을 가져올 수도 있습니다.

"-version" 옵션을 사용하는 경우 실행 스크립트는 도구를 실제로 호출하지 않고 버전 정보를 인쇄합니다.

HTTPS 업로드에 대한 프록시 지원

기본적으로 업로드에는 시스템 정의 프록시를 사용합니다. Windows의 경우 JRE는 Internet Explorer의 프록시 구성을 사용합니다. Unix의 경우 JRE는 GNOME 2의 프록시 구성을 사용합니다. 이 구성을 대체하고 다른 프록시를 입력하려면 "-promptForProxy" 옵션을 사용하십시오(콘솔 모드 전용).
