Введение

Программа IBM® Support Assistant Data Collector - это вспомогательное средство для устранения неполадок в программных продуктах IBM. Данный инструмент предназначен для автоматического сбора данных о неполадках. С помощью него можно также анализировать симптомы различных неполадок продуктов IBM. С целью выявления причины исследуемой неполадки выполняется поиск и анализ связанной информации. Данный инструмент уменьшает время, необходимое для воспроизведения неполадки с заданными уровнями трассировки, а также облегчает процедуру сбора информации из протоколов. В данном руководстве приведена информация об установке и использовании программы IBM Support Assistant Data Collector.


Специальные возможности

Компания IBM® стремится предоставлять продукты, которые может использовать кто угодно вне зависимости от возраста и физических возможностей. Интерфейс консоли IBM ISA Data Collector поддерживает только ввод с клавиатуры и совместим с программами чтения с экрана.


Пошаговое знакомство

В большинстве случаев следующая последовательность шагов позволит вам приступить к работе с программой. В случае затруднений или для получения дополнительной информации о любом из этих шагов обратитесь к последующим разделам.

  1. Для установки инструмента извлеките файлы из загруженного архивного файла.
    • Извлеките инструмент в любой каталог.
    • Подробная информация об извлечении содержится в следующих разделах.
  2. Запустите инструмент в режиме графического пользовательского интерфейса или в режиме консоли командной строки.
    • При работе в режиме консоли с командной строкой настройте переменную среды JAVA_HOME, выполнив описанную ниже процедуру. После этого можно выполнить сценарий запуска.
    • После запуска инструмента следуйте приведенным инструкциям для взаимодействия с инструментом в процессе сбора данных.

Установка программы

Установка программы

Для установки программы IBM Support Assistant Data Collector достаточно извлечь файлы из загруженного архива. Файлы могут быть извлечены в любое место файловой системы, выбранное вами для запуска инструмента. При этом в целевом каталоге будет создан подкаталог \isadc.

Использование программы

Настройка переменной среды JAVA_HOME

Если программа запущена в режиме консоли с командной строкой, то вызовите соответствующий сценарий запуска из командной строки. В случае системы Windows эти сценарии запуска представляют собой пакетные файлы. В других средах они являются сценариями командного процессора.

Поскольку инструмент реализуется как приложение Java, перед его запуском необходимо задать переменную среды JAVA_HOME. Если Java в PATH отсутствует, то придется настроить переменную среды JAVA_HOME вручную. Для программы IBM Support Assistant Data Collector необходима среда JRE версии 1.5 или более поздней, поэтому перед запуском программы убедитесь в наличии совместимой среды JRE. Если она установлена, необходимо с помощью команды операционной системы настроить переменную JAVA_HOME так, чтобы она указывала на эту JRE. Microsoft JVM/JDK и gij (GNU libgcj) не поддерживаются. Например, если jdk1.5 установлен в каталоге c:\jre1.5 на платформе Windows, то переменную среды JAVA_HOME можно настроить с помощью следующей команды:

SET JAVA_HOME=c:\jre1.5

В Linux, AIX, HP-UX и Solaris синтаксис команды, задающей переменную JAVA_HOME, зависит от используемой командной оболочки. Например, если используется оболочка bash, и JDK установлен в каталоге /opt/jre15, то для настройки JAVA_HOME следует использовать следующую команду:

export JAVA_HOME=/opt/jre15

Запуск программы в графическом режиме

Откройте файл index.html из каталога установки isadc в веб-браузере.

Запуск программы в режиме командной строки

Если графический интерфейс недоступен, программу можно запустить в режиме командной строки. Для этого следует вызвать сценарий запуска из командной строки.

  1. В системе Windows запустите файл isadc.bat из каталога \isadc.
  2. В системе Linux, AIX, HP-UX или Solaris запустите файл isadc.sh из каталога /isadc.

Размещение файлов, создаваемых во время установки

По умолчанию каталог установки isadc используется для хранения файлов, созданных во время выполнения. В ряде систем каталог установки isadc доступен только для чтения. В этом случае следует использовать параметр -useHome (в режиме консоли). При этом временные файлы будут записываться в системный каталог временных файлов, а остальные - в домашний каталог пользователя.

Взаимодействие с программой в графическом режиме

После запуска программы IBM Support Assistant Data Collector в веб-браузере будет показан графический интерфейс программы.

Перед тем как приступать к сбору и анализу данных с помощью программы необходимо выбрать тип неполадки на главной странице.

После выбора типа неполадки необходимо нажать кнопку Запустить. При выполнении сценария сбора данных может быть запрошена дополнительная информация. Например, сценарий может запросить имя установочного каталога продукта, дополнительные сведения о конфигурации, информацию о последовательности событий, вызывающих неполадку, или ваши предпочтения относительно сбора данных. После сбора всей нужной информации сценарий выполняет заключительную часть операции.

После завершения сбора данных можно отправить результаты в службу IBM Support. Можно выбрать передачу данных по FTP (незашифрованное соединение), либо по HTTPS (зашифрованное соединение). Имя файла zip, отправляемого в IBM Support, показано в области состояния сбора данных.

Взаимодействие с программой в режиме командной строки

При использовании IBM Support Assistant Data Collector в режиме консоли с командной строкой отсутствуют списки вариантов или поля для ввода данных пользователем. Доступные опции представлены в виде пронумерованных списков, в которых требуется ввести номер нужной опции и нажать клавишу Enter. Вместо полей ввода применяются приглашения, в которых требуется ввести ответ и нажать клавишу Enter.

Для завершения работы инструмента сбора данных введите quit в режиме консоли.

Инструмент запросит имена файлов. На платформах Unix не поддерживается использование символа "~" в качестве обозначения домашнего каталога пользователя. Данный символ используется для обозначения подкаталога.

Определение версии

По умолчанию версия инструмента и его компонентов выводится в консоль, из был запущен данный инструмент. При работе в графическом режиме сведения о версиях компонентов можно просмотреть в меню Справка->О программе. Если Java недоступна или приложение Java не может быть запущено, то версию инструмента можно просмотреть с помощью опции "-version".

При наличии опции "-version" информация о версии будет выведена без запуска инструмента.

Поддержка Proxy для загрузок по протоколу HTTPS

По умолчанию при передаче данных используется системный Proxy. В Windows в качестве параметров Proxy для JRE используются настройки Internet Explorer. В Unix среда JRE использует параметры Proxy, заданные для GNOME 2. Для переопределения Proxy укажите параметр "-promptForProxy" (в режиме консоли).
