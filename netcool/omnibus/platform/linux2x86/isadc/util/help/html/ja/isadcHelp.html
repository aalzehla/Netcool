概要

IBM® Support Assistant Data Collector ツールは、IBM ソフトウェア製品に関する問題のトラブルシューティングの支援機能として提供されています。このツールは、問題データの自動的な収集に特化しています。また、IBM ソフトウェア製品において発生するさまざまなカテゴリーの問題に対する症状分析の支援も行います。問題のタイプに関する情報が収集および分析され、調査中の問題の原因の特定を支援します。 このツールは、適切なログ情報を IBM サポートに送信するために必要となる労力を削減し、また適切なトレース・レベル・セットで問題を再現する時間を短縮することで、ユーザーを支援します。このヘルプ・ガイドでは、IBM Support Assistant Data Collector ツールを開始して稼働させる方法をご説明します。


アクセシビリティー

IBM® は、年齢や障害の有無にかかわらず誰もが使用できる製品の提供に努めています。IBM ISA Data Collector のコンソール・インターフェースは、 キーボードのみの操作をサポートし、画面読み取りのアプリケーションと共に使用することもできます。


ステップバイステップ

ほとんどの場合、以下の一連のステップによってツールを稼働状態にすることができます。問題が生じた場合、またはこれらのステップのいずれかの詳細が必要な場合には、この後のセクションを参照してください。

  1. ダウンロードしたアーカイブ・ファイルからファイルを解凍して、ツールをインストールします。
    • 選択した任意のディレクトリーにツールを解凍します。
    • 解凍の実行方法の詳細については、以降のセクションを参照してください。
  2. ツールを GUI モードまたはコマンド行コンソール・モードで実行します。
    • コマンド行コンソール・モードで実行中の場合は、下記の手順に従って JAVA_HOME 環境変数を設定してください。これを行うと、起動スクリプトを実行できるようになります。
    • ツールの開始後に、収集を実行する際に、下記の説明に従って対話します。

ツールのインストール

ツールのインストール

すべての場合で、IBM Support Assistant Data Collector ツールのインストールは、単に、ダウンロードしたアーカイブからファイルを解凍するだけのことです。ファイルは、ツールを実行するシステム上で選択した任意のファイル・システムの場所に解凍できます。これにより、このターゲット・ディレクトリーにサブディレクトリー \isadc が作成されます。

ツールの使用法

JAVA_HOME 環境変数の設定

ツールをコマンド行コンソール・モードで実行中の場合は、コマンド行から該当する起動スクリプト を呼び出します。Windows システムの場合は、起動スクリプトはバッチ・ファイルです。 他の環境の場合は、シェル・スクリプトです。

ツールは Java アプリケーションとして実装されるため、ツールの開始前に JAVA_HOME 環境変数を設定する必要があります。 PATH に Java がない場合には、JAVA_HOME 環境変数を手動で設定する必要があります。 IBM Support Assistant Data Collector ツールでは JRE のレベル 1.5 以上が必要であるため、 まず、ツールを実行するシステムに適切なJRE がインストールされていることを確認してください。インストールされている場合には、オペレーティング・システム固有のコマンドを発行して、JAVA_HOME 変数をその JRE を指すように設定する必要があります。 Microsoft JVM/JDK および gij (GNU libgcj) はサポートされていません。例えば、Windows プラットフォームで、jdk1.5 が c:\jre1.5 にインストールされている場合、 次のコマンドを使用して JAVA_HOME を設定します。

SET JAVA_HOME=c:\jre1.5

Linux、AIX、HP-UX、または Solaris のプラットフォームでは、JAVA_HOME を設定するための正確なコマンド構文は、使用するシェルによって異なります。例えば、bash シェルを使用していて、JDK が /opt/jre15 にインストールされている場合は、次のコマンドを使用して JAVA_HOME を設定します。

export JAVA_HOME=/opt/jre15

GUI モードでツールを開始する

isadc インストール・ディレクトリーにある index.html ファイルを Web ブラウザーで開く必要があります。

コマンド行コンソール・モードでツールを開始する

GUI が使用できない場合、ツールはコマンド行モードで開始します。コマンド行から起動スクリプトを実行します。

  1. Windows 環境では、\isadc ディレクトリーから isadc.bat を実行します。
  2. Linux、AIX、HP-UX、または Solaris の環境では、/isadc ディレクトリーから isadc.sh を実行します。

実行中に作成されたファイルの場所

デフォルトでは、実行中に作成されたファイルを保管するために、isadc インストール・ディレクトリーが使用されます。一部のシステムでは、isadc インストール・ディレクトリーは読み取り専用です。この場合、-useHome パラメーターを使用します (コンソール・モードのみ)。このパラメーターにより、一時ファイルはシステムの一時ディレクトリーに書き込まれ、パーシスタント・ファイルはユーザーのホーム・ディレクトリーに書き込まれます。

GUI モードでツールと対話する

IBM Support Assistant Data Collector ツールが Web ブラウザーで開始されると、グラフィカル・インターフェースが表示されます。

ツールを使用してデータの収集および分析を行う前に、まずメインページから問題のタイプを選択する必要があります。

問題タイプを選択したら、次のステップは「開始」ボタンを押すことです。続行するにあたり、収集アクティビティーを完了するために必要な追加情報を、コレクション・スクリプトが要求します。また、スクリプトから、構成情報、現在対処している問題につながるイベントの順序、または収集の完了方法に関する設定について尋ねられる場合があります。 必要な情報がすべて揃ったら、スクリプトは、残りの収集を続行します。

ツールがデータの収集を完了したら、その結果を IBM サポートに送信することができます。ファイル転送の方法は、FTP (非暗号化)、HTTPS (暗号化) のいずれかを選択できます。ツールによって IBM サポートに送信される zip ファイルの名前が、コレクション状況域に表示されます。

コマンド行コンソール・モードでツールと対話する

IBM Support Assistant Data Collector をコマンド行コンソール・モードで実行する場合、ユーザーが入力を行うための選択リストや入力フィールドは使用できません。その代わりに、有効な選択項目が番号付きとして示され、選択項目の番号を入力してから Enter キーを押します。入力フィールドは、ユーザーが応答を入力して Enter を押すと、プロンプトに変わります。

コレクター・ツールを停止するには、コンソール・モードで quit オプションを入力します。

ツールがプロンプトを出し、ファイル名を要求します。Unix プラットフォームでは、ユーザーのホーム・ディレクトリーの指定における「~」の使用はサポートされていません。「~」が使用された場合、現在の作業ディレクトリー内の「~」という名前のサブディレクトリーが参照されます。

バージョン情報の判別

デフォルトでは、ツール (およびその各種サブコンポーネント) のバージョンは、そのツールが起動されたコンソールに出力されます。GUI モードで実行中、バージョン情報は「ヘルプ」->「製品情報」メニュー・オプションを使用して確認することもできます。 Java が使用できない場合、または Java アプリケーションを始動できない場合、「-version」オプションを指定して起動スクリプトを実行して、そのツールのバージョンを取得することができます。

「-version」オプションを使用すると、起動スクリプトは実際にツールを起動することなくバージョン情報を出力します。

HTTPS アップロードのプロキシー・サポート

デフォルトでは、アップロードはシステム定義のプロキシーを使用します。Windows では、JRE は Internet Explorer のプロキシー構成を使用します。Unix では、JRE は GNOME 2 のプロキシー構成を使用します。これを無効にして異なるプロキシーを入力するには、「-promptForProxy」オプションを使用します (コンソール・モードのみ)。
