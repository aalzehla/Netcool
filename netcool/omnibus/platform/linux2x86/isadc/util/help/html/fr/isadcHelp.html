Introduction

L'outil IBM® Support Assistant Data Collector est fourni pour aider à identifier et résoudre les problèmes liés aux produits logiciels IBM. Cet outil recueille automatiquement les données d'incident. Il offre également une prise en charge de l'analyse des symptômes des différentes catégories d'incidents rencontrés par les produits logiciels IBM. Les informations pertinentes pour un type d'incident sont collectées et analysées pour permettre d'identifier l'origine de l'incident dans le cadre d'un examen détaillé. L'outil aide les clients en réduisant le temps nécessaire à la reproduction des niveaux de suivi corrects définis, ainsi qu'en réduisant l'effort nécessaire à l'envoi des informations de journal appropriées au support IBM. Ce guide explique comment installer et utiliser l'outil IBM Support Assistant Data Collector.


Accessibilité

IBM® s'efforce de fournir des produits accessibles à tous, quel que soit l'âge ou les capacités. L'interface IBM ISA Data Collector Console permet d'exécuter des opérations à partir du clavier uniquement et elle peut être également utilisée avec des applications de lecteur d'écran.


Etape par étape

Dans la plupart des cas, la séquence d'étapes suivante va vous permettre d'optimiser votre utilisation de l'outil. Si vous rencontrez des problèmes ou si vous avez besoin d'informations supplémentaires sur l'une des étapes, vous pouvez consulter les sections qui suivent la présente section.

  1. Installez l'outil en procédant à l'extraction des fichiers du fichier archive que vous avez téléchargé.
    • Extrayez l'outil dans le répertoire de votre choix.
    • Reportez-vous aux sections ci-dessous pour obtenir des détails sur les modalités d'exécution des extractions.
  2. Exécutez l'outil en mode interface graphique ou en mode console de ligne de commande.
    • Si vous utilisez le mode Console de ligne de commande, suivez la procédure décrite ci-dessous pour définir la variable d'environnement JAVA_HOME. Après quoi, vous pouvez exécuter le script de lancement.
    • Une fois que vous avez lancé l'outil, suivez les instructions ci-dessous pour interagir avec ce dernier au moment d'une collecte.

Installation de l'outil

Installation de l'outil

Dans tous les cas, l'installation de l'outil IBM Support Assistant Data consiste simplement à extraire les fichiers de l'archive que vous avez téléchargée. Il est possible d'extraire les fichiers vers n'importe quel emplacement de système de fichiers que vous choisissez, sur le système où vous exécuterez l'outil. Un sous-répertoire \isadc est ainsi créé sous ce répertoire cible.

Utilisation de l'outil

Définition de la variable d'environnement JAVA_HOME

Si vous utilisez l'outil en mode Console de ligne de commande, exécutez le script de lancement approprié depuis une ligne de commande. Dans le cas d'un système Windows, ces scripts de lancement sont des fichiers de commandes. Pour les autres environnements, il s'agit de scripts de shell.

Comme l'outil est implémenté en tant qu'application Java, il est nécessaire de définir la variable d'environnement JAVA_HOME avant que l'outil puisse démarrer. Si Java n'est pas disponible dans la variable PATH, vous devez définir la variable d'environnement JAVA_HOME manuellement. L'outil IBM Support Assistant Data Collector requiert un environnement JRE de niveau 1.5 ou supérieur ; vous devez donc vous assurer, dans un premier temps, qu'un environnement JRE adéquat est installé sur le système où l'outil sera exécuté Si tel est le cas, vous devez émettre une commande propre au système d'exploitation pour définir la variable JAVA_HOME de façon à ce qu'elle pointe vers cet environnement JRE. Microsoft JVM/JDK et gij (GNU libgcj) ne sont pas pris en charge. Par exemple, si vous avez installé jdk1.5 dans c:\jre1.5 sur une plateforme Windows, définissez la variable d'environnement JAVA_HOME en utilisant la commande suivante :

SET JAVA_HOME=c:\jre1.5

Sur une plateforme Linux, AIX, HP-UX ou Solaris, la syntaxe de commande exacte pour définir JAVA_HOME varie en fonction de l'interpréteur de commandes que vous utilisez. Par exemple, si vous utilisez un interpréteur de commandes bash et que le JDK est installé dans /opt/jre15, définissez JAVA_HOME à l'aide de la commande suivante :

export JAVA_HOME=/opt/jre15

Démarrage de l'outil en mode Interface graphique

Vous devrez ouvrir le fichier index.html dans le répertoire d'installation isadc dans un navigateur Web.

Démarrage de l'outil en mode Console de ligne de commande

Si aucune interface graphique n'est disponible, l'outil doit être démarré automatiquement en mode Ligne de commande. Exécutez simplement le script de lancement depuis la ligne de commande.

  1. Dans un environnement Windows, exécutez isadc.bat depuis le répertoire \isadc.
  2. Dans un environnement Linux, AIX, HP-U ou Solaris, exécutez isadc.sh depuis le répertoire /isadc.

Emplacement des fichiers créés lors de l'exécution

Par défaut, le répertoire d'installation isadc est utilisé pour enregistrer les fichiers créés lors de l'exécution. Sur certains systèmes, ce répertoire est en lecture seule. Dans ce cas, utilisez le paramètre -useHome (mode Console uniquement). Ce paramètre permet d'écrire les fichiers temporaires dans le répertoire temporaire du système et les fichiers permanents dans le répertoire de base de l'utilisateur.

Interaction avec l'outil dans le mode Interface graphique

Une fois que l'outil IBM Support Assistant Data Collector a été démarré dans un navigateur Web, l'interface graphique s'affiche.

Pour pouvoir utiliser l'outil pour collecter et analyser des données, vous devez sélectionner un type de problème dans la page principale.

Une fois le type de problème sélectionné, cliquez sur le bouton Démarrer. Ensuite, le script de collecte va vous demander les autres informations dont il a besoin pour achever ses activités de collecte. Un script peut également vous demander des informations de configuration, des informations sur la séquence d'événements à l'origine de l'incident que vous gérez ou vos préférences concernant la façon de terminer la collecte. Une fois toutes les informations réunies, le script poursuit la collecte.

Lorsque l'outil a terminé la collecte des données, vous pouvez envoyer les résultats au support IBM. Pour le transfert de fichiers, vous pouvez choisir entre FTP, qui n'est pas chiffré, et HTTPS, qui est chiffré. Le nom du fichier compressé que l'outil envoie au support IBM s'affiche dans la zone d'état de la collecte.

Interaction avec l'outil en mode Console de ligne de commande

Lorsque l'outil IBM Support Assistant Data Collector s'exécute en mode Console de ligne de commande, il n'existe aucune liste de sélection ni zone d'entrée destinée à l'utilisateur. Les choix disponibles sont présentés sous la forme de listes numérotées ; vous entrez le numéro de votre sélection et vous appuyez sur la touche Entrée. Les zones de saisie sont transformées en invites, dans lesquelles vous entrez votre réponse avant d'appuyer sur la touche Entrée.

Pour arrêter l'outil de collecte, tapez quit en mode Console.

L'outil demande des noms de fichier. Sur les plateformes Unix, l'utilisation de "~" pour désigner le répertoire de base de l'utilisateur n'est pas prise en charge. Si "~" est utilisé, un sous-répertoire dans le répertoire de travail actuel avec le nom de "~" sera référencé.

Détermination des informations de version

Par défaut, la version de l'outil (et ses différents sous-composants) est affichée sur la console à partir de laquelle il a été lancé. Lors d'une exécution en mode interface graphique, les informations de version sont également disponibles à partir de l'option de menu Aide->A propos de. Si Java n'est pas disponible, ou si l'application Java ne peut pas démarrer, vous pouvez également obtenir la version de l'outil en exécutant les scripts de lancement avec l'option "-version".

Lorsque vous utilisez l'option "-version", les scripts de lancement afficheront les informations de version sans réellement appeler l'outil.

Support de proxy pour les envois HTTPS

Par défaut, les envois utiliseront un proxy défini par le système. Sous Windows, l'environnement JRE utilise la configuration du proxy de l'Explorateur Internet. Sous Unix, l'environnement JRE utilise la configuration de proxy GNOME 2. Pour remplacer ce proxy et entrer un proxy différent, utilisez l'option "-promptForProxy" (mode Console uniquement).
