簡介

「IBM® Support Assistant Data Collector 工具旨在協助疑難排解 IBM 軟體產品的問題。此工具著重於自動收集問題資料。此外，也針對 IBM 軟體產品所發現的各種種類問題，提供症狀分析支援。它會收集並分析和某種問題類型有關的資訊，以協助找出調查中問題的根源。The tool assists customers by reducing the amount of time 此工具可協助您縮短重建問題（設有適當追蹤層次）的時間 ，以及減少將適當日誌資訊傳送至「IBM 支援中心」 所需的心力。在此說明手冊中， 我們會告訴您如何啟動並執行 IBM Support Assistant Data Collector 工具。


協助工具

IBM® 致力於為每個人（不管年齡或能力）提供可存取的有用產品。IBM ISA Data Collector Console 介面支援全鍵盤作業，並且可以與螢幕讀取應用程式搭配使用。


逐步

在大多數情況下，執行下列步驟序列可讓您啟動並執行此工具。如果您遇到問題，或需要其中任何步驟的相關資訊，則可以參閱此步驟下面的區段。

  1. 若要安裝此工具，請解壓縮所下載保存檔中的檔案。
    • 將此工具解壓縮到您所選擇的任何目錄。
    • 請參閱下列章節，以取得有關如何執行解壓縮作業的詳細資料。
  2. 在 GUI 模式或指令行主控台模式下執行此工具。
    • 如果是在指令行主控台模式下執行，請遵循如下說明的程序來設定 JAVA_HOME 環境變數。在執行此作業後，您可以執行啟動 Script。
    • 在啟動此工具以執行收集程序後，請遵循下面的指示，與此工具進行互動。

工具安裝

安裝工具

無論如何，只需解壓縮所下載保存檔中的檔案，即可完成 IBM Support Assistant Data Collector 工具的安裝作業。可以將這些檔案解壓縮到預期執行此工具的檔案系統位置 (您可以在系統上選擇任何檔案系統位置來執行此工具)。這會在此目標目錄中建立子目錄 \isadc。

工具用法

設定 JAVA_HOME 環境變數

如果是在指令行主控台模式下執行此工具，請從指令行呼叫適當的啟動 Script。若為 Windows 系統，則這些啟動 Script 是批次檔。若為其他環境，則它們是 Shell Script。

由於此工具是實作為 Java 應用程式，因此必須先設定 JAVA_HOME 環境變數才能啟動此工具。 如果 PATH 上沒有 Java，則您將必須手動設定 JAVA_HOME 環境變數。IBM Support Assistant Data Collector 工具需要層次 JRE 1.5 或更高版本，因此您必須先確定預期執行此工具的系統上有安裝適當的 JRE。如果有安裝適當的 JRE，則您將需要發出作業系統特定指令，以便將 JAVA_HOME 變數設為指向此 JRE。 不支援 Microsoft JVM/JDK 及 gij (GNU libgcj)。例如，在 Windows 平台上，如果您將 jdk1.5 安裝在 c:\jre1.5 中，請使用下列指令來設定 JAVA_HOME：

SET JAVA_HOME=c:\jre1.5

在 Linux、AIX、HP-UX 或 Solaris 平台上，用來設定 JAVA_HOME 變數的確切指令語法視您使用的 Shell 而定。例如，如果您使用 Bash Shell，且將 JDK 安裝在 /opt/jre15 中，請使用下列指令來設定 JAVA_HOME：

export JAVA_HOME=/opt/jre15

在 GUI 模式下啟動工具

您將需要在 Web 瀏覽器中開啟位於 isadc 安裝目錄的 index.html 檔。

在指令行主控台模式下啟動工具

如果無法使用 GUI，則會在指令行模式下啟動此工具。只要從指令行執行啟動 Script 即可。

  1. 在 Windows 環境中，請從 \isadc 目錄執行 isadc.bat。
  2. 在 Linux、AIX、HP-UX 或 Solaris 環境中，請從 /isadc 目錄執行 isadc.sh。

執行期間所建立檔案的位置

依預設，isadc 安裝目錄會用來儲存在執行期間所建立的檔案。在某些系統上，isadc 安裝目錄是唯讀目錄。在此實例中，請使用 -useHome 參數（僅限主控台模式）。 此參數會使暫存檔寫入系統暫存目錄，而持續檔案則會寫入使用者起始目錄。

在 GUI 模式下與工具互動

在 Web 瀏覽器中啟動 IBM Support Assistant Data Collector 工具之後，即會顯示圖形介面。

您必須先從主頁面中選取問題類型，才能使用此工具來執行資料收集及分析。

選取問題類型之後，下一步是按「啟動」按鈕。作業繼續執行時，收集 Script 會要求您提供完成其收集活動所需的任何其他資訊。Script 可能也會要求您提供配置資訊、會導致您目前所處理問題的事件順序相關資訊，或者是您希望如何完成收集的相關喜好設定。一旦它具備所有的必要資訊後，Script 會繼續處理後段的收集程序。

當此工具完成資料收集時，您可以將結果傳送給「IBM 支援中心」。您可以選擇使用未加密的 FTP 或加密的 HTTPS 來傳送檔案。收集狀態區中將會顯示此工具傳送給「IBM 支援中心」的 ZIP 檔名稱。

在指令行主控台模式下與工具互動

當 IBM Support Assistant Data Collector 在指令行主控台模式下執行時，不會提供選項清單或輸入欄位供使用者輸入。可選的選項採編號清單呈現，請輸入您的選項編號，然後按 Enter 鍵。輸入欄位則轉變成提示，請輸入您的回應，然後按 Enter 鍵。

若要停止收集器工具，請在主控台模式下鍵入 quit 選項。

此工具會提示您輸入檔名。在 Unix 平台上，不支援使用 "~" 來指定使用者的 HOME 目錄。如果使用 "~"，則會參照目前工作目錄下名稱為 "~" 的子目錄。

判斷版本資訊

依預設，此工具（及其各種子元件）的版本會列印到在其中啟動此工具的主控台中。在 GUI 模式下執行時，使用「說明 -> 關於」功能表選項，也可以找到版本資訊。 如果 Java 無法使用，或者 Java 應用程式無法啟動，則也可以藉由執行含有 "-version" 選項的啟動 Script，來取得此工具的版本。

使用 "-version" 選項時，啟動 Script 會印出版本資訊，但不會實際呼叫此工具。

HTTPS 上傳的 Proxy 支援

上傳會使用系統定義的 Proxy。在 Windows 上，JRE 會使用 Internet Explorer 的 Proxy 配置。在 Unix 上，JRE 會使用 GNOME 2 的 Proxy 配置。若要置換這個 Proxy 配置並輸入不同的 Proxy，請使用 "-promptForProxy" 選項（僅限主控台模式）。
